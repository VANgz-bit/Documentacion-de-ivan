## 🔹 ¿Qué es `display`?

La propiedad `display` determina **el tipo de caja de renderizado** de un elemento:

- Si es un **bloque**, **en línea**, **flexible**, **rejilla**, etc.

- Cómo **interactúa con otros elementos** en el layout de la página.


## 🔸 Categorías principales de valores `display`

### 1. **Valores clásicos**

|Valor|Descripción|
|---|---|
|`block`|El elemento se comporta como un **bloque**. Ocupa todo el ancho disponible. Se apila uno debajo del otro. Ej: `<div>`, `<p>`, `<section>`.|
|`inline`|El elemento se comporta como **en línea**. Solo ocupa el espacio de su contenido. No acepta `width`, `height`, `margin-top`, etc. Ej: `<span>`, `<a>`.|
|`inline-block`|Como `inline`, pero **permite tamaño (`width`/`height`)**. Ideal para botones y menús.|
|`none`|El elemento **no se renderiza**, no ocupa espacio, es como si no existiera.|
|`contents`|Elimina el contenedor, pero **mantiene los hijos en el DOM**. Ojo con accesibilidad y estilos heredados.|

---

### 2. **Modelo de cajas modernas**

|Valor|Descripción|
|---|---|
|`flex`|Convierte al elemento en un **contenedor flexible** (Flexbox). Permite distribuir espacio entre hijos (`flex items`) fácilmente.|
|`inline-flex`|Lo mismo que `flex`, pero el contenedor es en línea.|
|`grid`|Activa **CSS Grid**, un sistema de rejillas de filas y columnas. Mucho más controlado que Flexbox para layouts complejos.|
|`inline-grid`|Igual que `grid`, pero el contenedor es en línea.|

---

### 3. **Valores de listas y tablas**

|Valor|Descripción|
|---|---|
|`list-item`|Se comporta como un elemento de lista (`<li>`). Añade viñetas o números automáticamente.|
|`table`|Se comporta como una **tabla HTML**.|
|`inline-table`|Igual que `table`, pero se comporta como elemento en línea.|
|`table-row`, `table-cell`, `table-caption`, etc.|Usados para **construir tablas manualmente** con `div`s o sin usar etiquetas `<table>`.|
