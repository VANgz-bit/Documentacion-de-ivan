### ¿Qué son las Container Queries?

Las **Conteiner Queries** (consultas de contenedor) son una nueva característica de CSS que permite aplicar estilos a un elemento **basado en el tamaño de su contenedor**, no del `viewport` como sucede con los **media queries** tradicionales.
Esto es ideal para componentes que necesitan ser _responsivos_ según el espacio disponible dentro de su contenedor, sin depender del tamaño total de la ventana.


### ¿Cómo se usan?

#### 1. Define el contenedor con `container-type`

Primero tienes que indicar qué elemento actuará como _contenedor_:

.card-container {
  container-type: inline-size; /* o size */
}

- `inline-size`: solo mide el ancho del contenedor.
    
- `size`: mide ancho y alto.

También puedes darle un nombre opcional con `container-name`.

.card-container {
  container-type: inline-size;
  container-name: card;
}


#### 2. Escribe la container query

Luego, aplicas estilos a los elementos hijos basándote en el tamaño del contenedor:

@container (min-width: 400px) {
  .card {
    flex-direction: row;
  }
}

O si usaste un nombre:

@container card (min-width: 400px) {
  .card {
    flex-direction: row;
  }
}

### Compatibilidad

Las container queries están **soportadas por todos los navegadores modernos**, incluyendo Chrome, Edge, Firefox y Safari recientes. Si estás aprendiendo responsive, es una técnica moderna y poderosa que complementa muy bien los `media queries`.
