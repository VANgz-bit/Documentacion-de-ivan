
## 1. ¿Qué son las variables en CSS?

Las **variables en CSS**, también llamadas **Custom Properties**, son una funcionalidad del lenguaje CSS que permite **almacenar valores en una especie de contenedor reutilizable**, con el objetivo de mejorar la organización, la consistencia y el mantenimiento del código.

A diferencia de otras propiedades CSS, su valor **puede cambiarse en distintos contextos**, como media queries, pseudoclases, selectores o incluso dinámicamente desde JavaScript. Esta flexibilidad convierte a las variables en una herramienta poderosa, especialmente en proyectos grandes o con múltiples temas (modo claro/oscuro, diseño responsive, etc.).


## 2. ¿Cómo se escriben y se usan?

Las variables CSS siempre deben tener **dos guiones delante (`--`)** en su nombre. Se definen dentro de un selector y se accede a ellas mediante la función `var()`.


:root {
  --color-principal: #3498db;
  --espaciado: 1rem;
}

.card {
  background-color: var(--color-principal);
  padding: var(--espaciado);
}

- `:root` es un selector especial que apunta al elemento raíz del documento (`<html>`), ideal para definir **variables globales**.
    
- `--color-principal` y `--espaciado` son nombres de variables personalizados.
    
- `var(--nombre-variable)` se usa para **acceder** al valor de esa variable en cualquier lugar del CSS.


## 3. Ventajas clave de las variables CSS

###  Reutilización

Podemos usar el mismo valor en múltiples lugares, lo que evita repeticiones.

###  Mantenimiento

Si necesitamos cambiar un color, una medida o una fuente, solo lo modificamos en un lugar.

###  Cascada e herencia

Como cualquier propiedad CSS, las variables se comportan según la cascada: **pueden ser sobrescritas por otras variables más específicas** o definidas dentro de contextos limitados.

###  Fallback (valor alternativo)

Podemos indicar un segundo valor que se use si la variable no está definida:

color: var(--no-existe, #000);


## 4. Limitaciones importantes de las variables

Las variables CSS son muy útiles, pero tienen algunas limitaciones:

- **No pueden ser usadas para definir selectores** (solo valores de propiedades).
    
- **No se pueden animar** por defecto, a menos que se use una extensión como `@property` (ver más abajo).
    
- **No hacen cálculos automáticamente**, aunque podemos combinarlas con `calc()`:


width: calc(var(--ancho-base) * 2);

## 5. ¿Qué es `@property`?

`@property` es una **regla avanzada de CSS** que permite **registrar una variable CSS personalizada con un tipo de dato específico, una configuración de herencia y un valor inicial**.

Esta regla forma parte del módulo **CSS Houdini** y resuelve una limitación fundamental: por defecto, **las variables CSS no pueden transicionarse ni animarse**, ya que el navegador no sabe qué tipo de valor tienen.

Con `@property`, se le indica al navegador qué tipo de dato es la variable, permitiéndole **interpolar valores entre estados (animación)** y optimizar el rendimiento.


Sintaxis de `@property`

@property --nombre-variable {
  syntax: '`<tipo>`';
  inherits: true | false;
  initial-value: valor;
}

#### Descripción de cada parámetro:

- **`--nombre-variable`**: nombre de la variable, siempre comenzando con `--`.
    
- **`syntax:`** indica el tipo de valor que puede tener. Ejemplos:
    
    - `<color>` → colores (`#fff`, `rgb(0,0,0)`, etc.)
        
    - `<length>` → medidas (`10px`, `1rem`)
        
    - `<number>` → valores numéricos sin unidad
        
    - `<percentage>` → porcentajes (`50%`)
        
    - `<angle>` → ángulos (`45deg`, `0.5turn`)
        
- **`inherits:`**
    
    - `true`: la variable se hereda desde su elemento padre (como muchas propiedades CSS).
        
    - `false`: no se hereda.
        
- **`initial-value:`** valor inicial de la variable si no se define otra cosa.


### Ejemplo con animación

@property --color-tema {
  syntax: '`<color>`';
  inherits: false;
  initial-value: red;
}

:root {
  --color-tema: red;
}

.box {
  background-color: var(--color-tema);
  transition: --color-tema 1s;
}

.box:hover {
  --color-tema: blue;
}

En este ejemplo, cuando pasamos el mouse por encima del `.box`, el color de fondo transiciona de rojo a azul. Eso solo es posible porque `--color-tema` fue registrada con `@property`.


## 6. Diferencias entre variables normales y con `@property`

| Característica                    | Variable común (`--`) | Registrada con `@property` |
| --------------------------------- | --------------------- | -------------------------- |
| Reutilizable                      | si                    | si                         |
| Se puede redefinir en media query | si                    | si                         |
| Funciona con herencia             | si                    | si / configurable          |
| Animable                          | no                    | si                         |
| Declara tipo de valor             | no                    | si                         |
| Tiene valor inicial por defecto   | no                    | si                         |
| Requiere soporte moderno          | no (muy compatible)   | ⚠️ (no en Firefox aún)     |

---

## 7. Soporte de navegador (actualizado a 2025)

- **Google Chrome, Microsoft Edge, Safari** → ✅ Soportan `@property`.
    
- **Mozilla Firefox** → ❌ No lo soporta completamente todavía (puede cambiar pronto).
    
- Las variables normales funcionan en **todos los navegadores modernos desde hace años**.

## 8. ¿Qué es CSS Houdini?

**CSS Houdini** es un conjunto de APIs modernas que permiten a los desarrolladores web **acceder y extender el motor de renderizado de CSS**, es decir, crear comportamientos y propiedades personalizadas que antes solo eran posibles con hacks o JavaScript externo.

Houdini **"abre la caja negra" de CSS**, dando más control sobre cómo se interpretan y dibujan los estilos en pantalla. Esto permite optimizaciones más profundas y personalizaciones visuales más avanzadas.

### Algunas características clave:

- Crear propiedades CSS personalizadas con `@property`.
    
- Leer y escribir valores CSS directamente desde JavaScript.
    
- Animar propiedades personalizadas.
    
- Dibujar directamente en el fondo de un elemento (`Paint API`).
    
- Medir y adaptar estilos al layout (`Layout API`, `Typed OM`, etc).
    

### ¿Qué tiene que ver con las variables CSS?

La regla `@property` que viste antes **forma parte de CSS Houdini**. Gracias a Houdini, se puede registrar una variable con su tipo y animarla correctamente, cosa que antes era imposible con CSS puro.
