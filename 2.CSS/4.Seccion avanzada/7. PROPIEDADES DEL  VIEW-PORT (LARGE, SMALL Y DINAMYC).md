
## ¿Qué son las unidades del viewport?

Las unidades del viewport (`vw`, `vh`, `vmin`, `vmax`) son **unidades relativas al tamaño de la ventana gráfica del navegador**. Sirven para diseñar elementos que se adapten al tamaño de la pantalla sin importar la resolución o el dispositivo.

---

## Unidades clásicas del viewport

### 1. `vw` – Viewport Width

- Representa un **1% del ancho** del viewport.
    
- `100vw` equivale al ancho total de la pantalla.
    

**Ejemplo**:

```css
div {
  width: 50vw; /* ocupa la mitad del ancho de la pantalla */
}
```

### 2. `vh` – Viewport Height

- Representa un **1% del alto** del viewport.
    
- `100vh` cubre toda la altura visible del navegador.
    

**Problema clásico**: en **móviles**, `100vh` puede causar problemas porque no toma en cuenta la barra de direcciones (que aparece y desaparece al hacer scroll). Esto puede hacer que el contenido se corte o no encaje correctamente.

### 3. `vmin` y `vmax`

- `vmin`: 1% del **menor** valor entre el ancho y el alto del viewport.
    
- `vmax`: 1% del **mayor** valor entre el ancho y el alto del viewport.
    

**Ejemplo**:

```css
font-size: 5vmin; /* la fuente se adapta al tamaño más pequeño entre ancho y alto */
```

---

## Nuevas unidades del viewport

A partir de CSS nivel 4 (muy útil para móviles), se introdujeron nuevas unidades que **resuelven problemas de inconsistencia** en altura al usar `vh`.

### 1. `lvh` – **Large Viewport Height**

- Equivale al **100% del alto del viewport cuando la barra de dirección NO está visible**.
    
- Ideal para mostrar contenido a **pantalla completa**.
    

### 2. `svh` – **Small Viewport Height**

- Equivale al **100% del alto del viewport cuando la barra de dirección SÍ está visible**.
    
- Se usa cuando se necesita que el contenido se adapte incluso si la barra del navegador ocupa espacio.
    

### 3. `dvh` – **Dynamic Viewport Height**

- **Equivale al alto real del viewport en el momento actual**, considerando si la barra de navegación está o no.
    
- Esta es la unidad **más confiable** hoy en día para lograr una altura 100% en móviles.
    

**Ejemplo comparativo**:

```css
/* Uso moderno recomendado en móviles */
section {
  height: 100dvh;
}
```

---

## Comparativa general

|Unidad|Significado|Uso recomendado|
|---|---|---|
|`vh`|1% del alto del viewport (puede ser inconsistente)|No recomendada sola en móviles|
|`lvh`|Alto con barra oculta (modo expandido)|Pantallas completas|
|`svh`|Alto con barra visible (modo comprimido)|Elementos que no deben desbordar|
|`dvh`|Alto dinámico, actual|**Mejor opción** para responsive en móviles|
|`vw`|1% del ancho del viewport|Siempre útil|
|`vmin`/`vmax`|1% del mínimo o máximo entre ancho y alto|Escalado flexible|

---

## Buenas prácticas

- En diseño responsive moderno, **reemplazar `100vh` por `100dvh`** en la mayoría de los casos.
    
- En componentes fullscreen (por ejemplo, banners, headers), usar `100lvh` para garantizar el máximo espacio.
    
- Si el contenido debe estar 100% visible incluso con barras del navegador, usar `100svh`.
