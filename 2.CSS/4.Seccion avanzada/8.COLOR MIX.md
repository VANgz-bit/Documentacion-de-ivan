
## ¿Qué es `color-mix()`?

`color-mix()` es una **función CSS** que permite **mezclar dos colores** en una proporción específica, dentro de un **espacio de color determinado**.

Su sintaxis básica es:

```css
color-mix(in <espacio-de-color>, <color1> <porcentaje1>, <color2> <porcentaje2>)
```

- `in <espacio-de-color>`: especifica el espacio de color donde se hará la mezcla (como `srgb` o `lab`).
    
- `<color1>` y `<color2>`: son los colores que se mezclarán.
    
- `<porcentaje1>` y `<porcentaje2>`: son las proporciones en las que se mezclan. **Si se omite uno, se calcula automáticamente el restante**.
    

---

## Ejemplo simple

```css
elemento {
  background-color: color-mix(in srgb, red 50%, blue 50%);
}
```

Este mezcla rojo y azul a partes iguales en el espacio de color `srgb`, dando como resultado **púrpura**.

---

## ¿Qué es un espacio de color?

Un **espacio de color** define **cómo se representan y mezclan los colores**.

### Los más usados en `color-mix()`:

1. **`srgb` (Standard RGB)** – Es el espacio de color más común en pantallas. Es el predeterminado si no se especifica otro.
    
2. **`srgb-linear`** – Variante lineal de `srgb`, con resultados más realistas en mezclas.
    
3. **`lab`** – Basado en la percepción humana del color (más natural).
    
4. **`oklab`** – Espacio moderno, muy similar a `lab` pero más estable en medios digitales.
    
5. **`lch` / `oklch`** – Representan el color con brillo, croma y tono. Muy útiles para **generar colores accesibles y armónicos**.
    

---

## ¿Qué diferencia hay al cambiar el espacio?

Usar `lab` o `oklab` suele dar **transiciones de color más suaves y naturales**, mientras que `srgb` puede generar colores más intensos o forzados. Veamos una comparación:

```css
/* En srgb: puede salir un color saturado y chillón */
color-mix(in srgb, red 50%, blue 50%);

/* En lab: da un color más equilibrado y natural */
color-mix(in lab, red 50%, blue 50%);
```

---

## ¿Para qué sirve `color-mix()`?

### 1. **Diseño adaptativo**

Podés mezclar un color base con blanco o negro para **hacerlo más claro u oscuro** sin definirlo manualmente.

```css
color-mix(in srgb, blue 80%, white 20%); /* azul aclarado */
```

### 2. **Modo oscuro/claro**

Permite generar colores que se ajustan **automáticamente** mezclando tonos base con otros, sin duplicar estilos:

```css
background-color: color-mix(in srgb, var(--color-base) 70%, black 30%);
```

### 3. **Temas personalizados**

Podés mezclar colores usando **variables CSS**, ideal para temas dinámicos:

```css
:root {
  --primary: #0a66c2;
  --secondary: color-mix(in srgb, var(--primary) 60%, white 40%);
}
```

### 4. **Animaciones de color suaves**

Aunque no es directamente una animación, usar `color-mix()` con variables que cambian según el estado puede simular **transiciones suaves entre tonos**.

---

## Relación con otras tecnologías y funciones

### 1. **Variables CSS**

- Podés usar `color-mix()` junto con `var()` para crear sistemas de color flexibles y escalables.
    

### 2. **`color-contrast()`**

- Otra función moderna que **elige automáticamente un color que haga contraste suficiente**. Se puede combinar con `color-mix()` para accesibilidad.
    

### 3. **Modos de color como `light/dark`**

- `color-mix()` te permite **adaptar los colores a diferentes modos sin duplicar reglas**.
    

### 4. **Sistemas de diseño (tokens)**

- En diseño con tokens (como en Tailwind o Design Systems), `color-mix()` puede ayudar a generar variantes automáticamente (`hover`, `disabled`, `active`, etc).
    

---

## Compatibilidad

- Actualmente está **disponible en navegadores modernos** (Chrome, Edge, Firefox, Safari).
    
- Asegurate de usar un **fallback** si trabajás con navegadores antiguos.
    

---

## Buenas prácticas

- Usar **porcentajes que sumen 100%**, aunque no es obligatorio.
    
- Elegir el **espacio de color adecuado** para lo que se busca (ej: `oklab` para diseño accesible, `srgb` para cosas rápidas).
    
- Utilizar con **variables CSS** para mantener un sistema organizado y dinámico.
