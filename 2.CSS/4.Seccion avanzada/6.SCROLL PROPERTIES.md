
Como ya vimos anteriormente propiedades del scroll ligadas a las animaciones(scroll-linked animations) tenemos mas conceptos claves del scroll a lo largo de sus características. 

---

## 1. `overflow`

La propiedad `overflow` define **qué debe hacer el navegador cuando el contenido de un elemento se desborda** (es decir, cuando su tamaño excede el espacio disponible dentro de su contenedor).

### Valores posibles:

- **`visible`**:  
    Este es el valor por defecto. El contenido que se desborda simplemente se **muestra fuera del contenedor**, sin recorte ni barras de desplazamiento. Es útil cuando querés que algo sobresalga visualmente, pero puede romper el diseño si no se controla.
    
- **`hidden`**:  
    El contenido desbordado **se recorta y no se muestra**. No se genera scroll. Ideal cuando querés ocultar partes sobrantes que no te interesan o controlar efectos visuales específicos, como animaciones que se mueven fuera del contenedor.
    
- **`scroll`**:  
    Siempre genera una barra de desplazamiento (horizontal o vertical), **aunque no sea necesaria**. Esto asegura que siempre haya scroll disponible, útil en algunos diseños donde querés indicar que el contenido puede desplazarse.
    
- **`auto`**:  
    Muestra barras de scroll **solo si el contenido lo necesita**, es decir, si efectivamente se desborda. Es el comportamiento más común y recomendado para la mayoría de los diseños responsivos.
    
- **`clip`**:  
    Similar a `hidden`, recorta el contenido, pero **no permite desplazamiento en absoluto**. Es un valor más específico, a veces útil para combinaciones con otras propiedades visuales.
    

### También existen:

- `overflow-x`: controla el desbordamiento en eje horizontal.
    
- `overflow-y`: en eje vertical.


## 2. `scroll-behavior`

Esta propiedad define **cómo se comporta la animación del scroll** cuando ocurre una navegación interna, como hacer clic en un enlace con `#id`.

### Valores:

- **`auto`** (valor por defecto):  
    El desplazamiento ocurre de forma **instantánea**, sin animación.
    
- **`smooth`**:  
    El scroll ocurre de forma **suave y animada**, mejorando la experiencia de usuario. Es muy útil en sitios de una sola página (`one page`) donde se navega entre secciones.
    

### Ejemplo:

html {
  scroll-behavior: smooth;
}

Esto hace que todos los desplazamientos internos de página sean fluidos, sin necesidad de JavaScript.


## 3. `scroll-snap-type`

Esta propiedad permite **crear un sistema de "anclaje"** en el scroll. Cuando el usuario se desplaza, el contenido **se "encaja" automáticamente en puntos específicos** que definas, generando una experiencia parecida a pasar páginas o diapositivas.

### Sintaxis:

scroll-snap-type: eje modo;


- **Ejes**:
    
    - `x`: scroll horizontal.
        
    - `y`: scroll vertical.
        
    - `block`, `inline`: según dirección del texto.
        
    - `both`: aplica a ambos ejes.
        
- **Modos**:
    
    - `mandatory`: el scroll **se detiene obligatoriamente** en los puntos definidos.
        
    - `proximity`: el scroll se detiene **solo si se acerca a un punto de anclaje**.
        

### Ejemplo:

.scroll-container {
  scroll-snap-type: x mandatory;
}


Esto crea un scroll horizontal que se detiene en cada ítem alineado.

## 4. `scroll-snap-align`

Esta propiedad se aplica a los **elementos hijos** dentro de un contenedor con `scroll-snap-type`. Define **dónde se va a alinear cada elemento** cuando el scroll se detenga.

### Valores:

- `start`: alinea el inicio del hijo con el inicio del contenedor.
    
- `center`: lo centra en el contenedor.
    
- `end`: lo alinea al final.
    
- `none`: sin alineación especial.


### Ejemplo:

.card {
  scroll-snap-align: center;
}


Es muy útil para sliders, carruseles o secciones con scroll fijo.


## 5. `scroll-snap-stop`

Controla si el scroll puede **"saltarse" puntos de anclaje** si el desplazamiento es rápido.

### Valores:

- `normal`: el scroll puede continuar de largo si el usuario se desplaza rápido.
    
- `always`: el scroll se **detiene siempre** en cada punto, incluso con scroll fuerte o rápido.
    

Esto mejora la precisión en interfaces donde es importante no saltear secciones.


## 6. `scroll-padding`

Define un **espacio interno adicional** dentro del área visible de scroll, que se tiene en cuenta **al usar scroll-snap o navegación con anclas**. No cambia la posición del contenido, pero afecta **dónde se detiene el scroll**.

### Ejemplo:


.scroll-container {
  scroll-padding-top: 80px;
}

Útil para compensar headers fijos (`position: sticky`) que podrían tapar el contenido cuando el scroll se detiene.



## 7. `scroll-margin`

Parecida a `scroll-padding`, pero aplicada a los elementos **que se anclan**. Define un **margen visual desde los bordes del scroll** hasta el punto donde se quiere anclar el elemento.

### Ejemplo:

.section {
  scroll-margin-top: 50px;
}

Esto hace que la sección se detenga 50px **antes** del borde superior, evitando que un header fijo la tape.


## 8. `scrollbar-width` (solo Firefox)

Permite controlar el **grosor** de la barra de desplazamiento.

### Valores:

- `auto`: grosor por defecto.
    
- `thin`: barra más delgada.
    
- `none`: **oculta** la barra, pero el scroll sigue funcionando.
    

---

## 9. `scrollbar-color` (solo Firefox)

Permite definir los colores del scrollbar: **la manija (thumb)** y **el fondo (track)**.

scrollbar-color: #666 #ddd;

- Primer color: manija.
    
- Segundo color: fondo de la pista.


## 10. `overscroll-behavior`

Controla **qué pasa cuando el usuario llega al final del scroll**. Sirve para evitar que el scroll se propague a elementos padre o para evitar el "rebote" en móviles.

### Valores:

- `auto`: comportamiento por defecto.
    
- `contain`: el scroll **se contiene** dentro del elemento actual. No se propaga.
    
- `none`: **bloquea totalmente** efectos de rebote o scroll anidado.
    

Se puede aplicar por eje:

overscroll-behavior-y: contain;

## 11. Propiedades modernas para Scroll Animado (Scroll-Linked Animations)

Estas propiedades permiten **vincular animaciones CSS al scroll**, para que una animación se active o progrese según el desplazamiento del usuario.

### Propiedades clave:

- `scroll-timeline-name`
    
- `scroll-timeline-axis`
    
- `animation-timeline`
    
- `animation-range`
    
- `view-timeline`
