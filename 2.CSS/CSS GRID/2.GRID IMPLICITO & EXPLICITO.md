En CSS Grid, cuando trabajamos con `display: grid`, los elementos hijos (las "cajas" del grid) se colocan en un sistema de filas y columnas. Este sistema puede ser **explícito** o **implícito**, y la diferencia radica en **quién decide cuántas filas/columnas hay**: nosotros o el navegador.


## Grid Explícito
El **grid explícito** es la parte del grid que se define directamente con CSS, mediante propiedades como:
`grid-template-columns`
    
 `grid-template-rows`
Con estas propiedades, **estamos decidiendo cuántas columnas o filas hay, y de qué tamaño son**.
Ejemplo

.container {
  display: grid;
  grid-template-columns: 200px 1fr 1fr;
  grid-template-rows: 100px 100px;
}

En este caso estamos creando explícitamente:

- **3 columnas**: la primera de 200px y dos iguales que comparten el espacio restante (`1fr`).
    
- **2 filas** de 100px.

Esto forma un grid de **3 columnas por 2 filas (3×2)**, y el navegador intentará colocar los elementos dentro de esas celdas **definidas por nosotros**

Cuando nos encontramos en una situación donde tenemos mas elementos que celdas definidas, esto nos lleva a :

## GRID IMPLICITO

El **grid implícito** es **la parte del grid que el navegador genera automáticamente** cuando **no hay suficientes filas o columnas explícitas** para colocar todos los elementos.
Esto ocurre cuando:

- Hay **más elementos** de los que caben en el grid explícito.
    
- Un elemento se coloca en una posición que está **fuera del grid definido**.

### Cómo se controla:

Cuando el navegador necesita agregar filas o columnas adicionales, lo hace con **tamaño por defecto**, **a menos que nosotros lo definamos** usando:

- `grid-auto-rows` → define **la altura de las filas implícitas**, es decir, de **las filas que el navegador crea automáticamente** cuando no alcanzan las filas que definimos explícitamente con `grid-template-rows`.


- `grid-auto-columns` → define **el ancho de las columnas implícitas**, o sea, de **las columnas que el navegador genera automáticamente** cuando no alcanzan las columnas que  definimos con `grid-template-columns`..

Ejemplo

.container {
  display: grid;
  grid-template-columns: 1fr 1fr;
  grid-auto-rows: 50px;
}

`<div class="container">`
  `<div>1</div>`
  `<div>2</div>`
  `<div>3</div> <!-- ¡No hay espacio! -->`
`</div>`

Solo definimos **2 columnas**, pero hay **3 elementos**.  
Los primeros dos van en la **primera fila** del grid explícito.  
El tercero no tiene espacio… así que el navegador crea una **segunda fila automáticamente** (grid implícito), y le da una altura de `50px` gracias a `grid-auto-rows`.  Y si se llegara a formar otras columnas usaremos `grid-auto-columns` .

Otra propiedad para controlar grids implícitos seria usando:

### Grid-auto-flow

Define **la dirección en la que el navegador coloca automáticamente los ítems dentro del grid**, **cuando nosotros no les asignamos una posición manualmente**


###  `row` (valor por defecto)

- Los elementos se colocan **fila por fila**, de izquierda a derecha.
- Si no hay suficiente espacio en una fila, se crea una **nueva fila implícita**.

###  `column`

- Los elementos se colocan **columna por columna**, de arriba hacia abajo.
- Si no hay suficiente espacio en una columna, se crea una **nueva columna implícita**.


Ejemplo:

.container {
  display: grid;
  grid-template-columns: 100px 100px;
  grid-auto-rows: 50px;
  grid-auto-flow: row;
}
Este código:
- Crea 2 columnas.
- Agrega filas automáticamente si hay más elementos.
- Los coloca **de izquierda a derecha**, **fila por fila**.

De estas formas definimos como se van a ir formando los nuevos `grid-items`  si en filas o en columnas.
Pero cuando nos encontramos ante la situación en donde un elemento ocupa una area grande y deja huecos. Usamos el valor:

### DENSE

`dense` intenta **rellenar los huecos que quedan** en el grid con elementos más pequeños.
Entonces podemos decir que:

Sin `dense`: los elementos se colocan uno tras otro, dejando huecos si no caben.

Con `dense`: el navegador **reorganiza automáticamente** los elementos **más pequeños para llenar esos huecos**.

Este valor puede venir acompañado de 

#### `row dense`

 Coloca elementos **fila por fila**. **Rellena huecos automáticamente** moviendo hacia atrás los elementos más pequeños si caben. Puede alterar el orden visual.

### `column dense`

 Coloca elementos **columna por columna**. También **rellena huecos** como `row dense`, pero en dirección vertical. Útil para layouts verticales compactos.
