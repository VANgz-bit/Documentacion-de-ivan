
## ¿Qué son `grid-column` y `grid-row`?

Estas propiedades nos permiten **posicionar elementos individualmente** dentro de un `display: grid`. Te dicen **desde qué línea hasta qué línea debe abarcar** un elemento dentro del grid.

Como aclaramos anteriormente las partes de un grid dijimos que una parte tenia líneas invisibles que separaban a las celdas, de manera as clara podemos decir que:

Un grid está hecho de:

- **Líneas de columnas** verticales → delimitan las columnas.
    
- **Líneas de filas** horizontales → delimitan las filas.
    

**Cada línea está numerada, comenzando desde 1.**

## ¿Cómo nos posicionamos?


Hay tres formas para poder desplazarnos.

grid-column: start / end;
grid-row: start / end;

ejemplo:

.item {
  grid-column: 1 / 3;
  grid-row: 2 / 4;
}

**`grid-column: 1 / 3;`** → El elemento va **desde la línea 1 hasta la 3 de columna** (ocupa columnas 1 y 2).

**`grid-row: 2 / 4;`** → El elemento va **desde la línea 2 hasta la 4 de fila** (ocupa filas 2 y 3).

Otra forma seria :

grid-column-start: 1;
grid-column-end: 3;

Y para las filas:

grid-row-start: 2;
grid-row-end: 4;

## Usando `span`

Si no sabemos en qué línea termina, podemos usar `span` para decir cuántas columnas o filas debe ocupar:

grid-column: span 2; /* Ocupa 2 columnas desde donde esté */
grid-row: span 3;    /* Ocupa 3 filas desde donde esté */


Esto es útil cuando colocamos los elementos **automáticamente**, pero queremos que algunos ocupen más espacio.


Cosas a tener en cuenta:

- Puedes usar `-1` como valor final para ir hasta el **final del grid** (última línea).
- `span` funciona solo con valores positivos.
- Estas propiedades son **muy útiles para layouts personalizados**, como dashboards, tarjetas, interfaces de apps, etc.
