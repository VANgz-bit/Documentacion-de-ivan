
## ¿Qué es un Grid Dinámico?

Un **grid dinámico** es aquel que **se adapta automáticamente** al contenido y al contenedor. Es decir, **no tiene un número fijo de columnas o tamaños absolutos**, sino que:

- **Genera automáticamente** las columnas o filas necesarias según el espacio disponible.
- **Se reacomoda** cuando cambia el tamaño de la ventana o del contenedor (por ejemplo, cuando el usuario cambia el tamaño del navegador).

Esto se logra con propiedades como `auto-fit`, `auto-fill`, `minmax()`, etc., que veremos a continuación.

## Cómo se construye un Grid Dinámico y Responsive

###  Paso 1: Definir el contenedor como grid

.container {
  display: grid;
}

### Paso 2: Usar `grid-template-columns` dinámico

.container {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 1rem;
}

- **`repeat()`**: repite la definición de columnas.
- **`auto-fit`**: intenta encajar tantas columnas como quepan, **ocultando las que no caben**.
- **`auto-fill`**: similar, pero **rellena con columnas vacías si hay espacio**.
- **`minmax(250px, 1fr)`**: cada columna tendrá al menos 250px y como máximo ocupará una fracción (`1fr`) del espacio disponible.

 Esto permite que el número de columnas se ajuste automáticamente según el ancho del contenedor.

### Paso 3: Añadir responsividad adicional (opcional)

Aunque el `minmax()` ya da adaptabilidad, se puede complementar con **media queries** para cambiar estilos en puntos de quiebre clave.

@media (max-width: 600px) {
  .container {
    grid-template-columns: 1fr;
  }
}

## ¿Qué hacen `auto-fill` y `auto-fit`?

Ambos se usan dentro de `repeat()` en CSS Grid, como esto:

grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));

Tanto `auto-fill` como `auto-fit`:
- Intentan **meter tantas columnas como quepan** en el contenedor.
- Usan `minmax()` para decir cuánto puede crecer o reducirse una columna.


PERO su **comportamiento con el espacio sobrante es diferente**.

## Diferencias entre Auto-fill y Auto-fit  

### `auto-fill`: **"Reserva el espacio aunque no haya contenido"**

- Sigue generando columnas incluso si no hay suficientes elementos para llenarlas.
- **No colapsa columnas vacías.**
- Ideal si quieres que el diseño se mantenga con “huecos” o distribución constante.

###  `auto-fit`: **"Colapsa columnas vacías y estira las demás"**

- Solo crea columnas **para los elementos que existen**.
- Si sobran columnas vacías, **las colapsa** (desaparecen).
- Las columnas existentes se expanden para ocupar todo el espacio disponible.

## En resumen

| Propiedad   | Crea columnas vacías | Colapsa columnas vacías | ¿Estira las columnas? | Ideal para...                                  |
| ----------- | -------------------- | ----------------------- | --------------------- | ---------------------------------------------- |
| `auto-fill` | Sí                   | No                      | No                    | Distribuciones constantes, espacios reservados |
| `auto-fit`  | No                   | Sí                      | Sí                    | Diseños fluidos, adaptativos al contenido real |


## ¿Qué es un Grid Responsive?

Un **grid responsive** (adaptativo) es un grid que **cambia su estructura y distribución según el tamaño de pantalla** o el tipo de dispositivo. Es decir:

- En pantallas grandes puede mostrar muchas columnas.
- En pantallas medianas o pequeñas puede reducirse a menos columnas o incluso una sola columna.
- Puede usar **media queries** o bien técnicas de **grid dinámico** para lograrlo sin escribir reglas específicas para cada tamaño.

## ¿Cuándo usar Grid dinámico y responsive?

- Galerías de imágenes.
    
- Listas de productos (como en e-commerce).
    
- Dashboards.
    
- Secciones de contenido en tarjetas.
    
- Páginas que deben mantener orden visual sin escribir media queries constantes.
