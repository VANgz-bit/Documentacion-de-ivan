
## ¿Qué es `animation-range`?

`animation-range` define **cuándo debe comenzar y terminar una animación** dentro de una línea de tiempo basada en scroll (`scroll-timeline`) o visibilidad (`view-timeline`).

A diferencia de las animaciones clásicas que usan tiempo (`animation-duration: 2s`), acá el avance de la animación **se sincroniza con el desplazamiento del usuario o con la visibilidad del elemento en el viewport**.

---

## 🔧 ¿Cómo se usa?

Se usa para **controlar con precisión** el rango dentro del cual se ejecutará la animación, en función del progreso de la timeline (scroll o view):

Ejemplo:

`animation-range: <inicio> <final>;`

- **Inicio:** punto donde comienza la animación.
    
- **Final:** punto donde termina la animación.

Ejemplo:

`animation-range: entry 0% contain 100%;`

Significa: la animación arranca cuando el elemento empieza a entrar al viewport y termina cuando está completamente dentro.

---

##  Palabras clave (eventos visuales)

Estas palabras representan **momentos clave en la visibilidad del elemento**:

|Palabra|Qué indica|
|---|---|
|`entry`|Cuando el borde del elemento entra al viewport.|
|`exit`|Cuando empieza a salir.|
|`cover`|Cuando comienza a cubrir por completo el viewport.|
|`contain`|Cuando el elemento ya está completamente dentro del viewport.|

Estas palabras se combinan con porcentajes (`0%`, `50%`, `100%`, etc.) para especificar con más detalle cuándo ocurre el evento.

### Ejemplos:

- `entry 0%`: justo cuando entra al viewport.
    
- `exit 100%`: justo antes de desaparecer completamente.
    
- `contain 50%`: a mitad de estar contenido dentro del viewport.
    

---

##  Alternativas separadas: `animation-range-start` y `animation-range-end`

Podemos definir el inicio y el final de forma separada:

`animation-range-start: entry 25%;
`animation-range-end: contain 100%;

Esto es exactamente igual a escribir:

`animation-range: entry 25% contain 100%;`

---

## Requisitos para que funcione

1. **Debe haber una timeline activa**:

`animation-timeline: scroll() o view() o personalizada;`


2. **Duración en automático**:

`animation-duration: auto;`


3. **Animación definida con `@keyframes`**:

`@keyframes ejemplo {from { ... } to { ... } }`

---

##  Ejemplo completo

@keyframes aparecer {
  from { opacity: 0; transform: translateY(30px); }
  to { opacity: 1; transform: translateY(0); }
}

.elemento {
  view-timeline-name: scrollVista;
  animation-name: aparecer;
  animation-timeline: scrollVista;
  animation-duration: auto;
  animation-range: entry 0% contain 100%;
}


---

##  ¿Para qué sirve?

- Para **activar animaciones sólo en un rango específico de visibilidad**.
    
- Ideal para efectos como: desvanecer, escalar, mover, rotar... **a medida que el usuario scrollea o visualiza**.
    
- Te da **control absoluto sobre cuándo empieza y termina la animación**, sin depender de JavaScript.
    

---

##  Consideraciones importantes

- Si el elemento no entra al rango que definiste, **la animación no se dispara**.
    
- Si querés que el estilo se mantenga después de la animación, usá:

	`animation-fill-mode: both;`

- Es necesario que el navegador **soporte scroll-linked animations** (soporte todavía en evolución).
