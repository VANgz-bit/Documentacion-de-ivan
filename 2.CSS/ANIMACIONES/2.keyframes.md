## ¿Qué es `@keyframes` en CSS?

La regla **`@keyframes`** se usa para definir una animación en CSS. Permite describir **cómo deben cambiar los estilos de un elemento a lo largo del tiempo**.

Funciona marcando "fotogramas clave" (de ahí el nombre _keyframes_), que son puntos en el tiempo donde el elemento debe tener un estilo específico. El navegador interpolará automáticamente los estilos intermedios entre esos puntos.

### Sintaxis básica de `@keyframes`

@keyframes nombre-de-la-animacion {
  from {
    /* estilos iniciales */
  }
  to {
    /* estilos finales */
  }
}

O usando porcentajes:

@keyframes ejemplo {
  0% {
    transform: translateX(0);
  }
  50% {
    transform: translateX(100px);
  }
  100% {
    transform: translateX(0);
  }
}


## Características y reglas de `@keyframes`

- Podemos usar `from` (equivalente a `0%`) y `to` (equivalente a `100%`).
    
- También podemos usar cualquier valor porcentual entre `0%` y `100%` (por ejemplo `20%`, `80%`), lo que permite hacer animaciones complejas y controladas en distintos momentos del tiempo.
    
- Dentro de cada porcentaje (o `from`/`to`) se declaran propiedades CSS comunes como `transform`, `opacity`, `background-color`, etc.
    
- La interpolación entre valores es automática y suave (según la función de tiempo usada).

## Propiedades que se pueden animar en `@keyframes`

No todas las propiedades CSS se pueden animar. Algunas de las más comunes que **sí se pueden animar** son:

- `transform` (translate, scale, rotate, skew…)
    
- `opacity`
    
- `color`
    
- `background-color`
    
- `width`, `height`
    
- `top`, `left`, `right`, `bottom`
    
- `margin`, `padding`
    
- `border-color`, `border-width`
    
- `box-shadow`
    
- `filter`
    
- `clip-path`



# Propiedades de animación en CSS (usadas con `@keyframes`)

### 1. **`animation-name`**

- Define el nombre de la animación (el que le diste en `@keyframes`).
- **Valores**:
    - Nombre del keyframe (ej: `mover`, `desvanecer`, etc.).
        
    - `none`: desactiva la animación.

Ejemplo:

animation-name: mover;


### 2. **`animation-duration`**

- Establece cuánto dura la animación.
    
- **Valores**:
    
    - Un tiempo válido: `s` (segundos) o `ms` (milisegundos).  
        Ej: `2s`, `500ms`.
        

Ejemplo:

animation-animation-duration: 3s;


### 3. **`animation-timing-function`**

- Define la velocidad del cambio durante la animación (la curva de aceleración).
    
- **Valores comunes**:
    
    - `linear`: velocidad constante.
        
    - `ease`: comienza lento, se acelera y vuelve a frenar (valor por defecto).
        
    - `ease-in`: empieza lento, termina rápido.
        
    - `ease-out`: empieza rápido, termina lento.
        
    - `ease-in-out`: lento al inicio y al final.
        
    - `steps(n)`: la animación avanza en pasos (útil para efectos tipo sprite o tecleo).
        
    - `cubic-bezier(x1, y1, x2, y2)`: curva personalizada.

Ejemplo:

animation-timing-function: ease-in-out;


### 4. **`animation-delay`**

-  Espera una cierta cantidad de tiempo antes de comenzar la animación.
    
- **Valores**:
    - Tiempo en `s` o `ms`.
        
    - También se puede usar un valor negativo para comenzar a mitad de la animación.

Ejemplo:

animation-delay: 1s;


### 5. **`animation-iteration-count`**

-  Define cuántas veces se repite la animación.
    
- **Valores**:
    
    - Un número entero (`1`, `2`, `3`, etc.).
        
    - `infinite`: se repite para siempre.

Ejemplo:

animation-iteration-count: infinite;


### 6. **`animation-direction`**

- Define si la animación se reproduce en una sola dirección o si se alterna.
    
- **Valores**:
    
    - `normal`: se reproduce de principio a fin (por defecto).
        
    - `reverse`: de fin a principio.
        
    - `alternate`: normal → reverse → normal... alternando.
        
    - `alternate-reverse`: reverse → normal → reverse...
        

Ejemplo:

animation-direction: alternate;


### 7. **`animation-fill-mode`**

- Define cómo se aplican los estilos **antes y después** de que se ejecute la animación.
    
- **Valores**:
    
    - `none` (por defecto): el elemento vuelve a su estilo original.
        
    - `forwards`: mantiene el estilo final (`100%` o `to`).
        
    - `backwards`: aplica el estilo del inicio durante el delay.
        
    - `both`: combina `forwards` y `backwards`.

Ejemplo:

animation-fill-mode: forwards;


### 8. **`animation-play-state`**

-  Permite pausar o continuar la animación.
    
- **Valores**:
    
    - `running`: animación en ejecución.
        
    - `paused`: animación detenida (se puede reanudar con JavaScript o clases CSS).


Ejemplo:

animation-play-state: paused;


### 9. **`animation` (shorthand)**

- Es una forma de declarar **todas las propiedades anteriores en una sola línea** (en orden):

Ejemplo

animation: mover 2s ease-in-out 0s infinite alternate forwards;

**Orden recomendado del shorthand**:

animation: 
  animation-name 
  animation-duration 
  animation-timing-function 
  animation-delay 
  animation-iteration-count 
  animation-direction 
  animation-fill-mode 
  animation-play-state;
