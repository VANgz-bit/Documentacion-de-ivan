
## GIT- ALIAS

### ¿Qué es un alias en Git?

Un **alias en Git** es una forma de **crear atajos personalizados para comandos largos o complejos**.  
Te permite escribir comandos más cortos, fáciles de recordar o incluso combinaciones que normalmente te llevarían más tiempo.

Es útil para:

- Acortar comandos que se usan mucho (`git status` → `git s`).
    
- Agrupar comandos complejos en una palabra.
    
- Hacer que Git sea más cómodo y productivo.
    

---

## ¿Cómo se crea un alias?

Se usa el siguiente formato:

```bash
git config --global alias.nombre-alias 'comando-original'
```

- `--global`: guarda el alias para **todos los repositorios** del usuario.
    
- `alias.nombre`: define el **nombre del alias** que vas a escribir luego de `git`.
    
- `'comando-original'`: es el **comando real** que se ejecutará cuando uses el alias.
    

---

## Ejemplo básico

```bash
git config --global alias.s 'status'
```

Después de eso, se puede usar simplemente:

```bash
git s
```

Y será lo mismo que escribir `git status`.

---

## ¿Dónde se guardan los alias?

Se guardan en el archivo de configuración global:

```bash
~/.gitconfig
```

Podés abrirlo con:

```bash
code ~/.gitconfig
```

Y verás algo como:

```ini
[alias]
  s = status
  co = checkout
```

## Alias avanzado: ver hace cuánto se hizo un commit

### Comando:

```bash
git config --global alias.age "!f() { git log -1 --format='%cr' \"$@\"; }; f"
```

### ¿Qué hace este alias?

Crea un alias llamado `age` que, cuando lo uses así:

```bash
git age <hash>
```

Te dice **hace cuánto tiempo se hizo ese commit** (por ejemplo: _"3 weeks ago"_ o _"2 hours ago"_).

---

### Desglose del comando:

```bash
!f() { git log -1 --format='%cr' "$@"; }; f
```

Explicación paso a paso:

- `!`: le dice a Git que esto es un **comando de shell**, no un comando Git estándar.
    
- `f() { ... }; f`: define y ejecuta una **función en bash** llamada `f`, que permite recibir argumentos.
    
- `git log -1`: muestra solo el **último commit** relacionado con el hash que se pasa.
    
- `--format='%cr'`: muestra la fecha de ese commit en **formato relativo** (ej: “4 days ago”).
    
- `"$@"`: permite **pasar parámetros** (como el hash del commit) al alias.
    

---

### Ejemplo de uso real:

```bash
git age e3f9c81
```

Git mostrará algo como:

```text
2 weeks ago
```

---

## ¿Cómo ver todos los alias creados?

```bash
git config --get-regexp alias
```

Esto muestra todos los alias que tenés configurados con sus respectivos comandos reales.

---

## ¿Cómo borrar un alias?

```bash
git config --global --unset alias.nombre
```

Ejemplo:

```bash
git config --global --unset alias.age
```

---

## Resumen práctico

|Acción|Comando|
|---|---|
|Crear alias básico|`git config --global alias.s "status"`|
|Crear alias complejo (bash)|`git config --global alias.age "!f() { ... }; f"`|
|Ver todos los alias|`git config --get-regexp alias`|
|Borrar alias|`git config --global --unset alias.nombre`|
|Editar manualmente|`code ~/.gitconfig`|
