## 📂 `.gitignore` – ¿Qué es?

Es un **archivo especial de texto** que le dice a Git **qué archivos o carpetas debe ignorar**, o sea, **no rastrear ni subir** al repositorio.


### 🛡️ ¿Por qué es importante?

Sirve para:

* Archivos de configuración local (como `.env` con contraseñas).

* Carpetas temporales (`node_modules/`, `dist/`, etc.).

* Archivos del sistema (`.DS_Store`, `Thumbs.db`, etc.).
* 
---
## ⚙️ ¿Cómo se usa?


1. Creás un archivo llamado `.gitignore` (sin extensión).

2. Dentro escribís lo que querés que Git ignore, por ejemplo:

```

# Ignorar todo lo que termine en .log

*.log

# Ignorar la carpeta de fotos

fotos/

# Ignorar todo menos un archivo

*.txt

!readme.txt  

# Ignorar todo menos una imagen

fotos/

!fotos/background.png

```

Git **ignora archivos solo si todavía no los está rastreando**.

 Si ya están **trackeados**, hay que eliminarlos del seguimiento (más abajo explico cómo).

---
##  ¿Cómo ignoro todos los archivos pero quiero dejar algunos?

### Ejemplo:

```gitignore
# Ignora todo

* 

# ¡Pero no ignores el README!

!readme.md
```

  
---

##  ¿Y si Git ya rastrea el archivo?


Si ya hiciste `git add` o ya está en el historial, **Git no lo ignora aunque esté en `.gitignore`**.

### ¿Qué hacer?

1. Quitarlo del seguimiento:


```bash

git rm --cached nombre-del-archivo

```

2. Luego, agregarlo al `.gitignore`.

---

##  ¿Qué hace este comando?

  
```bash

git ls-tree -r --name-only <hash>

```


 Este comando **muestra los archivos que había dentro de un commit específico**.

Ideal para revisar el contenido de un commit sin hacer checkout.


---

## 🌍 ¿Se puede configurar un `.gitignore` global?


¡Sí! Se usa para ignorar archivos que **no querés que Git rastree en ningún proyecto**, como:


* `.DS_Store`

* Archivos de sistema

* Temporales que crea tu editor

### ¿Cómo se hace?

1. Crear el archivo:

```bash

touch ~/.gitignore_global

```

2. Decirle a Git que lo use:

```bash

git config --global core.excludesfile ~/.gitignore_global

```

---

##  Prioridad: `.gitignore` local vs global

| Nivel                       | Prioridad  |

| --------------------------- | ---------- |

| `.gitignore` en el proyecto | ✅ MÁS ALTA |

| `.gitignore_global`         | 🔽 Menor   |

> El local **siempre tiene prioridad**, ya que afecta directamente al proyecto.

---

###  Si Git ya rastrea un archivo, no lo ignorará. Para ignorarlo:

  

```bash

git rm --cached archivo.txt

```

---

##  Ver archivos dentro de un commit:


```bash

git ls-tree -r --name-only <hash>

```


---

##  Ignorar archivos a nivel global:


```bash

touch ~/.gitignore_global

git config --global core.excludesfile ~/.gitignore_global

```

  

* El archivo global ignora cosas en todos tus proyectos.

* El `.gitignore` local del proyecto **tiene más prioridad** que el global.
