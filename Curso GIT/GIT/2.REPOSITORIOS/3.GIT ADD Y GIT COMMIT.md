
##  Primeros pasos en Git: `git add` y `git commit` (con comandos detallados)

El flujo básico de trabajo en Git comienza luego de inicializar un repositorio. A partir de ese punto, se pueden registrar versiones de los archivos utilizando dos comandos fundamentales: **`git add` y `git commit`**.

Estos comandos permiten preparar y guardar los cambios que se realicen en un proyecto, manteniendo un historial organizado y controlado.

---

##  `git add`: Preparar archivos para el commit

### ¿Qué hace?

Coloca archivos desde el área de trabajo (working directory) en el área de preparación (staging area). Esto **no los guarda aún en el historial**, pero los marca para que sean incluidos en el próximo commit.

###  Comandos relacionados con `git add`:

|Comando|Explicación detallada|
|---|---|
|`git add <archivo>`|Agrega un archivo específico a la zona de preparación.|
|`git add archivo1 archivo2`|Agrega varios archivos específicos al mismo tiempo.|
|`git add .`|Agrega **todos los archivos modificados o nuevos**, excepto los que estén ignorados.|
|`git add -A`|Agrega **todos los archivos modificados, nuevos y eliminados**.|
|`git add -u`|Agrega **solo archivos modificados y eliminados**, no los nuevos.|
|`git add -p`|Agrega porciones de archivos interactivamente (útil para confirmar cambios por partes).|

###  Detalles importantes:

- Si se modifica un archivo luego de usar `git add`, será necesario volver a ejecutar `git add` si se quiere incluir el nuevo cambio.
    
- No se genera historial con `git add`, solo se **prepara** para que el commit lo registre.
    

---

##  `git commit`: Guardar los cambios en el historial

### ¿Qué hace?

Registra de forma permanente los archivos preparados en el **repositorio local**. Cada commit es una fotografía exacta del proyecto en un momento determinado.

###  Comandos relacionados con `git commit`:

|Comando|Explicación detallada|
|---|---|
|`git commit -m "mensaje"`|Crea un commit con los archivos preparados y le asigna un mensaje corto y descriptivo.|
|`git commit`|Abre el editor por defecto (como Vim o VS Code) para escribir un mensaje más largo.|
|`git commit -a -m "mensaje"`|Agrega automáticamente **archivos modificados** (no nuevos) y hace el commit.|
|`git commit --amend`|Permite modificar el último commit (mensaje o archivos) sin crear uno nuevo.|
|`git commit --allow-empty -m "mensaje"`|Crea un commit vacío (sin cambios) con un mensaje personalizado.|

###  Detalles importantes:

- Cada commit incluye: los archivos preparados, el autor, la fecha y un mensaje.
    
- Los commits son **identificados con un hash único** (como `f3a9c42...`).
    
- Los archivos no preparados (sin `git add`) **no se incluyen**.
    

---

##  Flujo completo en Git (Visualización del proceso)

```plaintext
1. Modificación de archivos (Working Directory)
      ↓
2. git add → Área de preparación (Staging Area)
      ↓
3. git commit → Repositorio local (historial de Git)
```

---

##  Buenas prácticas

- Usar `git add -p` para confirmar solo partes del archivo, ideal para hacer commits más limpios.
    
- Escribir mensajes de commit **claros y representativos** de los cambios hechos.
    
- Confirmar frecuentemente para evitar perder cambios o hacer commits muy grandes y desorganizados.
    

---

##  Conclusión

`git add` y `git commit` son los primeros comandos fundamentales en el uso diario de Git.

- `git add` permite **controlar exactamente qué cambios se van a registrar**.
    
- `git commit` los guarda **de manera permanente en el historial del proyecto**.
    

Estos comandos forman la base para trabajar de forma segura, organizada y profesional, tanto en proyectos individuales como en equipos colaborativos.
