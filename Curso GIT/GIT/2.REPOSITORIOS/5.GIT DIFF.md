Git diff es una herramienta poderosa que permite visualizar las diferencias entre versiones de archivos o commits. Esto es posible porque Git organiza el contenido del proyecto en tres zonas distintas:

- Working Directory → donde trabajás (archivos editados)
    
-  Staging Area → donde van los archivos luego de hacer git add
    
-  HEAD → representa el último commit realizado
    

Por eso existen diferentes formas de usar git diff según qué zonas querés comparar. Aunque todos los comandos se llaman igual, su función cambia dependiendo de los argumentos que reciben.

ℹ️ Nota: Git diff actúa como una herramienta flexible que compara distintas áreas del repositorio. El comando es el mismo, pero el resultado varía según lo que se compare (Working Directory, Staging Area, Commits).

### Comandos y funciones principales:

1. git diff  
    🔸 Compara: Working Directory vs Staging Area  
    🔹 Muestra lo que cambiaste pero aún no hiciste git add.
    
2. git diff --staged (o --cached)  
    🔸 Compara: Staging Area vs HEAD (último commit)  
    🔹 Muestra lo que ya agregaste con git add y está listo para commitear.
    
3. git diff HEAD  
    🔸 Compara: Working Directory completo vs HEAD  
    🔹 Muestra todos los cambios actuales (staged + no staged) respecto al último commit.
    
4. git diff `<commit>`  
    🔸 Compara: Working Directory vs un commit específico  
    🔹 Permite ver qué cambió desde ese commit hasta ahora.
    
5. git diff `<commit1> <commit2>`  
    🔸 Compara: dos commits específicos  
    🔹 Muestra los cambios que hubo entre uno y otro.
    
6. git diff --name-only  
    🔹 Muestra solo los nombres de los archivos modificados, sin contenido.
    
7. git diff --stat  
    🔹 Muestra un resumen: cuántas líneas se agregaron o eliminaron por archivo.
    
8. git diff --word-diff  
    🔹 Muestra los cambios palabra por palabra, útil para modificaciones dentro de una misma línea.
    
9. git diff --color-words  
    🔹 Muestra las diferencias en línea con resaltado de colores.
    

### Comandos adicionales útiles:

- git show  
    🔸 Muestra el contenido de un commit: autor, fecha, mensaje y cambios.
    
- git log / git log --oneline  
    🔸 Muestra el historial de commits (formato extendido o resumido).
    
- git config --global core.abbrev `<número>`  
    🔸 Define cuántos caracteres se muestran del hash de un commit (por ejemplo, 5).

## RESUMEN RÁPIDO DE COMANDOS:

|Acción|Comando|
|---|---|
|Ver cambios no agregados al staging|git diff|
|Ver cambios ya en staging|git diff --staged|
|Ver todos los cambios respecto al último commit|git diff HEAD|
|Ver archivos modificados (sin detalle)|git diff --name-only|
|Ver resumen de líneas cambiadas|git diff --stat|
|Ver cambios palabra por palabra|git diff --word-diff|
|Ver diferencias entre dos commits|git diff|
|Ver cambios desde un commit específico|git diff|
|Ver historial de commits|git log / git log --oneline|
|Mostrar detalles de un commit|git show|
|Acortar longitud visual del hash|git config --global core.abbrev <número>|
