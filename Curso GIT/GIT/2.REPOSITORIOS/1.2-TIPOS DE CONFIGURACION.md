 
 ##  ¿Qué es `git config`?

Cuando usamos Git por primera vez, tenemos que decirle quien somos. Esto es importante porque cada **commit** (registro de un cambio) lleva tu nombre y tu correo electrónico.

Para eso usamos:

git config [opción] user.name "Tu Nombre"
git config [opción] user.email "tucorreo@ejemplo.com"

## Tipos de configuración: `--local`, `--global`, `--system`

Git permite configurar cosas en **tres niveles**, que funcionan como una especie de jerarquía:

| Nivel  | Parámetro Git | Archivo donde se guarda             | A quién afecta                      | Prioridad |
| ------ | ------------- | ----------------------------------- | ----------------------------------- | --------- |
| Local  | `--local`     | `.git/config` (dentro del proyecto) | Solo a ese proyecto Git             | Alta      |
| Global | `--global`    | `~/.gitconfig`                      | A todos los proyectos de tu usuario | Media     |
| System | `--system`    | `/etc/gitconfig`                    | A todos los usuarios del sistema    |  Baja     |

 Si configuramos algo en **local**, sobrescribe lo que esté en global o system.

---

##  Ejemplos prácticos

### 1. Configuración global (la más común)

git config --global user.name "ivan Dev"
git config --global user.email "ivan@example.com"


- Se guarda en: `~/.gitconfig`
    
- Se usa automáticamente en todos los repositorios **nuevos o existentes** que no tengan configurado el `--local`.

## 2. Configuración local (por proyecto)

cd mi-proyecto/
git config --local user.name "ivan Proyectos"
git config --local user.email "otro@email.com"

- Se guarda en: `./.git/config`
    
- Solo se aplica dentro de ese proyecto.
    
- Ideal si trabajás con distintos nombres (por ejemplo, uno personal y otro laboral).


## 3. Configuración system (todo el sistema)

sudo git config --system user.name "Administrador Git"
sudo git config --system user.email "admin@sistema.com"


- Se guarda en: `/etc/gitconfig`
    
- Afecta a **todos los usuarios del sistema operativo**.
    
- Requiere permisos de administrador (`sudo`).
    
- No se suele modificar a menos que trabajes en una PC compartida.

##  Cómo ver tu configuración actual

Podés ver lo que tenemos configurado con:

git config --list


O por nivel:

git config --local --list
git config --global --list
git config --system --list


Y si querés ver una config específica:


git config user.name
git config --global user.email

##  Resumen visual

┌────────────────────┐
│ Configuración en Git │
└────────────────────┘

  user.name  → Tu nombre
  user.email → Tu correo

  Niveles:
    --local  → Solo este proyecto (más prioridad)
    --global → Todos tus proyectos
    --system → Todo el sistema (menos prioridad)

  Orden de prioridad: local > global > system
