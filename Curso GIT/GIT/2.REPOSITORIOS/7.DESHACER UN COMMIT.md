
# 🔁 REHACER UN COMMIT RECIENTE CON GIT RESET

Git reset permite retroceder en el historial de commits. Al hacerlo, podés borrar uno o más commits, modificar lo que quieras, y luego volver a hacer nuevos commits correctamente.

La idea es “borrar el commit” sin perder el trabajo (salvo que vos lo decidas).

---

## ¿Cómo funciona git reset?

Git reset mueve el puntero de HEAD (el historial actual) hacia atrás, como si los commits que eliminás nunca hubieran existido. Tenés tres formas de hacerlo:

---

### 1. git reset --soft HEAD~1

- Elimina el último commit
    
- Pero conserva los archivos en el staging area (como si ya hubieras hecho git add)
    
- Ideal si querés reescribir el mensaje del commit o agregar algo más sin perder lo que ya tenías listo
    

```bash
git reset --soft HEAD~1
# ahora podés rehacer el commit
git commit -m "Nuevo mensaje o contenido"
```

---

### 2. git reset --mixed HEAD~1

- Elimina el último commit
    
- Saca los archivos del staging area, pero deja las ediciones en tu carpeta
    
- Es el modo por defecto (si no ponés nada, Git usa --mixed)
    

```bash
git reset HEAD~1
# tus archivos editados siguen ahí, pero no están en staging
git add .
git commit -m "Nuevo commit corregido"
```

---

### 3. git reset --hard HEAD~1

- Elimina el último commit
    
- Borra los cambios en los archivos también
    
- ⚠️ Es destructivo: se pierden todos los cambios que no estén guardados en otro lado
    

```bash
git reset --hard HEAD~1
# ⚠️ todo vuelve al estado anterior al último commit
```

Solo usalo si estás seguro de que querés eliminar todo, sin dejar rastros.

---

## ¿Qué significa HEAD~1?

- HEAD representa el commit actual (el último).
    
- HEAD~1 es el commit anterior.
    
- HEAD~2 es dos commits atrás, etc.
    

Por ejemplo, si querés rehacer los últimos dos commits:

```bash
git reset --soft HEAD~2
```

---

## Ejemplo completo: borrar y rehacer un commit

1. Te equivocaste en el último commit
    
2. Querés rehacerlo correctamente
    

Pasos:

```bash
git reset --mixed HEAD~1        # borra el commit, deja los cambios
# hacés las correcciones que quieras
git add archivo-corregido.html  # o git add .
git commit -m "Commit bien hecho ahora"
```

---

## ¿Cuándo usar git reset para rehacer commits?

✅ Trabajás solo  
✅ Aún no subiste los commits con git push  
✅ Querés rehacer el trabajo de forma limpia, sin dejar rastros en el historial

🚫 No recomendado si ya subiste los commits a un repositorio remoto compartido. En ese caso, es mejor usar git revert.
