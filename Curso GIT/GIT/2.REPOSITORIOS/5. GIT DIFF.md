##  ¿Qué es `git diff`?

 

`git diff` te muestra **los cambios entre versiones** del código:

- lo que cambió desde el último commit,

- lo que cambió entre dos commits,

- lo que hay listo para hacer commit (staged) y lo que no.


##  1. `git show <file>` (o `git show <commit>`)

### ¿Qué hace?

Muestra **información de un commit** específico, incluyendo:

- el autor,

- la fecha,

- el mensaje del commit,

- y los **cambios realizados en los archivos**.

### Si ponemos:

```bash

git show HEAD

```

→ Te muestra el último commit.

### Si ponemos:


```bash

git show 8ccd4

```

→ Te muestra ese commit (con su hash).  

(Si escribís parte del hash, Git lo entiende si es único).


 No es lo mismo que `git diff`, porque esto **muestra un commit completo** con los cambios que se hicieron en él.


##  2. `git diff`

### ¿Qué hace?


Compara **el estado actual del código** con:

- los archivos **staged** (ya listos para commit),

- o con un commit anterior.

### Ejemplos útiles:

####  Cambios NO agregados (no staged):


```bash

git diff

```


→ Te muestra las diferencias **entre lo que editaste y el último commit**, pero solo lo que **aún no agregaste con `git add`**.

####  Cambios staged:

  
```bash

git diff --staged

```

  
También lo podés abreviar como:


```bash

git diff --cached

```

→ Te muestra **los cambios que ya agregaste con `git add`**, o sea, los que están listos para el próximo commit.

##  3. `git diff --word-diff 4437b 8ccd4`

### ¿Qué hace?

Compara **dos commits** (4437b y 8ccd4 en tu ejemplo), y muestra los cambios **palabra por palabra** en lugar de línea por línea.

### ¿Por qué usarlo?

Es útil cuando una línea tiene muchos cambios y querés ver exactamente **qué palabra o símbolo cambió**.

### Ejemplo:


```bash

git diff --word-diff abc123 def456

```

Verás algo así:

```

The quick [-brown-]{+red+} fox jumps

```

Eso significa que cambió la palabra "brown" por "red".

##  4. git log`

### ¿Qué hace?

Muestra el historial completo de commits, con:


- el autor,

- la fecha,

- el mensaje del commit,

- y el hash completo.

### Abreviación:


```bash

git log --oneline

```


→ Muestra el historial de commits en una línea por commit:  

(hash corto + mensaje).


Ejemplo:

```

8ccd4 Fix header style

4437b Add login button

```

  
Más legible y práctico cuando estás viendo muchos commits.


##  5. `git config --global core.abbrev 5`

### ¿Qué hace?


Cambia la **longitud de los hashes de commit** que Git muestra cuando están abreviados.

Por defecto, Git muestra los primeros 7 caracteres del hash:  

`8ccd4b9...`  

Pero con este comando:


```bash

git config --global core.abbrev 5

```



Git mostrará solo 5 caracteres:  

`8ccd4`

Esto **no cambia los hashes reales**, solo cómo se muestran por defecto (ej: en `git log --oneline` o `git show`).

##  Resumen con un ejemplo real:


Imaginá que hiciste cambios en `index.html`.


1. Revisás los cambios sin hacer `add`:

```bash

git diff

```


2. Agregás el archivo al staging area:

```bash

git add index.html

```


3. Revisás lo que está staged:

```bash

git diff --staged

```


4. Hacés el commit:

```bash

git commit -m "Edité el index"

```


5. Querés ver ese commit:

```bash

git show

```


6. Comparás dos versiones:

```bash

git diff abc123 def456

```


##  1. `git diff HEAD`


### ¿Qué es `HEAD`?

  
`HEAD` es un puntero especial que siempre apunta al **último commit que hiciste**.

### ¿Qué hace este comando?


Compara lo que **tenés ahora editado en tu proyecto** con lo que hay en el último commit.

```bash

git diff HEAD

```


 Te muestra:

- Cambios en archivos que **todavía no están staged**.  

 **Útil para:** Ver si algo cambió desde tu último commit sin necesidad de hacer `git add`.


##  2. `git diff <commit>`


Ejemplo:


```bash

git diff 8ccd4

```


Compara ese commit (`8ccd4`) con el **estado actual** de tu proyecto.

 Te muestra:

- Lo que cambió desde ese commit hasta ahora.


 Si querés ver qué cambió desde un commit **más viejo**, este comando es ideal.

##  3. `git diff <commit1> <commit2>`


Ejemplo:

```bash

git diff 4437b 8ccd4

```

 Te muestra:


- Qué cambió entre dos commits específicos (comparación directa).
   
##  4. `git diff --name-only`

  
Te muestra **solo los nombres de archivos modificados**, no el contenido.


Ejemplo:


```bash

git diff --name-only

```

Salida:


```

index.html

styles.css

main.js

```


 **Útil para:** saber **rápido** qué archivos tocaste, sin ver los cambios completos.

##  5. `git diff --stat`


Muestra un **resumen estadístico**: cuántas líneas se agregaron o borraron por archivo.


```bash

git diff --stat

```


Salida:

  
```

index.html | 3 +++

app.js     | 2 +-

2 files changed, 4 insertions(+), 1 deletion(-)

```

   **Útil para:** ver una idea general de cuántos cambios hiciste sin ver el detalle.

##  6. `git diff --color-words`


Muestra los cambios **en línea y con color**, en vez de línea por línea.


Ejemplo:


```bash

git diff --color-words

```


🟢 Cosas nuevas se ven **resaltadas**, y lo que borraste también.

 Ideal cuando tenés **muchos cambios en pocas líneas**.


##  7. `git diff HEAD~1`, `HEAD~2`, etc.


Esto sirve para comparar **el commit anterior** o **anteriores**.

  

- `HEAD` → commit actual

- `HEAD~1` → uno antes

- `HEAD~2` → dos antes

  

Ejemplo:

  

```bash

git diff HEAD~1 HEAD

```

  

 **Comparás el commit anterior con el actual.**

  
##  Resumen visual (modo "chuleta")


| Comando                          | ¿Qué compara?                                  |

|----------------------------------|------------------------------------------------|

| `git diff`                       | Working directory vs último commit (no staged) |

| `git diff --staged` / `--cached` | Staged vs último commit                        |

| `git diff HEAD`                  | Working directory vs último commit             |

| `git diff <commit>`              | Working directory vs ese commit                |

| `git diff <commit1> <commit2>`   | Commit A vs Commit B                           |

| `git diff --name-only`           | Lista de archivos que cambiaron                |

| `git diff --stat`                | Resumen de líneas añadidas/borradas            |

| `git diff --color-words`         | Muestra los cambios en línea, con colores      |

| `git diff HEAD~1 HEAD`           | Commit anterior vs el actual                   |
