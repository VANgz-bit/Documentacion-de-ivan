## **¿Qué es Git?**

**Git** es un sistema de control de versiones.  
Fue creado por **Linus Torvalds** (el mismo que creó Linux) y sirve para **registrar y administrar los cambios** que hacés en tus archivos con el tiempo.

Es como una "máquina del tiempo" para tu código: podés guardar avances, volver atrás, trabajar en versiones diferentes, y colaborar con otras personas sin perder el control de nada.

---

## **¿Por qué usar Git?**

- Guardás cambios como si fueran "puntos de control".
    
- Podés volver a un estado anterior si rompés algo.
    
- Podés trabajar en distintas versiones al mismo tiempo.
    
- Podés colaborar con otros sin sobrescribir tu trabajo.
    
- Todo se guarda localmente (en tu propia computadora).
    

---

## **¿Cómo funciona Git? (Resumen lógico)**

1. **Iniciás un repositorio Git** en una carpeta (`git init`).
    
2. Git empieza a seguir los cambios en tus archivos.
    
3. Cada vez que hacés un cambio importante, lo **guardás con un "commit"** (como una captura del estado del proyecto).
    
4. Podés tener varias **ramas (branches)**, que son como versiones paralelas del mismo proyecto.
    
5. Si querés, subís todo a GitHub para guardar online y trabajar en equipo.
    

---

## **Términos básicos de Git**

|Término|¿Qué es?|
|---|---|
|**Repositorio**|Carpeta donde Git guarda todo el historial de tu proyecto.|
|**Commit**|Un "registro" de los cambios que hiciste (como una captura del momento).|
|**Branch (rama)**|Una versión alternativa del proyecto donde podés experimentar sin afectar la principal.|
|**Merge**|Combinar los cambios de una rama con otra.|
|**Staging area**|Zona donde preparás los archivos antes de confirmar un commit.|
|**HEAD**|El último commit donde estás trabajando ahora.|

---

## **Comandos básicos de Git**

|Comando|Función|
|---|---|
|`git init`|Inicializa un repositorio Git en una carpeta.|
|`git status`|Muestra los archivos modificados y su estado.|
|`git add archivo.html`|Agrega un archivo a la zona de preparación.|
|`git add .`|Agrega todos los archivos modificados.|
|`git commit -m "mensaje"`|Crea un commit con ese mensaje.|
|`git log`|Muestra el historial de commits.|
|`git checkout -b nueva-rama`|Crea y se cambia a una nueva rama.|
|`git merge otra-rama`|Une otra rama a la actual.|
