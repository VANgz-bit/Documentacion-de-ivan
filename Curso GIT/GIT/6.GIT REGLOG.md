

## ¿Qué es `git reflog`?

`git reflog` es una herramienta **interna y poderosa de Git** que permite **ver el historial de referencias que tuvo `HEAD`**, es decir, **todos los movimientos que hiciste**, incluso los que no se ven con `git log`.

> ¿Qué registra `reflog`?
> 
> - Commits nuevos
>     
> - Cambios de ramas
>     
> - Resets
>     
> - Rebases
>     
> - Merges
>     
> - Cherry-picks
>     
> - Commits que se "perdieron" (por error o accidente)
>     

En resumen: **si hiciste un cambio y lo perdiste**, `reflog` probablemente te puede ayudar a **recuperarlo**.

---

## ¿Cómo se usa?

```bash
git reflog
```

Este comando muestra un listado cronológico de las acciones recientes con sus hashes:

```bash
e2f9e3a HEAD@{0}: commit: fix errores del login
3f99b2a HEAD@{1}: checkout: moving from login to main
f5c67bb HEAD@{2}: reset: moving to f5c67bb
91b83f1 HEAD@{3}: commit: añadida navbar
```

Cada línea muestra:

- El **hash del commit**.
    
- La posición (`HEAD@{n}`), útil para referirse a ese punto.
    
- La acción realizada (commit, checkout, reset, etc.).
    
- El mensaje (si lo hubo).
    

---

## ¿Qué representa `HEAD@{n}`?

Es una forma de referirse a una **posición anterior del HEAD**.

- `HEAD@{0}` → posición actual del HEAD.
    
- `HEAD@{1}` → dónde estaba HEAD justo antes.
    
- `HEAD@{2}` → y así sucesivamente.
    

Esto es útil para volver a un punto anterior incluso si te equivocaste con un `reset`, `rebase`, o `commit --amend`.

---

## ¿Para qué sirve en la práctica?

### 1. Recuperar commits eliminados por error

Supongamos que se hace un:

```bash
git reset --hard HEAD~1
```

Esto borra el último commit. Pero `reflog` **sigue teniendo el registro**.

Podés usar:

```bash
git reflog
```

Identificar el commit que querés recuperar y hacer:

```bash
git checkout hash-del-commit
```

O incluso:

```bash
git reset --hard hash-del-commit
```

---

### 2. Volver a una rama después de un cambio

Si se cambia de rama accidentalmente:

```bash
git switch main
```

Y no recordás desde dónde viniste, podés revisar con `reflog`:

```bash
git reflog
```

Y volver con:

```bash
git switch -
```

O volver a un punto exacto con:

```bash
git checkout HEAD@{1}
```

---

### 3. Ver todos los movimientos ocultos (invisibles para `git log`)

`git log` solo muestra commits **que están en ramas actuales**.

`git reflog` también muestra:

- Commits descartados.
    
- Commits que existieron temporalmente.
    
- Cambios por `rebase`, `amend`, `reset`, etc.
    

---

## Comandos relacionados

### Ver solo el hash y la acción:

```bash
git reflog --date=relative
```

Esto muestra cuándo ocurrió cada acción (ej: _"2 minutes ago"_):

```text
HEAD@{0}: commit: arreglos finales          [2 minutes ago]
HEAD@{1}: checkout: moved from login to main [4 minutes ago]
```

---

### Volver a un estado anterior

```bash
git checkout HEAD@{2}
```

O restaurar por completo el estado con:

```bash
git reset --hard HEAD@{2}
```

> ⚠️ Cuidado: el `--hard` borra los cambios que no estén committeados.

---

### Eliminar el reflog (poco común)

Si necesitás limpiar completamente el reflog (por ejemplo, por seguridad o limpieza extrema):

```bash
git reflog expire --expire=now --all
git gc --prune=now
```

**¡Usar con cuidado! Esto elimina todo el historial de reflog.**

---

## Diferencia entre `git log` y `git reflog`

|Característica|`git log`|`git reflog`|
|---|---|---|
|Muestra historial de commits|Sí|Sí (aunque estén huérfanos)|
|Incluye cambios internos (reset, checkout)|No|Sí|
|Permite recuperar commits borrados|No directamente|Sí|
|Se basa en ramas activas|Sí|No, muestra todas las referencias|

---

## Resumen práctico

|Acción|Comando|
|---|---|
|Ver historial completo de HEAD|`git reflog`|
|Ver cuándo ocurrió cada cambio|`git reflog --date=relative`|
|Volver a un punto anterior|`git checkout HEAD@{n}`|
|Restaurar estado con reset|`git reset --hard HEAD@{n}`|
|Eliminar el reflog|`git reflog expire --expire=now --all`|
|Limpiar todo (garbage collector)|`git gc --prune=now`|
