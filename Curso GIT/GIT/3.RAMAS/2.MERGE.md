
## GIT – MERGE (sin conflictos)

### ¿Qué es `merge`?

`git merge` es un comando que se usa para **unir el contenido de una rama secundaria con la rama actual**, sin perder el historial de commits de ninguna de las dos.  
Esto permite **incorporar el trabajo hecho en otra rama** de forma segura.

---

### ¿Cuándo se usa?

Se usa típicamente después de que una rama secundaria (por ejemplo `pruebas`, `login`, `diseño`) ya tiene cambios listos para integrarse al desarrollo principal (`main` o `master`).

---

## ¿Cómo se hace un merge correctamente?

### 1. Cambiar a la rama donde se quieren recibir los cambios

Antes de fusionar, se debe estar ubicado en la rama que **va a recibir** los cambios.

```bash
git switch main
```

Esto posiciona el trabajo en la rama `main`.

---

### 2. Fusionar la otra rama

Luego se usa el comando `merge` para traer los cambios desde la rama secundaria:

```bash
git merge nombre-de-la-rama
```

**Ejemplo:**

```bash
git merge login
```

Esto fusiona los cambios que hay en la rama `login` con la rama actual (`main`).

---

### ¿Qué pasa después?

- Si Git **puede hacer el merge automáticamente**, se crea un **nuevo commit de merge**.
    
- El historial queda unido y se mantienen todos los commits originales de ambas ramas.
    
- No se pierde nada, y el resultado es un único flujo de trabajo.
    

---

### ¿Qué sucede con la rama fusionada?

Después de hacer el `merge`, la rama secundaria sigue existiendo. Se puede:

- **Seguir trabajando en ella**, si se van a hacer más cambios.
    
- **Borrarla**, si ya no se necesita, con:
    

```bash
git branch -d nombre-de-la-rama
```

**Ejemplo:**

```bash
git branch -d login
```

---

### ¿Puedo hacer merge de cualquier rama?

Sí. Se puede hacer merge desde cualquier rama hacia cualquier otra.  
Lo importante es **estar ubicado en la rama de destino**, es decir, la que va a recibir los cambios.

---

## Esquema visual simple

```text
main:   A---B---C
                \
pruebas:         D---E

(Después del merge en main)

main:   A---B---C-------F
                \     /
pruebas:         D---E
```

- `F` representa el commit de _merge_.
    
- La historia de ambas ramas se une.
    

---

## Resumen práctico

|Acción|Comando|
|---|---|
|Cambiar a la rama principal|`git switch main`|
|Unir otra rama con la actual|`git merge nombre-de-la-rama`|
|Ver ramas existentes|`git branch`|
|Borrar rama después del merge|`git branch -d nombre-de-la-rama`|
