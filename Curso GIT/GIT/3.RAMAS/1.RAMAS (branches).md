
## GIT – RAMAS (Branches)

### ¿Qué son las ramas?

En Git, una **rama** (_branch_) es una copia virtual del proyecto en un momento determinado.  
Permite trabajar de forma **aislada**, sin afectar el desarrollo principal, lo cual es útil para:

- Probar nuevas ideas o funciones.
    
- Hacer correcciones sin romper lo que ya funciona.
    
- Organizar tareas de forma ordenada.
    

Por defecto, cuando se crea un repositorio, se comienza con una rama principal que normalmente se llama `main` (en versiones antiguas, `master`).

---

### ¿Cómo funcionan las ramas?

Cada rama es como una **línea de tiempo paralela**, donde se puede hacer commits sin afectar otras ramas.  
Cuando se crea una rama nueva, se parte del punto actual de la rama activa.

 Por ejemplo: si se está en la rama `main` y se crea una nueva llamada `pruebas`, esta rama tendrá los mismos commits que `main` hasta ese momento. A partir de ahí, puede avanzar de forma independiente.

---

## COMANDOS RELACIONADOS A RAMAS

### 1. Crear una nueva rama

```bash
git branch nombre-de-la-rama
```

Este comando crea una rama nueva, **basada en la rama actual**, pero **no cambia a ella automáticamente**.

**Ejemplo:**

```bash
git branch diseño-header
```

Esto crea la rama `diseño-header`, pero se sigue trabajando en la rama donde uno estaba (por ejemplo, `main`).

---

### 2. Cambiar de rama

```bash
git switch nombre-de-la-rama
```

Este comando **cambia el contexto de trabajo** a la rama que se indica.

**Ejemplo:**

```bash
git switch diseño-header
```

A partir de ese momento, todos los commits nuevos van a la rama `diseño-header`.

---

### 3. Crear y cambiar a una nueva rama al mismo tiempo

```bash
git switch -c nombre-de-la-rama
```

Este comando es una combinación de los dos anteriores:

- Crea una nueva rama.
    
- Cambia inmediatamente a ella.
    

**Ejemplo:**

```bash
git switch -c seccion-contacto
```

Ahora se trabaja directamente en la rama `seccion-contacto`.

---

### 4. Ver todas las ramas

```bash
git branch
```

Este comando **muestra una lista de todas las ramas** del repositorio.

- La rama actual se indica con un asterisco `*`.
    
- Las otras ramas aparecen listadas debajo.
    

---

### 5. Renombrar una rama

#### a. Desde otra rama (forma general):

```bash
git branch -m nombre-viejo nombre-nuevo
```

Este comando **modifica el nombre de una rama existente**, pero se debe estar en otra rama para hacerlo.

**Ejemplo:**

```bash
git branch -m vieja-prueba nueva-prueba
```

#### b. Desde la misma rama (forma rápida):

Si se está **dentro de la rama que se desea renombrar**, se puede usar esta forma más directa:

```bash
git branch -m nuevo-nombre
```

**Ejemplo:**

```bash
git branch -m diseño-final
```

---

### 6. Borrar una rama

Para borrar una rama, **no se debe estar en la rama que se quiere eliminar**.

```bash
git branch -d nombre-de-la-rama
```

Este comando intenta borrar la rama, pero **solo si ya fue fusionada** o si Git considera que no hay riesgo de pérdida de información.

**Ejemplo:**

```bash
git branch -d seccion-contacto
```

#### Forzar el borrado:

Si se quiere borrar una rama sin que Git pregunte si es seguro (aunque no haya sido fusionada):

```bash
git branch -D nombre-de-la-rama
```

---

## Resumen práctico de comandos

|Acción|Comando|
|---|---|
|Crear una nueva rama|`git branch nombre-de-la-rama`|
|Cambiar a otra rama|`git switch nombre-de-la-rama`|
|Crear y cambiar a la vez|`git switch -c nombre-de-la-rama`|
|Ver todas las ramas|`git branch`|
|Renombrar desde otra rama|`git branch -m nombre-viejo nombre-nuevo`|
|Renombrar desde la misma rama|`git branch -m nombre-nuevo`|
|Borrar una rama (segura)|`git branch -d nombre-de-la-rama`|
|Borrar una rama (forzada)|`git branch -D nombre-de-la-rama`|
