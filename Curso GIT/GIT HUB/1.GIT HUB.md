
#  ¿Qué es GitHub? (explicación detallada)

**GitHub** es una **plataforma web** que te permite **almacenar**, **gestionar** y **colaborar** en proyectos de software que usan **Git** como sistema de control de versiones.

Podés pensar GitHub como una **nube especializada en código**. No es solo un lugar donde se suben archivos, sino un sistema completo que facilita el trabajo en equipo, el control de cambios, la revisión de código y hasta el despliegue de páginas web.

###  GitHub funciona junto con Git

Git es un programa que se usa de forma **local** en tu computadora. Pero si querés **compartir tu trabajo**, guardarlo en la **nube**, colaborar con otros o simplemente tener una copia de seguridad online, **necesitás una plataforma como GitHub**.

> En resumen:  
> **Git = sistema de control de versiones**  
> **GitHub = plataforma para alojar proyectos Git y colaborar en ellos**

---

#  ¿Para qué sirve GitHub?

GitHub tiene muchos usos y beneficios:

1. **Almacenar tus proyectos en línea** (código fuente, imágenes, documentación).
    
2. **Controlar el historial de cambios**: ver qué se cambió, cuándo y por quién.
    
3. **Colaborar en equipo** sin pisarse el trabajo.
    
4. **Revisar y aprobar cambios** mediante _pull requests_.
    
5. **Reportar errores o sugerencias** a través de _issues_.
    
6. **Mostrar tu portfolio profesional** como desarrollador.
    
7. **Publicar páginas web estáticas** con GitHub Pages.
    
8. **Automatizar tareas** con GitHub Actions (como pruebas o despliegues automáticos).
    

---

#  ¿Qué es un repositorio en GitHub?

Un **repositorio (repository)** es como una **carpeta especial en la nube** donde vive tu proyecto. Contiene:

- Archivos de código (HTML, CSS, JS, etc.).
    
- Documentación (como `README.md`).
    
- El historial de cambios (commits).
    
- Las ramas del proyecto (branches).
    
- Los colaboradores que tienen acceso.
    

Podés tener **repositorios públicos** (cualquiera los puede ver) o **privados** (solo vos o quienes invites).

#  ¿Cómo empezar a usar GitHub?

## 1. Crear una cuenta

1. Entrá a: [https://github.com](https://github.com/)
    
2. Hacé clic en **Sign up**.
    
3. Elegí un nombre de usuario, mail, contraseña, y confirmá tu cuenta.
    
4. Una vez registrado, ya podés crear repositorios y usar GitHub.
    

---

## 2. Crear un nuevo repositorio

1. Ingresá a tu cuenta.
    
2. En la parte superior derecha, hacé clic en el ícono de **"+" → "New repository"**.
    
3. Completá los siguientes campos:
    

|Campo|Explicación|
|---|---|
|**Repository name**|Nombre del proyecto. Ej: `mi-primer-web`.|
|**Description (opcional)**|Breve descripción del proyecto.|
|**Visibility**|Elegí entre `Public` (visible para todos) o `Private` (sólo vos o quien invites).|
|**Initialize with**|Podés marcar opciones como: `README.md`, `.gitignore`, o una licencia.|

4. Hacé clic en **Create repository**.
    

---

## 3. Subir un proyecto de tu computadora a GitHub

Para subir tu proyecto, primero tenés que tenerlo inicializado con Git. Ejemplo:

```bash
git init                      # Inicializa el repositorio local
git add .                     # Agrega todos los archivos al staging
git commit -m "Primer commit" # Crea un commit con mensaje
```

Luego lo conectás con GitHub:

```bash
git remote add origin https://github.com/TU_USUARIO/NOMBRE_REPO.git
git branch -M main
git push -u origin main
```

### ¿Qué hace cada comando?

- `git remote add origin URL`: le decís a Git que ese es tu repositorio remoto (en GitHub).
    
- `git branch -M main`: asegura que la rama se llame `main`.
    
- `git push -u origin main`: sube los cambios de la rama `main` al repositorio remoto `origin`.
    

---

## 4. Clonar un repositorio (descargarlo a tu PC)

Si querés descargar un proyecto de GitHub:

```bash
git clone https://github.com/usuario/repositorio.git
```

Esto **copia el proyecto completo** a tu computadora, junto con su historial.

---

## 5. Subir cambios nuevos a GitHub

Cada vez que editás archivos, hacés:

```bash
git add .
git commit -m "descripcion del cambio"
git push origin main
```

Así Git registra el cambio y lo sube a la nube.

---

# 📂 ¿Qué significa "origin" y "main"?

|Término|Significado|
|---|---|
|`origin`|Es el **alias del repositorio remoto** en GitHub (el que conectaste con `git remote add`).|
|`main`|Es el **nombre de la rama principal**. Antes era `master`, pero ahora se recomienda usar `main`.|
