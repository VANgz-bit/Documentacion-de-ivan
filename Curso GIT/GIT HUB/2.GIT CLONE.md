
## 📌 ¿Qué es `git clone`?

`git clone` es el comando que se usa para **copiar un repositorio remoto completo (como el de GitHub) a tu computadora**.
 En otras palabras: crea una **copia local exacta** de un proyecto remoto, incluyendo su historial de versiones, archivos, ramas, etc.

---

##  ¿Qué hace exactamente `git clone`?

1. Se conecta al repositorio remoto (GitHub, GitLab, etc).
    
2. Crea una carpeta con el mismo nombre del proyecto.
    
3. Descarga todos los archivos, commits, ramas y configuración Git.
    
4. Automáticamente crea una **conexión remota** llamada `origin`.
    

---

##  SINTAXIS GENERAL

```bash
git clone <URL-del-repositorio>
```

### Ejemplo:

```bash
git clone https://github.com/usuario/proyecto.git
```

Esto crea una carpeta llamada `proyecto` con todo el contenido del repositorio.

---

##  OPCIONES Y VARIANTES ÚTILES

###  1. Clonar en una carpeta con otro nombre

```bash
git clone https://github.com/usuario/proyecto.git nuevo-nombre
```

- Crea una carpeta con el nombre que vos quieras en lugar del original.
    

---

###  2. Clonar por SSH (si usás clave SSH)

```bash
git clone git@github.com:usuario/proyecto.git
```

- Es una forma más segura si ya configuraste una clave SSH en GitHub.
    

---

###  3. Clonar una rama específica

```bash
git clone -b nombre_rama https://github.com/usuario/proyecto.git
```

- Clona **solo una rama en particular**.
    
- Ejemplo:
    
    ```bash
    git clone -b dev https://github.com/usuario/proyecto.git
    ```
    

---

###  4. Clonar sin el historial completo (`--depth`)

```bash
git clone --depth=1 https://github.com/usuario/proyecto.git
```

- Clona **solo el último commit** (modo “shallow”).
    
- Ideal para ahorrar espacio si no necesitás todo el historial.
    

---

##  ¿Qué es la conexión `origin` que se crea?

Cuando clonás, Git guarda automáticamente la URL de origen (GitHub) como el **remoto llamado `origin`**.

Podés verificarlo con:

```bash
git remote -v
```

Ejemplo de salida:

```
origin  https://github.com/usuario/proyecto.git (fetch)
origin  https://github.com/usuario/proyecto.git (push)
```

---

##  ¿Qué pasa justo después de clonar?

Después de hacer `git clone`, Git:

1. Te deja ubicado dentro de la rama principal (por lo general `main`).
    
2. Ya podés comenzar a trabajar en tu copia local.
    
3. Si hacés `git log`, verás todo el historial del proyecto.
    
4. Si hacés `git remote -v`, ya está configurado el vínculo con GitHub.
    

---

##  ERRORES COMUNES
###  “Permission denied (publickey)”

- Intentaste clonar por SSH sin configurar la clave.
    
- Solución: agregá tu clave pública a GitHub o usá HTTPS.
    

---

###  “repository not found”

- La URL está mal o el repositorio es privado y no estás autenticado.
