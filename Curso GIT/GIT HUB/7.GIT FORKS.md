
### **¿Qué es un fork?**

**Fork** es una acción que realizas en plataformas de alojamiento de repositorios (como GitHub, GitLab o Bitbucket) y consiste en **crear una copia completa de un repositorio remoto en tu propia cuenta de usuario en ese servicio**.

**Características principales de fork:**

- Se usa principalmente en proyectos de colaboración y código abierto.
    
- Al hacer un fork, el repositorio original sigue existiendo y tu copia en tu cuenta es un repositorio completamente nuevo que mantiene un vínculo con el original (upstream).
    
- Sirve para proponer cambios al repositorio original (mediante _pull requests_) o para tener tu propia versión independiente del proyecto.
    

---

### **¿Qué es git clone?**

**`git clone`** es un comando que descargas y usas localmente en tu máquina.

- Toma un repositorio remoto (de GitHub, GitLab, o cualquier servidor Git) y lo copia en tu computadora, creando un repositorio local que incluye el historial completo de commits, ramas, y objetos del repositorio remoto.
    
- Puedes usarlo sobre repositorios propios o de otros, pero **no crea un nuevo repositorio en GitHub o en el servidor**, solo lo descarga a tu máquina.
    

---

### **Diferencias entre fork y git clone**

|Aspecto|Fork|Git Clone|
|---|---|---|
|**Dónde se hace**|En el servidor remoto (GitHub, GitLab, etc.)|En tu máquina local|
|**Qué crea**|Una copia del repositorio en tu cuenta remota|Una copia local del repositorio|
|**Propósito principal**|Colaborar y gestionar contribuciones con conexión al original (por ejemplo, pull requests)|Descargar un repositorio para trabajar localmente|
|**Relación con repositorio original**|Crea un vínculo explícito en la plataforma para gestión de contribuciones|No tiene vínculo en la plataforma; el vínculo es solo la configuración de la URL remota en Git|
|**Acceso**|Se hace a nivel de servidor y web|Se hace mediante la terminal y Git local|

---

### **Flujo típico de fork**

1. Haces un fork en GitHub → ahora tienes una copia del repositorio en tu cuenta.
    
2. Clonas tu fork a tu máquina local usando `git clone`.
    
3. Trabajas localmente, haces commits.
    
4. Subes cambios a tu fork remoto con `git push`.
    
5. Propones cambios al original haciendo un _pull request_ desde tu fork.
