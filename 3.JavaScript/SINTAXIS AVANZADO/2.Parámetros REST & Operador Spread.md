#  Parámetros REST & Operador Spread en JavaScript

---

##  1. Parámetros REST (`...rest`)

###  Definición

Los **parámetros REST** permiten que una función **reciba un número indefinido de argumentos** y los agrupe en un **array**.

Se colocan en la **definición de la función** como parámetro final, precedidos por `...`.

> **Dato clave:** Siempre que uses `...` como parámetro, lo que captura se convierte en un **array**, aunque no reciba nada.

---

###  Sintaxis

```js
function ejemplo(a, b, ...resto) {
  console.log(a);      // primer argumento
  console.log(b);      // segundo argumento
  console.log(resto);  // array con el resto de argumentos
  console.log(Array.isArray(resto)); // true
}

ejemplo(1, 2, 3, 4, 5);
// a = 1
// b = 2
// resto = [3, 4, 5]
// true
```

---

###  Características

- Solo puede haber **un parámetro REST** en una función.
    
- Debe ir **al final de los parámetros**.
    
- Siempre devuelve un **array** (aunque no se pasen valores → será `[]`).
    

---

###  Ejemplo práctico

```js
function sumar(a, b, ...c) {
  let resultado = a + b;

  c.forEach(function(n) {
    resultado += n;
  });

  return resultado;
}

console.log(sumar(1, 2));              // 3
console.log(sumar(1, 2, 3));           // 6
console.log(sumar(5, 10, 15, 20, 25)); // 75
```

En este ejemplo:

- `a` y `b` → son valores individuales.
    
- `...c` → siempre será un **array con los argumentos restantes**.

## 🔹 2. Operador Spread (`...spread`)

### Definición

El **operador Spread** también se escribe con `...`, pero funciona en el **lado contrario**:  
en lugar de agrupar, **expande un array u objeto en valores individuales**.

Se usa en **argumentos de funciones**, y también en la construcción de **arrays y objetos**.

---

###  Sintaxis

```js
const numeros = [1, 2, 3];
console.log(...numeros);
// Equivale a: console.log(1, 2, 3);
```

---

###  Usos comunes

1. **Pasar arrays como argumentos**
    
    ```js
    function sumar(a, b, c) {
      return a + b + c;
    }
    
    const numeros = [1, 2, 3];
    console.log(sumar(...numeros)); // 6
    ```
    
2. **Clonar arrays**
    
    ```js
    const arr1 = [1, 2, 3];
    const copia = [...arr1];
    console.log(copia); // [1, 2, 3]
    ```
    
3. **Combinar arrays**
    
    ```js
    const a = [1, 2];
    const b = [3, 4];
    const combinado = [...a, ...b];
    console.log(combinado); // [1, 2, 3, 4]
    ```
    
4. **Trabajar con objetos**
    
    ```js
    const obj1 = { nombre: "Iván", edad: 21 };
    const obj2 = { pais: "Argentina" };
    
    const persona = { ...obj1, ...obj2, activo: true };
    console.log(persona);
    // { nombre: "Iván", edad: 21, pais: "Argentina", activo: true }
    ```
    

---

## 🔍 Diferencia clave entre REST y Spread

|Aspecto|REST (`...`)|Spread (`...`)|
|---|---|---|
|**Acción**|Junta los argumentos restantes en un array|Expande un array/objeto en valores individuales|
|**Dónde se usa**|En la **definición** de funciones (parámetros)|En la **llamada** de funciones, arrays u objetos|
|**Resultado**|Siempre un **array**|Valores separados|
|**Ejemplo**|`function f(a, ...rest) {}`|`f(...[1, 2, 3])`|

---

##  Ejemplo combinado

```js
function mostrarDatos(nombre, ...hobbies) {
  console.log("Nombre:", nombre);
  console.log("Hobbies:", hobbies); // siempre un array
}

const hobbies = ["programar", "leer", "jugar voley"];

mostrarDatos("Iván", ...hobbies);

// Nombre: Iván
// Hobbies: ["programar", "leer", "jugar voley"]
```

👉 Aquí pasa lo interesante:

- `...hobbies` en la **definición** = **REST** → junta en array.
    
- `...hobbies` en la **llamada** = **SPREAD** → expande el array en argumentos.
    

---

##  Conclusión clara

- **REST** → Parámetros. Junta en un **array**.
    
- **SPREAD** → Argumentos / Arrays / Objetos. Expande en **valores individuales**.
    
- Mismo símbolo `...`, pero funciones **opuestas**.
