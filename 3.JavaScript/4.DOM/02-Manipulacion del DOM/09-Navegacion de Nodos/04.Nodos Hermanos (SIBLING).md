## 🔹 Propiedades de los hermanos (Siblings)

### 🔸 Introducción

En el árbol del DOM, los **hermanos** son los nodos que **comparten un mismo padre**.  
Por ejemplo, dentro de un contenedor `<ul>`, cada elemento `<li>` es hermano de los demás porque todos dependen del mismo nodo padre.  
Esta relación horizontal —a diferencia de la relación vertical entre padres e hijos— es esencial para recorrer estructuras secuenciales, como listas, tablas o grupos de elementos relacionados.

El DOM proporciona propiedades específicas para acceder a los nodos hermanos que están **antes o después** de un determinado elemento, lo cual resulta muy útil para operaciones de navegación y manipulación dinámica del contenido.

---

### 🔸 `.nextSibling`

La propiedad `.nextSibling` devuelve **el nodo hermano que aparece inmediatamente después** del nodo actual dentro del mismo padre.  
Es importante destacar que esta propiedad no discrimina el tipo de nodo, por lo que puede devolver nodos de texto (como saltos de línea o espacios) o comentarios.

**Ejemplo:**

```html
<ul>
  <li id="primero">Elemento 1</li>
  <li>Elemento 2</li>
</ul>

<script>
  const li = document.getElementById("primero");
  console.log(li.nextSibling);
</script>
```

**Salida:**

```
#text
```

**Explicación:**  
El navegador interpreta los saltos de línea y espacios entre etiquetas como nodos de texto.  
Por eso, el primer hermano real del `<li>` no es otro `<li>`, sino un nodo `#text` que representa el salto de línea.  
Esto demuestra que `.nextSibling` trabaja a nivel de nodos, no exclusivamente de elementos.

---

### 🔸 `.nextElementSibling`

Para evitar el problema anterior, el DOM ofrece la propiedad `.nextElementSibling`, que devuelve **únicamente el siguiente hermano que sea un nodo de tipo elemento**, ignorando los nodos de texto o comentarios.

**Ejemplo:**

```html
<ul>
  <li id="primero">Elemento 1</li>
  <li>Elemento 2</li>
</ul>

<script>
  const li = document.getElementById("primero");
  console.log(li.nextElementSibling);
</script>
```

**Salida:**

```
<li>Elemento 2</li>
```

**Explicación:**  
Ahora el acceso es limpio y preciso: se omiten los nodos no visuales y se obtiene directamente el siguiente elemento HTML.  
Esta propiedad resulta ideal cuando se trabaja con listas o bloques de contenido, ya que permite recorrer los elementos contiguos de forma predecible.

---

### 🔸 `.previousSibling`

La propiedad `.previousSibling` funciona de manera análoga a `.nextSibling`, pero en sentido contrario.  
Devuelve el **nodo hermano inmediatamente anterior**, sin importar su tipo.

**Ejemplo:**

```html
<ul>
  <li>Elemento 1</li>
  <li id="segundo">Elemento 2</li>
</ul>

<script>
  const li = document.getElementById("segundo");
  console.log(li.previousSibling);
</script>
```

**Salida:**

```
#text
```

**Explicación:**  
Una vez más, el salto de línea o espacio entre los dos `<li>` es interpretado como un nodo de texto.  
Por eso, el primer hermano anterior detectado no es un elemento HTML, sino un nodo textual.

---

### 🔸 `.previousElementSibling`

La propiedad `.previousElementSibling` devuelve **el hermano anterior que sea un nodo de tipo elemento**, ignorando cualquier nodo de texto o comentario.  
Esto garantiza una navegación estructural más confiable dentro del documento.

**Ejemplo:**

```html
<ul>
  <li>Elemento 1</li>
  <li id="segundo">Elemento 2</li>
</ul>

<script>
  const li = document.getElementById("segundo");
  console.log(li.previousElementSibling);
</script>
```

**Salida:**

```
<li>Elemento 1</li>
```

**Explicación:**  
El acceso mediante `.previousElementSibling` salta los nodos no relevantes, devolviendo directamente el elemento hermano anterior dentro de la jerarquía.  
Esto resulta especialmente útil cuando se recorre o manipula un conjunto de elementos secuenciales.

---

### 🔸 Consideraciones técnicas y de uso

Las propiedades relacionadas con los hermanos permiten navegar horizontalmente dentro del árbol DOM, complementando la navegación vertical basada en padres e hijos.  
Una de sus ventajas más notables es que permiten recorrer elementos **sin necesidad de acceder al nodo padre** de forma explícita.  
Por ejemplo, desde un `<li>` se puede pasar directamente al siguiente o al anterior sin subir al `<ul>` contenedor.

Sin embargo, es importante recordar que las propiedades que incluyen la palabra _“Element”_ (`nextElementSibling`, `previousElementSibling`) son las más seguras y convenientes para el trabajo cotidiano, ya que evitan los errores provocados por nodos de texto invisibles.

Por el contrario, las propiedades sin “Element” (`nextSibling`, `previousSibling`) pueden ser útiles cuando se necesita analizar el documento a nivel más profundo, incluyendo saltos de línea, espacios o comentarios, por ejemplo en scripts de análisis estructural o procesamiento de HTML.
