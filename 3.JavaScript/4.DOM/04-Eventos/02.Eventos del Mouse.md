# 🖱️ **Eventos del Mouse en JavaScript**

Los eventos del mouse son una de las formas más comunes de interacción entre el usuario y la página web. Estos eventos permiten detectar acciones como hacer clic, mover el cursor, presionar y soltar botones del mouse, entre otros.

Se asocian a elementos del DOM mediante **event listeners** o **event handlers**.

---

## 🟦 Lista de eventos más usados del mouse

|Evento|Se dispara cuando...|
|---|---|
|`click`|Se hace clic (presionar y soltar) sobre un elemento|
|`dblclick`|Se hace doble clic|
|`mousedown`|Se presiona (pero no se suelta) un botón del mouse|
|`mouseup`|Se suelta el botón del mouse|
|`mouseenter`|El mouse entra en el área del elemento (sin burbujeo)|
|`mouseover`|El mouse pasa por encima del elemento (con burbujeo)|
|`mouseleave`|El mouse sale del área del elemento (sin burbujeo)|
|`mouseout`|El mouse sale del elemento hacia otro (con burbujeo)|
|`mousemove`|El mouse se mueve sobre el elemento|
|`contextmenu`|Se hace clic con el botón derecho del mouse|

---

## 🟩 ¿Cómo se usan en el DOM?

Se suelen asociar así:

```js
boton.addEventListener("click", function(e) {
  console.log("Me clickeaste");
});
```

El primer parámetro es el tipo de evento: `"click"`, `"mouseenter"`, etc.

El segundo es el **callback** que se ejecuta cuando ocurre el evento.

---

## 🧠 ¿Qué trae el objeto `event` en eventos del mouse?

Cuando escuchamos un evento del mouse, el objeto `event` nos da información clave, como:

- `event.type` → tipo de evento (click, mouseover, etc).
    
- `event.target` → elemento donde ocurrió el evento.
    
- `event.clientX` y `clientY` → coordenadas en la ventana.
    
- `event.pageX` y `pageY` → coordenadas relativas a toda la página.
    
- `event.button` → botón del mouse que se presionó:
    
    - 0: izquierdo
        
    - 1: central (rueda)
        
    - 2: derecho
        
- `event.altKey`, `ctrlKey`, `shiftKey` → si están presionadas estas teclas al ocurrir el evento.
    

---

## 🧪 Ejemplo visual y práctico

```html
<button id="btn">Probá hacer click</button>
```

```js
btn.addEventListener("click", (event) => {
  console.log("Me hiciste click!");
  console.log("Coordenadas:", event.clientX, event.clientY);
  console.log("Botón:", event.button);
});
```

Si haces click, verás en consola:

```
Me hiciste click!
Coordenadas: 220 135
Botón: 0
```

## ✔️ ¿Cuál usar? (mouseenter vs mouseover)

- `mouseover`: se dispara **cada vez** que el mouse entra en un elemento o **en sus hijos**. Hace **burbujeo**.
    
- `mouseenter`: se dispara **solo cuando entra al elemento**, no cuenta sus hijos. **No hace burbujeo**.
    

Ejemplo:

```js
div.addEventListener("mouseover", ()=>console.log("Incluye hijos"));
div.addEventListener("mouseenter", ()=>console.log("Solo el div"));
```

---

## 🛑 Evitar acciones por defecto

Algunos eventos del mouse tienen acciones predeterminadas. Por ejemplo, el clic derecho abre el menú contextual.

Si querés evitarlo, usar:

```js
element.addEventListener("contextmenu", (e) => {
  e.preventDefault();
  console.log("Menú bloqueado");
});
```

---

## 🧱 Resumen esquemático

- Los **eventos del mouse** detectan interacciones físicas del usuario.
    
- Son esenciales para interfaces interactivas: botones, menús, drag & drop, etc.
    
- El **objeto `event`** trae información útil para reaccionar dinámicamente.
    
- `addEventListener()` es la forma **moderna y recomendada** de usarlos.
