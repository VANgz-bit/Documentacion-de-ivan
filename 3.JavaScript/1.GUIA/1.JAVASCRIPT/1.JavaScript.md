
## **¿Qué es JavaScript?**  
Es un lenguaje de programación interpretado, de alto nivel y multiparadigma que permite dotar de interactividad y dinamismo a las páginas web. Junto a HTML y CSS, es uno de los pilares del desarrollo frontend.

**Historia y ECMAScript:**  
Creado en 1995 por Brendan Eich en Netscape. En 1997 se estandarizó bajo el nombre ECMAScript (ECMA-262). ECMAScript es la especificación oficial del lenguaje, que define su sintaxis, características y funcionamiento. JavaScript es la implementación más popular de ECMAScript; en otras palabras, ECMAScript es el estándar y JavaScript es su versión práctica en navegadores. Desde ECMAScript 6 (2015), el lenguaje ha ganado muchas mejoras que lo hicieron más moderno y robusto.

**Sintaxis básica:**

- **Variables:** se usan `var`, `let` y `const` para declarar variables. Ejemplo:
    
    ```javascript
    let mensaje = "Hola mundo";
    console.log(mensaje);
    ```
    
- **Tipos de datos:** primitivos (number, string, boolean, undefined, null, symbol) y objetos.
    
- **Estructuras de control:** condicionales y bucles como `if`, `else`, `switch`, `for`, `while`.
    
- **Funciones:** declaración de funciones normales y funciones flecha.
    
- **Comentarios:** `//` para una línea, `/* ... */` para varias líneas.
    

**Características (explicadas):**

- **Interpretado y dinámico:** no requiere compilación y el tipo de las variables se determina en tiempo de ejecución.
	
- **Imperativo**: se lee el código de arriba a abajo como si fuese cascada.
	
- **Basado en prototipos:** usa prototipos para heredar propiedades, en lugar de clases tradicionales (aunque desde ES6 existe la sintaxis de clases).
    
- **Tipado débil:** no es necesario declarar tipos de variables; estos se asignan dinámicamente.
    
- **Multiparadigma:** permite usar estilos de programación funcional, orientado a objetos o imperativo.
    
- **Manipulación del DOM:** puede acceder y modificar los elementos de la página en tiempo real.
    
- **Programación asincrónica:** soporta asincronía mediante callbacks, promesas y async/await, lo que permite manejar tareas como peticiones a servidores sin bloquear el flujo principal.

**Ejemplo básico:**

```html
<!DOCTYPE html>
<html>
  <body>
    <h1 id="demo">Hola</h1>
    <button onclick="document.getElementById('demo').innerText='Texto cambiado'">Cambiar</button>
  </body>
</html>
```
