
### Formas de incluir JavaScript en HTML

JavaScript puede integrarse en páginas HTML de varias maneras. La elección depende del tamaño, complejidad del código y buenas prácticas para mantener el proyecto ordenado y eficiente.

---

#### 1. **JavaScript Inline (en línea) dentro de etiquetas HTML**

Esta es la forma más directa y simple: se inserta código JavaScript dentro de un atributo HTML, generalmente `onclick`, `onchange`, etc. Es útil para funciones muy simples o pruebas rápidas, pero no es recomendable para proyectos grandes por razones de mantenimiento y separación de responsabilidades.

**Ejemplo:**

```html
<button onclick="alert('Hola desde inline!')">Haz clic aquí</button>
```

_Explicación:_  
Cuando el usuario hace clic en el botón, se ejecuta la función `alert()` directamente declarada en el atributo `onclick`.

---

#### 2. **JavaScript dentro de la etiqueta `<script>` en el mismo archivo HTML**

Puedes escribir bloques completos de código JavaScript dentro de la etiqueta `<script>`. Esto se suele poner en el `<head>`, o al final del `<body>`, dependiendo del momento en que quieras que se ejecute el script.

**Ejemplo dentro del `<head>`:**

```html
<!DOCTYPE html>
<html>
<head>
  <script>
    console.log('JavaScript ejecutado en el head');
  </script>
</head>
<body>
  <h1>Hola Mundo</h1>
</body>
</html>
```

_Consideración:_  
Colocar scripts en el `<head>` puede bloquear la carga visual hasta que se ejecute el script, lo que puede ralentizar la experiencia del usuario.

---

**Ejemplo dentro del final del `<body>` (práctica recomendada):**

```html
<!DOCTYPE html>
<html>
<head>
  <title>Ejemplo Script al final</title>
</head>
<body>
  <h1>Hola Mundo</h1>

  <script>
    console.log('JavaScript ejecutado al final del body');
  </script>
</body>
</html>
```

_Explicación:_  
Al colocar el `<script>` justo antes de cerrar `</body>`, garantizas que todo el HTML se haya cargado antes de ejecutar el script. Esto previene errores al manipular elementos que aún no existen.

---

#### 3. **JavaScript externo: archivo `.js` vinculado con `<script src="archivo.js">`**

Para mantener el código limpio y modular, es mejor separar el JavaScript en archivos independientes. Luego se enlaza con la etiqueta `<script>` usando el atributo `src`.

**Ejemplo:**

1. Archivo `script.js`:
    

```javascript
console.log('JavaScript desde archivo externo');
function cambiarTexto() {
  document.getElementById('demo').innerText = 'Texto cambiado desde archivo externo';
}
```

2. Archivo HTML:
    

```html
<!DOCTYPE html>
<html>
<head>
  <title>Ejemplo JS Externo</title>
</head>
<body>
  <h1 id="demo">Texto original</h1>
  <button onclick="cambiarTexto()">Cambiar texto</button>

  <script src="script.js"></script>
</body>
</html>
```

_Beneficios:_

- Separación clara de estructura (HTML) y comportamiento (JS).
    
- Reutilización y mantenimiento sencillo.
    
- Mejora la legibilidad y escalabilidad del proyecto.
    

---

#### 4. **Atributos `defer` y `async` para controlar la carga de scripts externos**

Cuando incluyes scripts externos, puedes usar estos atributos para optimizar la carga y ejecución, evitando bloqueos en el renderizado de la página.

- **`defer`**  
    Indica que el script debe descargarse de forma paralela a la carga del HTML, pero su ejecución se pospone hasta que el documento HTML esté completamente parseado. Los scripts con `defer` se ejecutan en orden, en cuanto termina el parsing.
    

```html
<script src="script.js" defer></script>
```

- **`async`**  
    También descarga el script de forma paralela, pero su ejecución es inmediata cuando termina la descarga, sin esperar al parsing del HTML. Esto puede causar que se ejecute antes o después de otros scripts, no garantiza orden.
    

```html
<script src="script.js" async></script>
```

_Cuándo usar cada uno:_

- Usa `defer` cuando el script depende del DOM completo o de otros scripts.
    
- Usa `async` para scripts independientes que no afectan a otros (por ejemplo, Google Analytics).
    

---

#### 5. **Ejecutar JavaScript cuando el DOM está listo**

Algunos scripts necesitan que la estructura HTML esté cargada antes de ejecutarse. Hay varias formas de asegurarlo.

- Usar el evento `DOMContentLoaded`:
    

```html
<script>
  document.addEventListener('DOMContentLoaded', function() {
    console.log('DOM cargado y listo');
  });
</script>
```

- Usar `defer` en scripts externos (ya explicado).
    

---

#### 6. **Incluir JavaScript dentro de módulos (ES6 Modules)**

Desde ES6 se introdujo la posibilidad de usar módulos JavaScript con la etiqueta `<script type="module">`. Esto permite importar y exportar funciones, clases o variables entre archivos JS.

**Ejemplo:**

Archivo `module.js`:

```javascript
export function saludar() {
  console.log('Hola desde módulo');
}
```

Archivo HTML:

```html
<script type="module">
  import { saludar } from './module.js';
  saludar();
</script>
```

_Ventajas:_

- Código organizado y reutilizable.
    
- Soporte para dependencias y encapsulación.
    
- Mejor control sobre el ámbito de las variables.
    

---

### Resumen

|Forma|Uso recomendado|Ventajas|Ejemplo|
|---|---|---|---|
|Inline (en atributo)|Scripts simples y específicos|Fácil y rápido|`<button onclick="alert('Hola')">`|
|`<script>` en HTML|Código pequeño o scripts rápidos|Directo, no requiere archivo externo|`<script>console.log('Hola')</script>`|
|Archivo externo con `<script src="">`|Proyectos medianos a grandes|Código organizado y reusable|`<script src="app.js"></script>`|
|`<script defer>`|Scripts que dependen del DOM|No bloquea carga, ejecución ordenada|`<script src="app.js" defer></script>`|
|`<script async>`|Scripts independientes|Carga rápida|`<script src="analytics.js" async></script>`|
|`<script type="module">`|Código modular con import/export|Organización avanzada|`<script type="module">import ...</script>`|
