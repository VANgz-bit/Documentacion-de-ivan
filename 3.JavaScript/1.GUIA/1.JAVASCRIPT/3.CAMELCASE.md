
## 🐫 ¿Qué es el **Camel Case**?

**Camel Case** (o “notación de camello”) es una **convención de escritura** en la que se **unen varias palabras sin espacios**, y **cada palabra (excepto la primera) comienza con mayúscula**.

Se llama así porque las mayúsculas del medio se parecen a las jorobas de un camello 🐫.

---

##  Ejemplo de Camel Case

```js
let nombreUsuario;
function obtenerNombreCompleto() { ... }
const edadDelUsuario = 21;
```

---

##  Características del Camel Case

1. **No hay espacios** entre palabras.
    
2. La **primera palabra empieza en minúscula**.
    
3. Las **palabras siguientes comienzan en mayúscula**.
    
4. Se usa para mejorar la **legibilidad del código**.
    
5. Es una **convención común en JavaScript** y otros lenguajes como Java, Python, C#, etc.
    

---

### ¿Dónde se usa Camel Case en JavaScript?

|Elemento|Se recomienda usar Camel Case|
|---|---|
|✅ Variables|Sí|
|✅ Funciones|Sí|
|❌ Clases|No (usa Pascal Case)|
|❌ Constantes globales|No (usa UPPER_CASE)|

---

##  Comparación con otros estilos

|Nombre del estilo|Ejemplo|Uso típico|
|---|---|---|
|**camelCase**|`nombreUsuario`|Variables y funciones|
|**PascalCase**|`NombreUsuario`|Clases y constructores|
|**snake_case**|`nombre_usuario`|Muy usado en Python o bases de datos|
|**kebab-case**|`nombre-usuario`|Usado en nombres de archivos o CSS (no en JS)|

---

##  Buenas prácticas con Camel Case

- Usá **nombres descriptivos**, no abreviaciones confusas:
    
    ```js
    let temp = 23; // ❌
    let temperaturaActual = 23; // ✅
    ```
    
- Mantené la **coherencia** en todo tu código.
    
- No uses acentos, espacios ni caracteres especiales.
    

---

##  Malos ejemplos (NO es Camel Case)

```js
let nombre_usuario;     // snake_case ❌
let NombreUsuario;      // PascalCase ❌
let nombre-usuario;     // kebab-case ❌ (ni siquiera es válido en JS)
```

---

##  Práctica simple

Convertí los siguientes nombres a **camelCase**:

1. `Nombre_usuario_final` → `nombreUsuarioFinal`
    
2. `EDAD_TOTAL` → `edadTotal`
    
3. `es_mayor_de_edad` → `esMayorDeEdad`
    
4. `CONTAR-elementos` → `contarElementos`
