 # 📘 Variables y Tipos de Datos en JavaScript

## 🔹 ¿Qué es una variable?

Una **variable** es un **espacio de memoria con nombre** que permite **almacenar y manipular información** dentro de un programa.

Podés imaginarla como una **caja con etiqueta**:

- La **etiqueta** es el **nombre** (identificador).
    
- El **contenido** es el **valor** almacenado.
    

```js
let nombre = "Kevin";
```

En este ejemplo:

- `nombre` → identificador de la variable.
    
- `"Kevin"` → valor guardado, de tipo **string**.
    

⚠️ El valor de una variable puede **cambiar con el tiempo** si se usa `let` o `var`, mientras que con `const` permanece fijo.

---

## 🔹 Declarar, inicializar y modificar

1. **Declarar**: crear la variable sin asignarle aún un valor.
    
2. **Inicializar**: darle un valor por primera vez.
    
3. **Modificar**: cambiar su valor después de inicializarla.
    

```js
let numero;   // Declarar
numero = 29;  // Inicializar
numero = 45;  // Modificar
```

---

## 🔹 `var`, `let` y `const`

En JavaScript existen **tres formas de declarar variables**, cada una con sus particularidades:

|Forma|Descripción|¿Reasignable?|Ámbito (scope)|
|---|---|---|---|
|`var`|Forma antigua, en desuso.|Sí|Global o de función.|
|`let`|Forma moderna y recomendada.|Sí|Bloque `{}`.|
|`const`|Constante, no puede cambiar.|No|Bloque `{}`.|

Ejemplo:

```js
var edad = 18;        // variable global o de función
let nombre = "Kevin"; // variable de bloque
const pais = "Argentina"; // constante obligatoriamente inicializada
```

⚠️ **`const` debe inicializarse al declararse**:

```js
const numero; // ❌ Error
numero = 2;
```

---

## 🔹 Scope (Alcance)

El **scope** define **dónde puede ser utilizada una variable**.

- **Global:** accesible en todo el programa.
    
- **De función:** accesible solo dentro de la función donde se declara.
    
- **De bloque:** accesible únicamente dentro de `{ }` si se usa `let` o `const`.
    

```js
{
  let nombre = "Kevin"; // solo existe dentro de este bloque
}
console.log(nombre); // ❌ Error
```

---

## 🔹 Hoisting

El **hoisting** (elevación) significa que las **declaraciones** de variables y funciones se “mueven” al inicio del contexto en tiempo de ejecución.

- `var` → se eleva, pero su valor inicial es `undefined`.
    
- `let` y `const` → se elevan, pero quedan en la **zona muerta temporal (TDZ)** hasta ser declaradas.
    

```js
console.log(x); // undefined
var x = 10;

console.log(y); // ❌ ReferenceError
let y = 20;
```

---

## 🔹 Crear múltiples variables

Se pueden declarar varias en una sola línea o en varias:

```js
let a, b, c;
a = 1;
b = 2;
c = 3;

let x = 1, y = 2, z = 3;
```

# 📘 Identificadores en JavaScript

### 🔹 ¿Qué son los identificadores?

Un **identificador** es el **nombre** que se le da a una variable, función, clase, objeto, parámetro, etc.  
Son las **etiquetas** que usamos para referirnos a valores y entidades en el código.

```js
let edad = 21;
function saludar() {
  console.log("Hola");
}
```

Aquí, `edad` y `saludar` son **identificadores**.

---

### 🔹 Reglas para crear identificadores válidos

1. **Primer carácter permitido**: letra, guion bajo (`_`) o signo dólar (`$`).  
    ❌ No puede empezar con un número.
    

```js
let nombre;   // ✅
let _oculto;  // ✅
let $precio;  // ✅
let 2dato;    // ❌ inválido
```

2. **Caracteres posteriores**: pueden ser letras, números, `_` o `$`.
    
3. **Case-sensitive**: `nombre` ≠ `Nombre`.
    
4. **No usar palabras reservadas** (`class`, `function`, `return`, etc.).
    

---

### 🔹 Convenciones de uso

- **camelCase** → variables y funciones.
    
- **PascalCase** → clases.
    
- **MAYÚSCULAS_CON_GUIONES** → constantes inmutables.

# 📘 Tipos de Datos en JavaScript

## 🔹 Primitivos

- `string` → texto.
    
- `number` → números.
    
- `bigint` → enteros muy grandes.
    
- `boolean` → verdadero o falso.
    
- `undefined` → variable declarada sin valor.
    
- `null` → valor vacío intencional.
    
- `symbol` → valor único e inmutable.
    

👉 Son **inmutables** y se comparan **por valor**.

> NOTA: aca solo se menciona los tipos de datos pero los desarrollaremos en otro lugar mas a fondo cada tipo

---

## 🔹 De referencia

- `object` → colecciones de pares clave: valor.
    
- `array` → listas indexadas.
    
- `function` → bloques de código reutilizables.
    
- Otros especiales → `Date`, `RegExp`, `Map`, `Set`, etc.
    

👉 Son **mutables** y se comparan **por referencia**.

# 📘 El operador `typeof`

### 🔹 ¿Qué es?

`typeof` es un **operador unario** en JavaScript que devuelve una **cadena de texto con el tipo de dato** del valor evaluado.

Se utiliza para **verificar tipos dinámicamente** en tiempo de ejecución.

---

### 🔹 Ejemplos básicos

```js
typeof "hola";     // "string"
typeof 42;         // "number"
typeof true;       // "boolean"
typeof undefined;  // "undefined"
typeof null;       // "object"  ← bug histórico
typeof Symbol();   // "symbol"
```

---

### 🔹 Ejemplos con objetos y funciones

```js
typeof {};        // "object"
typeof [];        // "object"  ← los arrays son objetos especiales
typeof function() {}; // "function"
```

---

### 🔹 Casos curiosos

- `typeof NaN; // "number"` → NaN es considerado un número inválido.
    
- `typeof null; // "object"` → error heredado de las primeras versiones de JS.
    
- `typeof [] === "object"` → no diferencia arrays de objetos.
    

✅ Para detectar arrays correctamente:

```js
Array.isArray([1, 2, 3]); // true
```

---

# 📘 Objeto Global: `window` y `globalThis`

- En navegadores: `window`.
    
- En Node.js: `global`.
    
- En ES2020+: `globalThis` (portátil entre entornos).
    

```js
var miVar = 10;
console.log(window.miVar); // navegador
console.log(globalThis.miVar); // estándar
```

# 📘 Coerción de Tipos

### 🔹 ¿Qué es?

La **coerción** es la conversión de un valor de un tipo a otro. Puede ser:

1. **Explícita (casting)** → el programador fuerza el cambio.
    
2. **Implícita** → JavaScript convierte automáticamente según el contexto.
    

---

### 🔹 Coerción explícita

```js
String(123);    // "123"
Number("10");   // 10
Boolean(0);     // false
```

---

### 🔹 Coerción implícita

```js
1 + "2";     // "12" (concatena)
"5" - 2;     // 3   (convierte string a número)
true + true; // 2
```

---

### 🔹 Truthy y Falsy

- **Falsy:** `false`, `0`, `-0`, `0n`, `""`, `null`, `undefined`, `NaN`.
    
- **Truthy:** todo lo demás (`"hola"`, `[]`, `{}`, etc.).
    

---

### 🔹 Buenas prácticas

- Usar **coerción explícita** siempre que sea posible.
    
- Preferir `===` en lugar de `==`.
    
- Conocer valores falsy para condicionales.
    

---

📌 **Resumen general:**

- Variables → cajas con nombre para guardar datos.
    
- Identificadores → etiquetas que nombran a variables, funciones, etc.
    
- Tipos de datos → primitivos (inmutables) y de referencia (mutables).
    
- `typeof` → operador que devuelve el tipo de un valor.
    
- Coerción → conversión automática o forzada entre tipos.
