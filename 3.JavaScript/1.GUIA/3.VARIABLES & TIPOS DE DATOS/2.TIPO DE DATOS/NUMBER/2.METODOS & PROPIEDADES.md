# 📘 Número (`Number`) en JavaScript – Documentación Completa

---

## 🔹 1. Definición general de `Number`

En JavaScript, **`Number`** es un **objeto global** que representa valores numéricos primitivos y proporciona un conjunto de **propiedades y métodos** para trabajar con ellos.

Los números en JavaScript se almacenan siguiendo el **estándar IEEE 754 de doble precisión (64 bits)**, lo que significa que:

1. Todos los números (excepto `BigInt`) son de **coma flotante de 64 bits**.
    
2. Esto permite representar valores muy grandes o muy pequeños.
    
3. No todos los números pueden representarse exactamente, especialmente los decimales y los enteros muy grandes.
    
4. Por eso existen propiedades que muestran los **límites y valores especiales**, y métodos que ayudan a **validar, convertir y formatear** los números.
    

`Number` incluye tres grandes categorías:

1. **Propiedades estáticas** → constantes y límites (`MAX_VALUE`, `NaN`, `Infinity`, etc.).
    
2. **Métodos estáticos** → validación y conversión (`isNaN`, `parseInt`, `isFinite`, etc.).
    
3. **Métodos de instancia** → formateo y representación (`toFixed`, `toExponential`, `toPrecision`, `toString`, `valueOf`).
    

---

## 🔹 2. Propiedades de `Number`

Cada propiedad tiene **definición, ejemplo y caso especial**.

### 2.1 `Number.MAX_VALUE`

**Definición:**  
Es el **número positivo más grande** que JavaScript puede representar de manera aproximada. Los números mayores se representan como `Infinity`.

**Ejemplo bueno:**

```js
let limite = Number.MAX_VALUE;
console.log(limite); // 1.7976931348623157e+308
console.log(limite * 0.5); // 8.988465674311579e+307
```

**Ejemplo malo:**

```js
let x = 2e308; 
console.log(x); // Infinity (se pasó del límite sin controlarlo)
```

**Caso especial / curiosidad:**  
Multiplicar cualquier número mayor a `MAX_VALUE` devuelve `Infinity`.

**Buena práctica:**  
Usar `MAX_VALUE` para comparar rangos y evitar overflow en cálculos.

---

### 2.2 `Number.MIN_VALUE`

**Definición:**  
Representa el **número positivo más pequeño** mayor que cero que JavaScript puede manejar.

**Ejemplo:**

```js
console.log(Number.MIN_VALUE); // 5e-324
console.log(Number.MIN_VALUE / 2); // 0
```

**Caso especial:**  
No representa negativos, solo el mínimo positivo. Cualquier división que dé un número menor que esto se considera 0.

**Buena práctica:**  
Ideal para comparaciones de números muy pequeños o cercanos a cero.

---

### 2.3 `Number.MAX_SAFE_INTEGER`

**Definición:**  
Mayor entero representable con precisión exacta: `2^53 - 1`.

**Ejemplo bueno:**

```js
console.log(Number.MAX_SAFE_INTEGER); // 9007199254740991
console.log(Number.MAX_SAFE_INTEGER + 1); // 9007199254740992 ✅
```

**Ejemplo malo:**

```js
console.log(Number.MAX_SAFE_INTEGER + 2); // 9007199254740992 ❌ debería ser 9007199254740993
```

**Caso especial:**  
Los enteros mayores pierden exactitud.

**Buena práctica:**  
Usar `Number.isSafeInteger()` para validar cálculos grandes.

---

### 2.4 `Number.MIN_SAFE_INTEGER`

**Definición:**  
Menor entero negativo representable de manera exacta: `-(2^53 - 1)`.

**Ejemplo:**

```js
console.log(Number.MIN_SAFE_INTEGER); // -9007199254740991
```

**Buena práctica:**  
Verificar valores negativos grandes con `Number.isSafeInteger()`.

---

### 2.5 `Number.NaN`

**Definición:**  
“Not a Number”. Representa resultados de operaciones inválidas o que no se pueden calcular como números.

**Ejemplo bueno:**

```js
console.log(Number.NaN); // NaN
console.log(0 / 0); // NaN
```

**Ejemplo malo:**

```js
console.log(NaN === NaN); // false
```

**Caso especial:**  
Nunca es igual a sí mismo, por eso se debe usar `Number.isNaN()` para comprobarlo.

**Buena práctica:**  
Evitar comparaciones directas y siempre validar con `Number.isNaN(valor)`.

---

### 2.6 `Number.POSITIVE_INFINITY`

**Definición:**  
Representa el infinito positivo. Aparece cuando un cálculo excede `MAX_VALUE` o al dividir por cero.

**Ejemplo:**

```js
console.log(Number.POSITIVE_INFINITY); // Infinity
console.log(1 / 0); // Infinity
```

**Caso especial:**  
Cualquier número positivo comparado con `Infinity` será menor.

**Buena práctica:**  
Evitar depender de `Infinity` en lógica central; usarlo solo para validaciones o límites.

---

### 2.7 `Number.NEGATIVE_INFINITY`

**Definición:**  
Representa el infinito negativo.

**Ejemplo:**

```js
console.log(Number.NEGATIVE_INFINITY); // -Infinity
console.log(-1 / 0); // -Infinity
```

**Caso especial:**  
Cualquier número negativo comparado con `-Infinity` será mayor.

**Buena práctica:**  
Similar a `POSITIVE_INFINITY`, usar solo en validaciones de límite.

---

### 2.8 `Number.EPSILON`

**Definición:**  
Diferencia mínima entre 1 y el siguiente número representable en JavaScript.  
Se utiliza para comparaciones aproximadas de decimales.

**Ejemplo:**

```js
console.log(Number.EPSILON); // 2.220446049250313e-16
console.log(Math.abs(0.1 + 0.2 - 0.3) < Number.EPSILON); // true
```

**Buena práctica:**  
Usar para comparar decimales y evitar errores de precisión.

# 🔹 3. Métodos estáticos de `Number`

Cada método incluye **definición, ejemplo y casos especiales**.

### 3.1 `Number.isNaN(valor)`

**Definición:**  
Verifica si un valor es `NaN` y es de tipo número.

**Ejemplo:**

```js
console.log(Number.isNaN(NaN)); // true
console.log(Number.isNaN("hola")); // false
```

**Caso especial:**  
No convierte valores como el global `isNaN`, solo evalúa `number`.

**Buena práctica:**  
Siempre usar `Number.isNaN()` en lugar de `isNaN()` global.

---

### 3.2 `Number.isFinite(valor)`

**Definición:**  
Chequea si el valor es un número finito (no `Infinity`, `-Infinity` o `NaN`).

**Ejemplo:**

```js
console.log(Number.isFinite(10)); // true
console.log(Number.isFinite(Infinity)); // false
```

**Buena práctica:**  
Útil para validar que los resultados de cálculos sean valores numéricos válidos.

---

### 3.3 `Number.isInteger(valor)`

**Definición:**  
Verifica si un número es entero.

**Ejemplo:**

```js
console.log(Number.isInteger(25)); // true
console.log(Number.isInteger(25.5)); // false
```

**Buena práctica:**  
Usar cuando se necesitan valores enteros exactos.

---

### 3.4 `Number.isSafeInteger(valor)`

**Definición:**  
Verifica si el número es un entero dentro del rango seguro (`MIN_SAFE_INTEGER` a `MAX_SAFE_INTEGER`).

**Ejemplo:**

```js
console.log(Number.isSafeInteger(9007199254740991)); // true
console.log(Number.isSafeInteger(9007199254740992)); // false
```

**Buena práctica:**  
Ideal para validar cálculos con enteros grandes.

---

### 3.5 `Number.parseInt(cadena)`

**Definición:**  
Convierte una cadena en número entero, ignorando caracteres no numéricos después del número inicial.

**Ejemplo:**

```js
console.log(Number.parseInt("123px")); // 123
console.log(Number.parseInt("123.45")); // 123
```

**Caso especial:**  
Solo toma la parte inicial que puede convertirse a número.

---

### 3.6 `Number.parseFloat(cadena)`

**Definición:**  
Convierte una cadena en número decimal.

**Ejemplo:**

```js
console.log(Number.parseFloat("3.14m")); // 3.14
console.log(Number.parseFloat("10.5px")); // 10.5
```

**Caso especial:**  
Igual que `parseInt`, ignora caracteres no numéricos al final.

---

## 🔹 4. Métodos de instancia de `Number`

Estos métodos formatean o representan números.

### 4.1 `toExponential(n)`

**Definición:**  
Retorna el número en notación científica con `n` decimales.

**Ejemplo:**

```js
console.log((12345).toExponential(2)); // "1.23e+4"
```

**Caso especial:**  
`n` opcional; si no se define, usa todos los decimales necesarios.

---

### 4.2 `toFixed(n)`

**Definición:**  
Devuelve una cadena con exactamente `n` decimales.

**Ejemplo bueno:**

```js
let precio = 25.6789;
console.log(precio.toFixed(2)); // "25.68"
```

**Ejemplo malo:**

```js
let total = precio.toFixed(2) + 1;
console.log(total); // "25.681" ❌ concatenación de string
```

**Buena práctica:**  
Convertir a número antes de operar:

```js
let total = Number(precio.toFixed(2)) + 1; // 26.68
```

---

### 4.3 `toPrecision(n)`

**Definición:**  
Retorna el número con `n` cifras significativas.

**Ejemplo:**

```js
console.log((123.456).toPrecision(4)); // "123.5"
```

---

### 4.4 `toString(base)`

**Definición:**  
Convierte el número a cadena en la base indicada (2, 8, 16…).

**Ejemplo:**

```js
console.log((255).toString(16)); // "ff"
```

---

### 4.5 `valueOf()`

**Definición:**  
Devuelve el valor primitivo del objeto `Number`.

**Ejemplo:**

```js
console.log((new Number(10)).valueOf()); // 10
```

---

## 🔹 5. Representación interna: Coma flotante de doble precisión (64 bits)

### 5.1 Coma flotante

**Definición:**  
Cada número se guarda en **3 partes**: signo, exponente y mantisa. Esto permite representar números **muy grandes y muy pequeños**.

**Ejemplo:**

```js
console.log(0.1 + 0.2); // 0.30000000000000004
```

**Caso especial:**  
Errores de decimales se deben a la representación binaria.

---

### 5.2 Doble precisión (64 bits)

|Parte|Bits|Función|
|---|---|---|
|Signo|1|Positivo (0) o negativo (1)|
|Exponente|11|Controla la magnitud del número|
|Mantisa|52|Contiene los dígitos significativos|

**Ejemplo:**

```js
console.log(9007199254740991 + 1); // 9007199254740992 ✅
console.log(9007199254740991 + 2); // 9007199254740992 ❌
```

**Buena práctica:**  
Para enteros mayores usar `BigInt`; para decimales exactos usar librerías especializadas.

---

## 🔹 6. Resumen final y buenas prácticas

1. Conocer los **límites y propiedades de Number** evita errores de precisión.
    
2. Siempre usar **`Number.isNaN()`**, **`Number.isSafeInteger()`** y **`Number.EPSILON`** cuando corresponda.
    
3. Convertir a número los métodos que retornan string antes de operar (`toFixed`, `toPrecision`).
    
4. Usar `BigInt` para enteros enormes y librerías como `Decimal.js` para decimales precisos.
    
5. Entender la **representación interna** ayuda a prever errores en cálculos y comparaciones.
