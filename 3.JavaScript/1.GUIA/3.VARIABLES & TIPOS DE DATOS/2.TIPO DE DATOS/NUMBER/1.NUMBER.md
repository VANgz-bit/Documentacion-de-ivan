# 📌 `Number` en JavaScript

## 1. Definición

En JavaScript, **`Number`** es el tipo de dato primitivo que representa valores numéricos, ya sean **enteros** o **decimales (de coma flotante)**.

- Todos los números en JavaScript se representan en el **formato de punto flotante de doble precisión (64 bits)** definido por el estándar **IEEE 754**.
    
- Esto significa que **no existen enteros puros** como en otros lenguajes (Java, C), sino que todo número, incluso `5`, es internamente un número de punto flotante.
    
- JavaScript permite representar:
    
    - Enteros: `10`, `-34`, `0`
        
    - Decimales: `3.14`, `-0.5`
        
    - Valores especiales: `Infinity`, `-Infinity`, `NaN`
        

Además de ser un tipo primitivo, existe el **objeto envolvente (`Number`)**, que permite usar métodos y propiedades para manipular o convertir números.

---

## 2. Sintaxis

Un número se puede declarar de distintas formas:

```js
// Entero
let entero = 42; 

// Decimal
let decimal = 3.14; 

// Notación científica
let cientifico = 1.5e3; // 1500

// Números negativos
let negativo = -25;

// Valores especiales
let infinito = Infinity;
let noNumero = NaN;

// Usando el objeto Number
let numeroObjeto = new Number(100); // Objeto, no primitivo
```

📍 **Señaladores importantes**:

- `let entero = 42;` → número entero.
    
- `let decimal = 3.14;` → número decimal.
    
- `1.5e3` → notación científica (`e` significa "exponente base 10").
    
- `Infinity` y `NaN` son valores numéricos especiales.
    
- `new Number()` crea un objeto, pero en la práctica se recomienda usar valores primitivos.
    

---

## 3. Características

- **Unificación de números**: No existe un tipo separado para enteros, todos los números son de punto flotante de 64 bits.
    
- **Precisión limitada**: Los números grandes o con muchos decimales pierden exactitud.
    
    - Ejemplo: `0.1 + 0.2 === 0.3` → `false` (por cómo funciona IEEE 754).
        
- **Valores especiales**:
    
    - `Infinity` y `-Infinity`: representan valores infinitos.
        
    - `NaN` ("Not a Number"): indica un valor inválido en una operación matemática.
        
- **Objeto envolvente**: `Number` como objeto tiene métodos útiles (`toFixed()`, `isNaN()`, etc.).
    

---

## 4. Ejemplos buenos y malos

✅ **Ejemplo correcto**

```js
let precio = 99.99;
let cantidad = 3;
let total = precio * cantidad;
console.log(total); // 299.97
```

❌ **Ejemplo problemático (precisión)**

```js
console.log(0.1 + 0.2); // 0.30000000000000004
```

📍 Este error ocurre por la representación binaria de los decimales.

---

## 5. Casos especiales o curiosos

- `NaN` es de tipo `number`:
    

```js
typeof NaN; // "number"
```

- `Infinity` también es considerado un número:
    

```js
typeof Infinity; // "number"
```

- `parseInt()` y `parseFloat()` convierten cadenas a números, pero con diferencias:
    

```js
parseInt("10.5"); // 10
parseFloat("10.5"); // 10.5
```

- Comparaciones con `NaN`:
    

```js
NaN === NaN; // false
Number.isNaN(NaN); // true
```

---

## 6. Buenas prácticas

- Usar **valores primitivos** (`let x = 10;`) en lugar de `new Number()`.
    
- Validar con `Number.isNaN()` en vez de `isNaN()` global (este último convierte antes de comprobar).
    
- Para operaciones financieras o con alta precisión, considerar librerías como **BigInt** o librerías externas (`decimal.js`, `big.js`).
    
- Usar `Number.isFinite()` para validar que un valor es numérico finito.
    

---

## 7. Resumen

- `Number` representa todos los valores numéricos en JavaScript bajo el estándar IEEE 754 (64 bits).
    
- Incluye enteros, decimales, `Infinity`, `-Infinity` y `NaN`.
    
- Puede usarse como **primitivo** o como **objeto envolvente**.
    
- Tiene limitaciones de precisión que deben tenerse en cuenta.
    
- Posee métodos útiles para conversión, redondeo y validación.
