# 📌 Literales de Expresiones Regulares (`RegExp`)

## 🔹 Definición general

Un **literal de expresión regular** en JavaScript es una manera directa de **crear un objeto `RegExp`**.  
La notación consiste en escribir el patrón entre barras inclinadas (`/`), a lo que opcionalmente se le añaden **banderas (flags)** que ajustan cómo funciona la búsqueda:

```js
/patrón/flags
```

A diferencia de otros literales (como los números `10`, los strings `"hola"` o el booleano `true`), un literal `RegExp` **no crea un valor primitivo**, sino un **objeto especial de la clase `RegExp`**.  
Esto significa que un literal regex es un caso particular: parece un simple valor escrito directamente en el código, pero en realidad es un objeto con propiedades y métodos propios.

---

## 🔹 Estructura del literal

El literal se compone de tres partes principales:

- **Delimitadores** `/.../` → Indican el inicio y fin de la expresión regular.
    
- **Patrón (pattern)** → Es la secuencia de caracteres y metacaracteres que define lo que queremos buscar. Ejemplo: `\d+` significa “uno o más dígitos”.
    
- **Banderas (flags)** → Modifican el comportamiento del patrón. Algunas de las más usadas son:
    
    - `g` → búsqueda global (todas las coincidencias, no solo la primera).
        
    - `i` → insensible a mayúsculas y minúsculas.
        
    - `m` → multilínea, afecta `^` y `$`.
        
    - `s` → dotAll, permite que `.` capture también saltos de línea.
        
    - `u` → soporte para Unicode.
        
    - `y` → sticky, obliga a que coincida en la posición exacta donde quedó la búsqueda anterior.
        

---

## 🔹 Literal vs. Objeto `RegExp`

Un literal `/hola/` y un objeto `new RegExp("hola")` son, en el fondo, lo mismo: ambos crean una instancia de la clase `RegExp`. La diferencia es la **forma de construcción**:

- El **literal** se escribe directamente en el código y es más conciso, más rápido de leer y más eficiente en ejecución, porque se construye una sola vez en tiempo de compilación.
    
- El **constructor** (`new RegExp()`) se usa cuando el patrón debe crearse dinámicamente, a partir de variables o cadenas construidas en tiempo de ejecución.
    

Ejemplo con literal:

```js
let regex1 = /hola/i;
console.log(regex1.test("HOLA mundo")); // true
```

Ejemplo con constructor dinámico:

```js
let palabra = "hola";
let regex2 = new RegExp(palabra, "i");
console.log(regex2.test("HOLA mundo")); // true
```

Con literal, siempre buscarías `/palabra/` literalmente. Con el constructor podés usar el valor de una variable para armar la expresión.

## 🔹 Métodos principales de `RegExp`

Un literal de regex por sí mismo no hace nada. La potencia aparece cuando lo combinamos con los métodos del objeto `RegExp` o con los de las cadenas de texto (`String`).

Los más importantes son:

- **`.test(cadena)`** → Devuelve `true` o `false` según si el patrón aparece en la cadena.
    
    ```js
    /gato/.test("Tengo un gato"); // true
    /gato/.test("Tengo un perro"); // false
    ```
    
- **`.exec(cadena)`** → Devuelve un objeto con detalles de la coincidencia (texto exacto, índice, grupos capturados). Si no encuentra nada, devuelve `null`.
    
    ```js
    /gato/.exec("Tengo un gato");
    // Devuelve ["gato", index: 9, input: "Tengo un gato"]
    ```
    
- **`.match(regex)`** (en `String`) → Devuelve un array con todas las coincidencias o `null` si no hay ninguna.
    
    ```js
    "uno dos tres".match(/\w+/g);
    // ["uno", "dos", "tres"]
    ```
    
- **`.replace(regex, reemplazo)`** (en `String`) → Sustituye coincidencias según un patrón.
    
    ```js
    "123-456".replace(/\d/g, "X");
    // "XXX-XXX"
    ```
    
- **`.search(regex)`** (en `String`) → Devuelve la posición de la primera coincidencia o `-1` si no encuentra nada.
    
    ```js
    "Hola mundo".search(/mundo/);
    // 5
    ```
    
- **`.split(regex)`** (en `String`) → Divide el texto usando el patrón como separador.
    
    ```js
    "manzana, pera, banana".split(/, /);
    // ["manzana", "pera", "banana"]
    ```
    

---

## 🔹 ¿Para qué sirve un literal `RegExp`?

El literal regex es una herramienta para **definir patrones de búsqueda en cadenas de texto**. Con él podemos:

- Buscar si un texto contiene algo específico.
    
- Extraer fragmentos que coincidan con una regla.
    
- Validar si el texto sigue un formato concreto (ejemplo: emails, contraseñas).
    
- Reemplazar partes de un texto de forma controlada.
    
- Dividir cadenas en secciones complejas, más allá de separadores simples como comas o espacios.
    

---

## 🔹 Ejemplos prácticos

**1. Buscar una palabra**

```js
let regex = /gato/;
regex.test("Tengo un gato"); // true
```

**2. Validar números**

```js
let regex = /^[0-9]+$/;
regex.test("12345"); // true
regex.test("12a45"); // false
```

**3. Reemplazar texto**

```js
"Mi número es 123-456".replace(/\d/g, "X");
// "Mi número es XXX-XXX"
```

**4. Extraer correos electrónicos**

```js
let texto = "Correos: uno@mail.com, dos@mail.com";
texto.match(/\w+@\w+\.\w+/g);
// ["uno@mail.com", "dos@mail.com"]
```

---

📌 En conclusión: el **literal `RegExp`** es una forma compacta de crear un objeto de la clase `RegExp`. A diferencia de otros literales de JavaScript que generan valores primitivos, este crea un objeto complejo con métodos muy potentes para trabajar con texto. La diferencia con el constructor está en el **momento y la flexibilidad de creación**, pero ambos producen exactamente el mismo tipo de objeto.
