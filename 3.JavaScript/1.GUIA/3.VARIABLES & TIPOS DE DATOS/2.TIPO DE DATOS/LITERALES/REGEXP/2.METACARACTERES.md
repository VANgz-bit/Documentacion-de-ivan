# 📖 Metacaracteres en Expresiones Regulares (RegExp)

---

## 📌 Introducción

En JavaScript, las **expresiones regulares (RegExp)** permiten buscar y manipular texto mediante **patrones**.  
La clave de su potencia radica en los **metacaracteres**, símbolos especiales que representan posiciones, repeticiones, tipos de caracteres o grupos.  
A diferencia de los caracteres normales (que se buscan literalmente), los metacaracteres permiten **describir reglas abstractas** para localizar coincidencias.

---

## 🔹 1. Metacaracteres de posición

Estos no representan un carácter, sino un **lugar** en la cadena:

- `^` → Inicio de la cadena.
    
- `$` → Final de la cadena.
    

**Ejemplo:**

```js
/^Hola/.test("Hola mundo"); // true
/^Hola/.test("Mundo Hola"); // false
```

---

## 🔹 2. Metacaracteres de tipo de carácter

Sirven para representar **categorías de caracteres**:

- `\d` → Un dígito (`0–9`).
    
- `\w` → Un carácter alfanumérico o guion bajo.
    
- `\s` → Un espacio en blanco.
    
- `.` → Cualquier carácter, excepto salto de línea.
    

Sus versiones negadas son:

- `\D` → No es un dígito.
    
- `\W` → No es alfanumérico.
    
- `\S` → No es un espacio en blanco.
    

**Ejemplo:**

```js
/\d\d/.test("23");   // true
/\d\d/.test("a2");   // false
```

---

## 🔹 3. Metacaracteres de repetición

Definen **cuántas veces** debe aparecer un patrón:

- `*` → 0 o más veces.
    
- `+` → 1 o más veces.
    
- `?` → 0 o 1 vez (opcional).
    
- `{n}` → Exactamente `n` repeticiones.
    
- `{n,}` → Al menos `n` repeticiones.
    
- `{n,m}` → Entre `n` y `m` repeticiones.
    

**Ejemplo:**

```js
/a{2,4}/.test("aa");    // true
/a{2,4}/.test("aaaaa"); // false
```

---

## 🔹 4. Metacaracteres de agrupación y alternativas

- `( )` → Agrupan patrones en una sola unidad.
    
- `|` → Alternativa (funciona como un “o”).
    

**Ejemplo:**

```js
/(perro|gato)/.test("Tengo un perro"); // true
/(perro|gato)/.test("Tengo un gato");  // true
/(perro|gato)/.test("Tengo un pez");   // false
```

---

## 🔹 5. Escapado de caracteres especiales

Los siguientes símbolos tienen un **significado especial**:  
`. * + ? | ( ) { } [ ] ^ $ \`

Para buscarlos literalmente, se debe anteponer `\`.

**Ejemplo:**

```js
/\./.test("Hola."); // true
/\./.test("Hola");  // false
```

---

## 🔹 6. Casos de uso prácticos de expresiones regulares

Para entender mejor cómo y **cuándo aplicar los metacaracteres**, veamos ejemplos comunes en validaciones y búsquedas:

---

### 📌 1. Validar una dirección de email básica

```js
let regex = /^[\w.-]+@[a-zA-Z\d.-]+\.[a-zA-Z]{2,}$/;

console.log(regex.test("usuario@gmail.com")); // true
console.log(regex.test("usuario.gmail.com")); // false
```

- `^` y `$` → Aseguran que coincida toda la cadena.
    
- `[\w.-]+` → Nombre del usuario (letras, números, guion y punto).
    
- `@` → Arroba obligatoria.
    
- `[a-zA-Z\d.-]+` → Dominio (ej: gmail, outlook).
    
- `\.[a-zA-Z]{2,}` → Extensión (.com, .org, .ar, etc.).
    

---

### 📌 2. Validar un número de teléfono (formato internacional simplificado)

```js
let regex = /^\+?\d{1,3}?[-.\s]?\d{7,10}$/;

console.log(regex.test("+54 3511234567")); // true
console.log(regex.test("351-123-4567"));   // true
console.log(regex.test("123"));            // false
```

- `\+?` → El `+` es opcional.
    
- `\d{1,3}?` → Código de país (1 a 3 dígitos).
    
- `[-.\s]?` → Separador opcional (- . o espacio).
    
- `\d{7,10}` → El número principal (7 a 10 dígitos).
    

---

### 📌 3. Validar una contraseña segura

```js
let regex = /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[\W_]).{8,}$/;

console.log(regex.test("Abc123!@"));  // true
console.log(regex.test("abcdef"));    // false
```

- `(?=.*[a-z])` → Al menos una minúscula.
    
- `(?=.*[A-Z])` → Al menos una mayúscula.
    
- `(?=.*\d)` → Al menos un número.
    
- `(?=.*[\W_])` → Al menos un carácter especial.
    
- `.{8,}` → Mínimo 8 caracteres.
    

---

### 📌 4. Validar una fecha en formato `dd/mm/yyyy`

```js
let regex = /^(0?[1-9]|[12][0-9]|3[01])\/(0?[1-9]|1[0-2])\/\d{4}$/;

console.log(regex.test("01/12/2025")); // true
console.log(regex.test("31/02/2025")); // true (ojo: no valida calendario real)
console.log(regex.test("2025/12/01")); // false
```

- `(0?[1-9]|[12][0-9]|3[01])` → Días válidos (01–31).
    
- `(0?[1-9]|1[0-2])` → Meses válidos (01–12).
    
- `\d{4}` → Año de 4 cifras.
    

> ⚠️ Nota: valida el **formato** pero no la lógica (ej: “31/02/2025” aún pasará). Para validar calendario real se necesita código adicional.

---

### 📌 5. Buscar todas las palabras en un texto

```js
let texto = "Hola mundo, esto es JavaScript.";
let regex = /\b\w+\b/g;

console.log(texto.match(regex));
// ["Hola", "mundo", "esto", "es", "JavaScript"]
```

- `\b` → Límite de palabra.
    
- `\w+` → Una o más letras/números.
    
- `g` → Busca todas las coincidencias.
    

---

### 📌 6. Extraer números de un texto

```js
let texto = "El precio es 1500 pesos y el descuento es 200.";
let regex = /\d+/g;

console.log(texto.match(regex)); // ["1500", "200"]
```

- `\d+` → Uno o más dígitos.
    
- `g` → Encuentra todas las apariciones.
    

---

## 🎯 Conclusión extendida

Los **metacaracteres** permiten construir desde patrones simples hasta validaciones muy complejas.  
En la práctica, las expresiones regulares se usan en:

- Validación de **formularios** (emails, contraseñas, teléfonos).
    
- Procesamiento de **texto y logs**.
    
- Búsqueda y **extracción de datos específicos**.
    
- Automatización de reemplazos en cadenas.
    

Saber **qué metacaracter usar en cada situación** es lo que da la verdadera fuerza a las RegExp.
