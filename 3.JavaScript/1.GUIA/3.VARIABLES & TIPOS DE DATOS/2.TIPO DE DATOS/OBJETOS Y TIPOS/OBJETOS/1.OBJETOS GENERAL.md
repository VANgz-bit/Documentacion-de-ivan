# 📘 `Object` en JavaScript (General)

## Definición

En JavaScript, un **objeto** es una **estructura dinámica de pares clave-valor**, donde:

- Las **claves** siempre son `String` o `Symbol`.
    
- Los **valores** pueden ser de cualquier tipo (primitivos u otros objetos).
    

Los objetos son fundamentales porque:

1. Son la base de estructuras complejas como `Array`, `Function`, `Date`, `RegExp`, etc.
    
2. Permiten organizar datos y comportamientos en un mismo contenedor.
    
3. Implementan el modelo de **herencia prototípica**: cada objeto tiene un enlace interno a un prototipo (`[[Prototype]]`) del cual hereda propiedades y métodos.
    

👉 Todo en JavaScript que no sea un tipo primitivo es un objeto o está basado en `Object`.

---

## Sintaxis

Un objeto general puede crearse de varias formas:

### 1. Constructor directo

```js
let obj = new Object();
```

- `new` → indica que se está creando una nueva instancia.
    
- `Object` → es la función constructora base para todos los objetos.
    
- `()` → ejecuta la construcción.
    

### 2. Usando métodos de clase estáticos

```js
let obj = Object.create(proto);
```

- `Object.create()` → permite especificar manualmente el prototipo (`proto`) del nuevo objeto.
    
- Si se pasa `null`, se crea un objeto sin herencia (ideal como diccionario puro).
    

👉 Señalador: **constructor (`new Object`)** crea un objeto vacío con `Object.prototype` como prototipo.  
👉 Señalador: **`Object.create`** te da más control, permitiendo definir un prototipo distinto o `null`.

---

## Características

1. **Dinámicos**: se pueden añadir, modificar o eliminar propiedades en cualquier momento.
    
2. **Encadenamiento prototípico**: si una propiedad no existe en el objeto, se busca en su prototipo, y así sucesivamente hasta `Object.prototype`.
    
3. **Descriptores de propiedades**: cada propiedad tiene atributos internos como `value`, `writable`, `enumerable`, `configurable` o (`get`/`set` en propiedades de acceso).
    
4. **Mutabilidad por referencia**: los objetos se asignan y pasan por referencia, no por valor.
    
5. **Comparación por identidad**: dos objetos distintos nunca son iguales aunque tengan el mismo contenido (`{} === {}` → `false`).
    
6. **Amplio ecosistema de métodos**: `Object.keys`, `Object.assign`, `Object.freeze`, entre otros, permiten manipularlos con precisión.

## Ejemplos

### Ejemplo correcto

```js
let persona = new Object();
persona.nombre = "Iván";
persona.edad = 21;

console.log(persona.nombre); // "Iván"
```

### Ejemplo con `Object.create`

```js
let animal = { hablar() { return "..." } };
let perro = Object.create(animal);
perro.nombre = "Firulais";

console.log(perro.hablar()); // "..."
```

### Ejemplo malo (mutar prototipo global)

```js
Object.prototype.saludar = function() { return "hola"; };
let obj = {};
console.log(obj.saludar()); // "hola"
// ⚠️ Malo: contamina todos los objetos y rompe librerías
```

### Ejemplo malo (comparación errónea)

```js
let a = {};
let b = {};
console.log(a === b); // false
// ⚠️ Aunque ambos estén vacíos, no son el mismo objeto
```

---

## Casos especiales o curiosos

1. **Objetos sin prototipo**
    

```js
let dict = Object.create(null);
dict["clave"] = "valor";
console.log(dict.toString); // undefined
```

→ Útil para diccionarios puros, ya que no heredan métodos de `Object.prototype`.

2. **Uso de `Object.is`**
    

```js
console.log(Object.is(NaN, NaN)); // true
console.log(NaN === NaN); // false
```

→ Diferencia con `===`: `Object.is` trata a `NaN` como igual a sí mismo y distingue `+0` de `-0`.

3. **Congelación de objetos**
    

```js
let config = { modo: "oscuro" };
Object.freeze(config);
config.modo = "claro"; // Ignorado en modo estricto → error
```

---

## Buenas prácticas

- **Preferí el literal `{}`** para objetos simples, reservando `new Object()` para casos específicos.
    
- **Usá `Object.create(null)`** cuando quieras un diccionario sin herencia.
    
- **Nunca modifiques `Object.prototype`**, ya que impacta en todo el código.
    
- **Congelá (`Object.freeze`) o sellá (`Object.seal`) objetos** que no deban alterarse.
    
- **Usá `Map`** si necesitás claves que no sean strings/símbolos o cuando el orden de inserción importa.
    

---
## Resumen

El objeto general (`Object`) es el **bloque de construcción principal en JavaScript**. Permite almacenar datos, definir comportamientos, heredar mediante prototipos y controlar el acceso a propiedades con descriptores. Es mutable, dinámico y comparado por referencia. Aunque podés crearlo con `new Object()` o `Object.create()`, en la práctica se recomienda `{}` para casos simples. Con métodos como `Object.keys`, `Object.assign`, `Object.freeze` y `Object.getOwnPropertyDescriptors`, es posible controlar cómo se manejan los objetos en memoria, cómo se clonan, cómo se protegen y cómo se heredan.
