# 📘 Métodos y propiedades Estáticos de `Object` en JavaScript

En JavaScript, el constructor global `Object` provee **métodos estáticos** que permiten trabajar con objetos de forma avanzada: inspeccionarlos, clonarlos, definir propiedades con precisión, congelarlos, etc.

---

## 🔹 `Object.keys()`

### Definición

Devuelve un **array con las claves enumerables propias** de un objeto (no incluye símbolos ni propiedades heredadas).

### Sintaxis

```js
Object.keys(obj)
```

### Ejemplo correcto

```js
let persona = { nombre: "Iván", edad: 21 };
console.log(Object.keys(persona)); // ["nombre", "edad"]
```

### Ejemplo incorrecto

```js
Object.keys("texto");
// []
// ⚠️ Un string primitivo no es un objeto, aunque internamente se convierte temporalmente.
```

### Casos especiales

- No devuelve propiedades heredadas:
    

```js
let padre = { apellido: "Gómez" };
let hijo = Object.create(padre);
hijo.nombre = "Iván";

console.log(Object.keys(hijo)); // ["nombre"]
```

### Buenas prácticas

Usar `Object.keys()` cuando se quiere **iterar solo propiedades propias y enumerables**. Para incluir símbolos → usar `Reflect.ownKeys`.

---

## 🔹 `Object.values()`

### Definición

Devuelve un array con los **valores de las propiedades enumerables propias** de un objeto.

### Sintaxis

```js
Object.values(obj)
```

### Ejemplo

```js
let persona = { nombre: "Iván", edad: 21 };
console.log(Object.values(persona)); // ["Iván", 21]
```

### Caso especial

Los valores se devuelven en **el mismo orden** que `Object.keys()`.

---

## 🔹 `Object.entries()`

### Definición

Devuelve un array de arrays, donde cada subarray tiene `[clave, valor]`.

### Ejemplo

```js
let persona = { nombre: "Iván", edad: 21 };
console.log(Object.entries(persona));
// [["nombre", "Iván"], ["edad", 21]]
```

### Buenas prácticas

Ideal para convertir objetos en estructuras iterables y poder usar `for...of`:

```js
for (let [k, v] of Object.entries(persona)) {
  console.log(`${k}: ${v}`);
}
```

---

## 🔹 `Object.fromEntries()`

### Definición

Hace la operación inversa de `Object.entries()`: convierte un array de pares `[clave, valor]` en un objeto.

### Ejemplo

```js
let entradas = [["nombre", "Iván"], ["edad", 21]];
let obj = Object.fromEntries(entradas);
console.log(obj); // { nombre: "Iván", edad: 21 }
```

### Caso de uso

Transformar un `Map` en un objeto:

```js
let map = new Map([["a", 1], ["b", 2]]);
console.log(Object.fromEntries(map)); // { a: 1, b: 2 }
```

---

## 🔹 `Object.assign()`

### Definición

Copia las propiedades **enumerables propias** de uno o más objetos origen hacia un objeto destino. Devuelve el objeto destino.

### Sintaxis

```js
Object.assign(destino, ...origenes)
```

### Ejemplo correcto

```js
let destino = { a: 1 };
let origen = { b: 2 };
Object.assign(destino, origen);

console.log(destino); // { a: 1, b: 2 }
```

### Ejemplo incorrecto

```js
let a = { x: 1 };
let b = { x: 2, y: 3 };
let resultado = Object.assign(a, b);

console.log(a); // { x: 2, y: 3 }
// ⚠️ Mutó al objeto `a` original.
```

### Buenas prácticas

Si se quiere clonar un objeto, mejor usar:

```js
let copia = Object.assign({}, original);
```

---

## 🔹 `Object.create()`

### Definición

Crea un nuevo objeto especificando su prototipo.

### Sintaxis

```js
Object.create(proto, propiedadesOpcionales)
```

### Ejemplo

```js
let animal = { tipo: "mamífero" };
let perro = Object.create(animal);
perro.raza = "Labrador";

console.log(perro.tipo); // "mamífero" (heredado)
```

### Caso especial

```js
let diccionario = Object.create(null);
console.log(diccionario.toString); // undefined
```

→ Se crea sin prototipo.

---

## 🔹 `Object.freeze()`

### Definición

Congela un objeto: no se pueden agregar, eliminar ni modificar propiedades.

### Ejemplo

```js
let config = { modo: "oscuro" };
Object.freeze(config);
config.modo = "claro"; // Ignorado o error en modo estricto
```

---

## 🔹 `Object.seal()`

### Definición

Sella un objeto: no se pueden agregar o eliminar propiedades, pero sí modificar los valores de las existentes.

```js
let obj = { a: 1 };
Object.seal(obj);
obj.a = 2; // ✅ permitido
delete obj.a; // ❌ prohibido
```

---

## 🔹 `Object.is()`

### Definición

Compara dos valores con semántica más precisa que `===`.

### Diferencias con `===`

```js
console.log(NaN === NaN); // false
console.log(Object.is(NaN, NaN)); // true

console.log(+0 === -0); // true
console.log(Object.is(+0, -0)); // false
```

---

## 🔹 `Object.getOwnPropertyDescriptor()` y `Object.getOwnPropertyDescriptors()`

### Definición

Permiten inspeccionar los **descriptores de propiedades**: atributos internos (`writable`, `enumerable`, `configurable`, `get`, `set`).

### Ejemplo

```js
let obj = { x: 1 };
let desc = Object.getOwnPropertyDescriptor(obj, "x");
console.log(desc);
// { value: 1, writable: true, enumerable: true, configurable: true }
```

---

## 🔹 Otros métodos útiles

- `Object.getPrototypeOf(obj)` → obtiene el prototipo de un objeto.
    
- `Object.setPrototypeOf(obj, proto)` → cambia el prototipo (⚠️ poco recomendable).
    
- `Object.hasOwn(obj, prop)` → (ES2022) verifica si una propiedad es propia.
    
- `Object.preventExtensions(obj)` → impide agregar nuevas propiedades.
    
- `Object.isFrozen(obj)` / `Object.isSealed(obj)` → verifican estado de un objeto.
    

---

## 📌 Resumen general

Los métodos estáticos de `Object` permiten:

- **Inspeccionar** (`keys`, `values`, `entries`, `getOwnPropertyDescriptors`).
    
- **Construir/transformar** (`fromEntries`, `create`, `assign`).
    
- **Controlar mutabilidad** (`freeze`, `seal`, `preventExtensions`).
    
- **Comparar valores** (`Object.is`).
    
- **Trabajar con prototipos** (`getPrototypeOf`, `setPrototypeOf`).
    

Gracias a estas herramientas, `Object` no es solo el "contenedor base", sino también un conjunto de funciones que nos permiten manejar la **herencia, la inmutabilidad, y la definición detallada de propiedades** en JavaScript.

---

 > **Importante**: los métodos de `Object` no distinguen si el objeto fue creado como literal, con `new Object()` o con `Object.create()`.  
> Lo único que cambia es la relación de prototipo y qué propiedades se consideran “propias” o “heredadas”.
