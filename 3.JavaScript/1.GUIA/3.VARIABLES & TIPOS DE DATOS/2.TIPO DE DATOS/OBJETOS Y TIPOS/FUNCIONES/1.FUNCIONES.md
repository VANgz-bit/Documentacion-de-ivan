
## 📌 ¿Qué es una función en JavaScript?

Una **función** en JavaScript es un **bloque de código reutilizable** que se puede ejecutar (invocar) cuando se necesite. Se utiliza para **organizar el código**, evitar la repetición, y **realizar tareas específicas**.

> Las funciones permiten dividir un programa en partes lógicas, facilitando su mantenimiento, legibilidad y reutilización.

---
## 📌 ¿Para qué sirve una función?

Las funciones sirven para:

- **Reutilizar código** sin repetirlo.
    
- Dividir problemas grandes en pequeños pasos lógicos.
    
- Encapsular lógica que puede cambiar.
    
- Mejorar la organización del código.
    
- Trabajar con **parámetros** y **valores devueltos** para hacerlo más dinámico.
    
- Controlar el **flujo de ejecución** del programa.
    

---

## 📌 Sintaxis básica de una función declarada

```javascript
function nombreDeLaFuncion(parámetro1, parámetro2, ...) {
  // Bloque de código que se ejecuta
}
```

### ✅ Ejemplo:

```javascript
function saludar() {
  console.log("Hola, bienvenido!");
}
```

---

## 📌 ¿Cómo se **llama** (invoca) una función?

Para ejecutar una función, simplemente escribimos su nombre seguido de paréntesis:

```javascript
saludar(); // Llama a la función saludar
```

---

## 📌 Funciones con parámetros

Un **parámetro** es una variable que se define dentro del paréntesis de la función y sirve para **recibir valores externos** al momento de ejecutar la función.

### Ejemplo:

```javascript
function saludar(nombre) {
  console.log("Hola, " + nombre);
}

saludar("Iván"); // Imprime: Hola, Iván
```

---

## 📌 ¿Qué es `return`?

La palabra clave `return` **devuelve un valor** desde una función hacia quien la llamó.

### Características de `return`:

- **Devuelve datos**: permite que la función entregue un resultado.
    
- **Finaliza la función**: cuando se ejecuta `return`, se termina la ejecución de la función inmediatamente.
    
- **Opcional**: si no se escribe `return`, la función devuelve `undefined` por defecto.
    

### Ejemplo:

```javascript
function sumar(a, b) {
  return a + b;
}

let resultado = sumar(5, 3);
console.log(resultado); // Imprime: 8
```

---

## 📌 ¿Qué son los parámetros?

Los **parámetros** son las variables que se colocan entre los paréntesis cuando se define una función. Permiten **pasar información a la función**.

Cuando llamás a la función, los **valores reales** que pasás se llaman **argumentos**.

### Ejemplo:

```javascript
function multiplicar(x, y) {
  return x * y;
}

multiplicar(4, 2); // x=4, y=2
```

---

## 📌 ¿Los parámetros están relacionados con el **scope**?

Sí. El **scope** (alcance) determina **dónde son accesibles las variables**.  
Los parámetros de una función tienen **scope local**, es decir, solo existen dentro de la función.

```javascript
function ejemplo(num) {
  console.log(num); // num existe aquí
}

console.log(num); // Error: num no está definido fuera de la función
```

## 📌 Formas de crear funciones en JavaScript

Hay **varias formas** de crear funciones:

### 1. **Declaración de función (function declaration)**

```javascript
function saludar() {
  console.log("Hola");
}
```

✅ Se puede llamar **antes** de ser declarada gracias al **hoisting**.

---

### 2. **Expresión de función (function expression)**

Una función puede **asignarse a una variable**. En este caso no tiene nombre, es **anónima**.

```javascript
const saludar = function() {
  console.log("Hola");
};
```

❌ No se puede llamar antes de su creación porque **no se eleva** (no hay hoisting).

---

### 3. **Funciones flecha (arrow functions)**

Es una **forma más corta** de escribir funciones anónimas.  
No tienen su propio `this` (tema avanzado relacionado con objetos y clases).

```javascript
const saludar = () => {
  console.log("Hola");
};
```

#### Con un solo parámetro y una sola línea:

```javascript
const saludar = nombre => console.log("Hola " + nombre);
```

#### Con `return` implícito:

```javascript
const sumar = (a, b) => a + b;
console.log(sumar(3, 4)); // 7
```

> Si usás llaves `{}`, tenés que escribir `return` explícitamente:

```javascript
const restar = (a, b) => {
  return a - b;
};
```

> NOTA: esta sección de funciones flechas esta mejor detallado mas adelante.
---

## 📌 Diferencias clave entre formas de crear funciones

|Tipo de función|¿Tiene nombre?|¿Hoisting?|Sintaxis corta|Tiene `this` propio|
|---|---|---|---|---|
|Declaración (`function`)|Sí|✅ Sí|❌ No|✅ Sí|
|Expresión (`const = func`)|Opcional|❌ No|❌ No|✅ Sí|
|Flecha (`=>`)|Opcional|❌ No|✅ Sí|❌ No (hereda)|

---

## ✅ Resumen general

- Las **funciones** son bloques de código reutilizables.
    
- Se pueden **declarar**, **asignar a variables**, o escribir como **funciones flecha**.
    
- Se llaman escribiendo su nombre seguido de paréntesis.
    
- Pueden recibir **parámetros** y usar `return` para **devolver un valor**.
    
- Los parámetros tienen **scope local**, y las funciones flecha tienen una sintaxis más concisa.
