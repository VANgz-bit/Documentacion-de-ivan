# 📌 `arguments` en JavaScript

---

## 🔹 Definición

`arguments` es un **objeto especial disponible dentro de las funciones normales** (definidas con `function`).  
Su función es **guardar todos los valores que se pasaron a la función cuando se la llama**, incluso si esos valores **no corresponden a un parámetro declarado**.

> Es importante aclarar que **`arguments` no es un “argumento” individual**, sino un **conjunto de todos los argumentos reales** que se pasaron a la función.

---

## 🔹 Diferencia entre parámetro, argumento y `arguments`

|Concepto|Qué es|Ejemplo|
|---|---|---|
|Parámetro|Variable declarada en la función|`function f(a, b)` → `a` y `b` son parámetros|
|Argumento|Valor real que se pasa al llamar a la función|`f(10, 20, 30)` → `10, 20, 30` son argumentos|
|`arguments`|Objeto que contiene **todos los argumentos pasados**|Dentro de `f`, `arguments` = `[10, 20, 30]`|

---

## 🔹 Ejemplo práctico

```js
function ejemplo(a, b) {
  console.log("a:", a);           // 10
  console.log("b:", b);           // 20
  console.log("arguments:", arguments); // [10, 20, 30, 40]
}

ejemplo(10, 20, 30, 40);
```

📌 Explicación:

- Los **parámetros** declarados son `a` y `b`.
    
- Los **argumentos pasados** fueron `10, 20, 30, 40`.
    
- `arguments` captura **todos los valores pasados**, incluso los que no tienen parámetro correspondiente.
    

---

## 🔹 Características importantes

1. **No es un array real**, aunque se vea similar:
    
    ```js
    console.log(arguments.length); // Número de argumentos
    console.log(arguments[0]);     // Primer argumento
    // No se puede usar .map() directamente
    ```
    
2. **Solo está disponible en funciones normales**, no en **funciones flecha**:
    
    ```js
    const flecha = () => console.log(arguments);
    flecha(1, 2, 3); // ❌ ReferenceError
    ```
    
3. **Alternativa moderna:** operador **rest (`...`)**  
    Funciona en funciones normales y flechas, devuelve un **array real**:
    
    ```js
    const mostrar = (...args) => {
      console.log(args);
    };
    mostrar(1, 2, 3); // [1, 2, 3]
    ```
    

---

## 🔹 Resumen de dudas aclaradas

- **¿`arguments` es un argumento?**  
    ❌ No, es un **objeto que guarda todos los argumentos pasados**.
    
- **¿Por qué aparece más valores que los parámetros declarados?**  
    Porque `arguments` captura **todos los argumentos pasados**, no solo los que tienen parámetro.
    
- **¿Funciona en funciones flecha?**  
    ❌ No, las flechas no crean su propio `arguments`; se recomienda usar **`...rest`** en su lugar.
    
- **¿Para qué sirve?**  
    Para poder manejar funciones que reciben un **número variable de argumentos** sin tener que declarar todos los parámetros.
