# **Uso de caracteres especiales en cadenas**

## **Definición**

Los **caracteres especiales** en JavaScript son símbolos que no se pueden escribir directamente en una cadena de texto y que se representan mediante **secuencias de escape**. Estas secuencias comienzan con la barra invertida `\` y permiten insertar comillas, saltos de línea, tabulaciones, caracteres Unicode y otros símbolos especiales dentro de un string.

---

## **Conceptos clave**

1. **Secuencia de escape** → Una combinación de `\` seguida de un carácter especial que tiene un significado distinto dentro de un string.
    
2. **Inmutabilidad del string** → Los caracteres especiales no modifican el string original, sino que forman parte de la representación del nuevo string generado.
    
3. **Compatibilidad con delimitadores** → Permiten incluir comillas simples o dobles dentro de un string sin romper la sintaxis.
    

---

## **Tabla de secuencias de escape más comunes**

|Secuencia|Significado|
|---|---|
|`\'`|Comilla simple dentro de un string delimitado por `'`|
|`\"`|Comilla doble dentro de un string delimitado por `"`|
|`\\`|Barra invertida `\`|
|`\n`|Nueva línea|
|`\r`|Retorno de carro|
|`\t`|Tabulación horizontal|
|`\b`|Retroceso (backspace)|
|`\f`|Form feed|
|`\v`|Tabulación vertical|
|`\uXXXX`|Carácter Unicode especificando el código hexadecimal|

---

## **Ejemplos señalados**

```javascript
// Nueva línea y tabulación
let saludo = "Hola,\n\tSoy Iván";
console.log(saludo);

/* Salida:
Hola,
    Soy Iván
*/

// Comillas dentro de la cadena
let frase = 'Ella dijo: "Hola mundo"';
console.log(frase);

// Barra invertida
let path = "C:\\Users\\Iván\\Documents";
console.log(path);

// Carácter Unicode
let corazon = "\u2764";
console.log(corazon); // ❤
```

---

## **Casos y notas importantes**

- Permiten **incluir caracteres que de otra forma romperían la cadena**, como comillas o barras invertidas.
    
- Facilitan **dar formato al texto**, como saltos de línea o tabulaciones, especialmente en la consola o al generar contenido dinámico.
    
- Son compatibles con **strings delimitados por `'`, `"` y template literals**, aunque en estos últimos los saltos de línea se pueden escribir directamente sin `\n`.
