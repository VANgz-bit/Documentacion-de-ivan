# **Concatenación de Strings en JavaScript**

## **Definición**
La **concatenación** es el **proceso de unir dos o más cadenas de texto (strings) en una sola**.  
En JavaScript, esto se realiza principalmente mediante el **operador `+`**, que actúa como **concatenador** cuando al menos uno de los valores es un string.

> Importante: La concatenación **no altera los strings originales**, sino que crea **un nuevo string** con el resultado de la unión.

---

## **Conceptos clave**

1. **Operador `+` como concatenador**
    
    - Si uno de los operandos es un string, el operador `+` **convierte automáticamente** cualquier otro tipo (número, booleano, etc.) a string y realiza la unión.
        
    - Esto se conoce como **coerción de tipo automática**.
        
2. **Concatenación de múltiples strings**
    
    - Se pueden unir varios strings usando múltiples operadores `+`.
        
    - Cada unión se realiza de izquierda a derecha.
        
3. **Coerción de tipos**
    
    - Si concatenamos un string con un número o un valor booleano, **JavaScript convierte el valor no string en texto** antes de unirlo.
        
4. **Alternativa moderna: Template literals**
    
    - Con **backticks `` ` ``** y `${}` podemos insertar variables o expresiones directamente dentro de la cadena, logrando una concatenación más clara y legible.
        

---

## **Ejemplos señalados**

### 1️⃣ Concatenación básica

```javascript
let nombre = "Iván";
let saludo = "Hola " + nombre;

console.log(saludo); // "Hola Iván"
```

### 2️⃣ Concatenación con números (coerción automática)

```javascript
let edad = 21;
let mensaje = "Tengo " + edad + " años";

console.log(mensaje); // "Tengo 21 años"
```

>
  La coerción de números a string tambien es mencionada en variables y coerción de tipos.
>

### 3️⃣ Concatenación de múltiples strings

```javascript
let a = "Hola";
let b = "¿cómo estás?";
let mensaje = a + ", " + b;

console.log(mensaje); // "Hola, ¿cómo estás?"
```

---
## **Template Literals (Concatenación moderna)**

### **Definición**
 Los **template literals** son **cadenas de texto delimitadas por backticks (`` ` ``)** que permiten **insertar variables y expresiones directamente dentro del string** usando la sintaxis `${}`. 
A diferencia de la concatenación con `+`, los template literals **son más legibles**, permiten **múltiples líneas** sin necesidad de `\n` y facilitan la construcción de strings dinámicos complejos.

---

### **Conceptos clave**

1. **Delimitadores**
    
    - Se usan **backticks `` ` ``** en lugar de comillas simples `' '` o dobles `" "`.
        
    - Permiten incluir comillas dentro del string sin usar secuencias de escape.
        
2. **Interpolación de variables**
    
    - La sintaxis `${variable}` permite **insertar el valor de una variable o expresión directamente** en el string.
        
    - La expresión dentro de `${}` se evalúa y se convierte automáticamente en texto.
        
3. **Soporte de múltiples líneas**
    
    - Se pueden escribir strings en **más de una línea** sin usar `\n` ni concatenaciones adicionales.
        
4. **Concatenación implícita**
    
    - Al usar `${}` se evita el uso del operador `+`; todo se combina automáticamente en un único string.
        

---

### **Ejemplo señalado**

```javascript
let nombre = "Iván";
let edad = 21;

// Interpolación de variables
let mensaje = `Hola, soy ${nombre} y tengo ${edad} años.`;
console.log(mensaje); // "Hola, soy Iván y tengo 21 años."

// Múltiples líneas
let multilinea = `Esta es
una cadena
en varias líneas.`;
console.log(multilinea);
/* Salida:
Esta es
una cadena
en varias líneas.
*/
```

---

### **Casos y notas importantes**

- Facilitan **leer y mantener** strings complejos con variables y saltos de línea.
    
- Permiten **expresiones dentro de `${}`**, no solo variables simples:
    
    ```javascript
    let a = 5, b = 10;
    console.log(`La suma de a + b es ${a + b}`); // "La suma de a + b es 15"
    ```
    
- Son recomendables **sobre el uso de `+`** cuando se concatena más de una variable o se necesitan múltiples líneas.
    
- Los template literals **no modifican los valores originales**, solo generan un string nuevo con el resultado de la interpolación.
    

---

## **Casos y notas importantes**

- La concatenación **siempre genera un nuevo string**; los originales permanecen inalterados.
    
- Evitar concatenar tipos distintos sin ser consciente de la coerción, para prevenir resultados inesperados.
    
- Para concatenaciones complejas, con múltiples variables o saltos de línea, **template literals** son más legibles y recomendables.
    
- La concatenación es **un concepto directamente relacionado con strings**, ya que su objetivo principal es unir texto; otros tipos de datos solo se convierten en texto automáticamente cuando se combinan con un string.
    

---

## **Resumen final**

|Concepto|Definición|
|---|---|
|**Concatenación**|Proceso de unir dos o más strings en uno solo.|
|**Operador `+`**|Concatena strings; convierte automáticamente otros tipos en texto si es necesario.|
|**Coerción de tipos**|JavaScript convierte valores no string en texto al concatenar con strings.|
|**Template literals**|Alternativa moderna para concatenar con backticks y `${}`; más legible y flexible.|
