# 📘 String en JavaScript

---
## 1. Definición

Un **`String`** en JavaScript es un **tipo de dato primitivo** que representa una **secuencia de caracteres de texto**.  
Estos caracteres pueden incluir letras, números, símbolos, espacios o cualquier carácter Unicode.

- En JavaScript, un `String` está delimitado por **comillas**:
    
    - **Simples** → `'Hola'`
        
    - **Dobles** → `"Hola"`
        
    - **Backticks (acento grave)** → `` `Hola` `` (template literals)
        

👉 Aunque `String` es un tipo primitivo, JavaScript lo **trata como objeto temporalmente** cuando se usan propiedades o métodos (`.length`, `.toUpperCase()`, etc.). Esto sucede gracias a lo que se llama **autoboxing**: el motor envuelve el valor primitivo en un objeto `String` de forma interna.

📌 **Contexto histórico:**

- `String` existe desde las primeras versiones de ECMAScript.
    
- Los **template literals** con backticks `` ` `` fueron introducidos en **ES6 (2015)** y expandieron muchísimo sus usos.
    

---

## 2. Sintaxis

Formas de crear un `String`:

```js
// ✔️ Forma literal (recomendada)
let texto1 = "Hola";   // comillas dobles
let texto2 = 'Mundo';  // comillas simples
let texto3 = `Hola Mundo`; // template literal

// ✔️ Usando el constructor String (NO recomendado)
let texto4 = new String("Hola");
```

🔎 **Señaladores**

- `"Hola"` → literal de tipo string.
    
- `new String("Hola")` → objeto String (envuelve el valor primitivo).
    

⚠️ Diferencia clave:

```js
typeof "Hola";           // "string"
typeof new String("Hola"); // "object"
```

👉 Siempre usar **literales**, no el constructor.

---

## 3. Características

- **Primitivo e inmutable:**  
    Un `String` **no se puede modificar**, aunque lo parezca.  
    Ejemplo:
    
    ```js
    let palabra = "Hola";
    palabra[0] = "h"; // ❌ no cambia nada
    console.log(palabra); // "Hola"
    ```
    
- **Acceso por índice:**  
    Se puede acceder a los caracteres como si fuera un array:
    
    ```js
    let palabra = "Hola";
    console.log(palabra[0]); // "H"
    console.log(palabra[3]); // "a"
    ```
    
- **Longitud con `.length`:**
    
    ```js
    let palabra = "Hola";
    console.log(palabra.length); // 4
    ```
    
- **Soporta caracteres Unicode:**  
    Incluyendo emojis y caracteres internacionales.
    
    ```js
    let saludo = "👋🏼 Hola 世界";
    console.log(saludo);
    ```
    
- **Métodos integrados:**
    
    - Manipulación → `.toUpperCase()`, `.toLowerCase()`, `.trim()`.
        
    - Búsqueda → `.includes()`, `.indexOf()`, `.startsWith()`, `.endsWith()`.
        
    - Extracción → `.slice()`, `.substring()`, `.substr()` (obsoleta).
        
    - Reemplazo → `.replace()`, `.replaceAll()`.
        
    - División → `.split()`.
        

---

## 4. Ejemplos buenos y malos

### ✔️ Buenos

```js
let nombre = "Iván";     // comillas simples o dobles
let saludo = `Hola ${nombre}`; // template literal con interpolación
console.log(saludo); // "Hola Iván"
```

### ❌ Malos

```js
let texto = new String("Hola"); 
// crea un objeto, no un primitivo → confunde en comparaciones

"Hola".lenght; 
// ❌ Error de escritura: debe ser .length
```

---

## 5. Casos especiales o curiosos

- **Comparación por valor y por referencia:**
    

```js
"hola" === "hola";               // true → primitivos comparados por valor
new String("hola") === new String("hola"); // false → objetos distintos
```

- **Concatenación automática con `+`:**
    

```js
"Hola" + " " + "Mundo"; // "Hola Mundo"
```

- **Conversión implícita:**
    

```js
1 + "2"; // "12" → número convertido a string
```

- **Caracteres Unicode especiales:**
    

```js
console.log("🐱".length); // 2 → algunos emojis ocupan 2 unidades
```

---

## 6. Buenas prácticas

- Usar **literales** (`"texto"`) en lugar de `new String()`.
    
- Preferir **template literals** con backticks para concatenar/interpolar.
    
- Usar métodos modernos como `.replaceAll()` en vez de hacks con regex.
    
- Cuidar el uso de `.length` con emojis o caracteres multibyte.
    
- Evitar abusar de concatenaciones largas → usar plantillas.
    

---

## 7. Resumen

- `String` → tipo primitivo que representa texto.
    
- Inmutable, pero con muchos métodos de manipulación.
    
- Puede declararse con `'`, `"`, o `` ` `` (backticks → más potentes).
    
- Comparar siempre con `===`.
    
- Evitar `new String()` porque crea un objeto, no un primitivo.
    
- Buen aliado en manipulación de texto, plantillas y concatenaciones.
    

---

📌 **Relaciones con otros temas:**

- Se conecta con **operadores aritméticos** (`+` para concatenación).
    
- Se relaciona con **literales** (String Literal, Template Literal).
    
- Está muy ligado a **coerción de tipos** (cuando números se convierten en strings).
