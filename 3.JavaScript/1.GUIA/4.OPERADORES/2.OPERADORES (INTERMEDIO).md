
##  OPERADORES DE COMPARACIÓN

###  ¿Qué son?

Los **operadores de comparación** se usan para **comparar dos valores** y devolver un resultado **booleano**:

- `true` si la comparación es verdadera.
    
- `false` si es falsa.
    

Son fundamentales en estructuras como `if`, `while`, `for`, etc., ya que determinan qué camino tomar.


###  Lista de Operadores de Comparación

| Operador | Nombre               | Descripción                                                                   |
| -------- | -------------------- | ----------------------------------------------------------------------------- |
| `==`     | Igualdad débil       | Compara dos valores **sin tener en cuenta el tipo**.                          |
| `===`    | Igualdad estricta    | Compara dos valores **y también su tipo**.                                    |
| `!=`     | Desigualdad débil    | Verifica que dos valores **no sean iguales** (sin importar el tipo).          |
| `!==`    | Desigualdad estricta | Verifica que dos valores **no sean iguales o que no tengan el mismo tipo**.   |
| `>`      | Mayor que            | Retorna `true` si el valor de la izquierda es **mayor** que el de la derecha. |
| `<`      | Menor que            | Retorna `true` si el valor de la izquierda es **menor**.                      |
| `>=`     | Mayor o igual que    | `true` si el valor de la izquierda es **mayor o igual**.                      |
| `<=`     | Menor o igual que    | `true` si el valor de la izquierda es **menor o igual**.                      |

---

###  Características de los Operadores de Comparación

1. **Retornan booleanos** (`true` o `false`).
    
2. Se usan principalmente en estructuras de control (`if`, `while`, etc.).
    
3. **Diferencia clave entre `==` y `===`**:
    
    - `==` hace una conversión implícita de tipo.
        
    - `===` **no hace conversión**, compara valor **y tipo**.
        
4. Se pueden usar con **números, cadenas, booleanos, null, undefined, etc.**
    
5. JavaScript tiene algunas comparaciones especiales:
    
    - `null == undefined` → `true`
        
    - `null === undefined` → `false`
        

---

###  Ejemplos de Comparación

```js
5 == '5'       // true  -> compara valor, ignora tipo
5 === '5'      // false -> diferente tipo (number vs string)
10 > 3         // true
4 <= 4         // true
'3' != 3       // false -> valor igual, ignora tipo
'3' !== 3      // true  -> tipo distinto
```

##  OPERADORES LÓGICOS

###  ¿Qué son?

Los **operadores lógicos** permiten **combinar múltiples condiciones** y devolver un valor booleano. Se utilizan cuando queremos que una condición dependa de **más de una comparación**.

---

###  Lista de Operadores Lógicos

|Operador|Nombre|Descripción|
|---|---|---|
|`&&`|**AND (Y)**|Retorna `true` **si ambas condiciones son verdaderas**.|
|`||`|
|`!`|**NOT (NO)**|Invierte el valor: `!true` → `false`, `!false` → `true`.|


### `&&` (AND / Y)

- Solo retorna `true` si **ambas condiciones** son verdaderas.
    
- Si encuentra una falsa, **detiene la evaluación** (cortocircuito).
    
- Se usa para requerir que **todo se cumpla**.
    

```js
true && true        // true
true && false       // false
false && true       // false
```

**Ejemplo real:**

```js
let edad = 22;
let tieneDNI = true;

if (edad >= 18 && tieneDNI) {
  console.log("Puede entrar");
} else {
  console.log("No puede entrar");
}
```

---

### `||` (OR / O)

- Retorna `true` si **al menos una** condición es verdadera.
    
- Si la primera condición es `true`, **ya no evalúa la segunda**.
    
- Se usa para permitir **opciones múltiples**.
    

```js
true || false       // true
false || false      // false
true || true        // true
```

**Ejemplo real:**

```js
let tieneEntrada = false;
let esInvitado = true;

if (tieneEntrada || esInvitado) {
  console.log("Puede ingresar");
}
```

---

### `!` (NOT / NO)

- **Niega** una condición.
    
- Convierte `true` en `false` y viceversa.
    

```js
!true     // false
!false    // true
```

**Ejemplo real:**

```js
let accesoDenegado = false;

if (!accesoDenegado) {
  console.log("Acceso permitido");
}
```

---

##  IMPORTANTE: "CORTOCIRCUITO" EN JAVASCRIPT

- `&&`: si el primer valor es falso, no evalúa el segundo.
    
- `||`: si el primer valor es verdadero, no evalúa el segundo.
    

Esto puede usarse para **asignar valores por defecto**:

```js
let nombre = input || "Invitado";
```

Y para proteger código:

```js
usuario && usuario.saludar();  // Solo llama a saludar si usuario existe
```

---

###  Ejemplo en contexto

```js
let edad = 20;
let tieneDocumento = true;

if (edad >= 18 && tieneDocumento) {
  console.log("Puede ingresar");
} else {
  console.log("No puede ingresar");
}
```

---

###  IMPORTANTE: Retorno no booleano

JavaScript permite que `&&` y `||` retornen el **último valor evaluado**, si no estás en un contexto estricto de condición:

```js
console.log("Hola" && "mundo") // mundo 
console.log(0 || "Valor por defecto");  // "Valor por defecto" (porque 0 es falso)

```

Esto se usa mucho para asignar valores predeterminados:

```js
let nombre = input || "Invitado";
```

---

##  DIFERENCIAS ENTRE COMPARACIÓN Y LÓGICA

| Comparación                               | Lógica                                               |
| ----------------------------------------- | ---------------------------------------------------- |
| Compara valores (números, strings…)       | Compara resultados de expresiones booleanas          |
| Retorna `true` o `false`                  | Combina condiciones booleanas                        |
| Usa `==`, `===`, `>`, `<=`, etc.          | Usa `&&`, `                                          |
| Se usa para evaluar igualdad o diferencia | Se usa para decidir si varias condiciones se cumplen |
