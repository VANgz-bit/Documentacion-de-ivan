# **Sentencias de control: `break` y `continue` en JavaScript**

## **1. Introducción**

Las sentencias **`break`** y **`continue`** son herramientas de control de flujo en JavaScript que permiten **modificar la ejecución de bucles y estructuras condicionales como `switch`**.

- **`break`**: detiene la ejecución de un bucle o de un `case` en un `switch`.
    
- **`continue`**: salta la iteración actual de un bucle y pasa a la siguiente.
    

Estas sentencias ayudan a **controlar el flujo de manera precisa**, evitando ejecuciones innecesarias o filtrando casos específicos.

---

## **2. Sentencia `break`**

### **Definición**

La sentencia `break` **detiene inmediatamente la ejecución** de un bucle (`for`, `while`, `do...while`, `for...in`, `for...of`) o de un `case` en un `switch`.

Cuando se ejecuta `break`:

1. La ejecución sale del bucle o del `switch` inmediatamente.
    
2. Ninguna iteración o `case` posterior se ejecuta (a menos que se use en bucles anidados con etiquetas).
    

---

### **2.1 Uso de `break` en bucles**

**Sintaxis:**

```js
break;
```

**Ejemplo con `for`:**

```js
for (let i = 1; i <= 5; i++) {
  if (i === 3) break;
  console.log(i);
}
```

**Salida:**

```
1
2
```

**Explicación:**

- La iteración se detiene cuando `i === 3`.
    
- No se ejecutan las iteraciones restantes (`3`, `4`, `5`).
    

**Ejemplo con `while`:**

```js
let numero = 1;
while (numero <= 10) {
  if (numero > 5) break;
  console.log(numero);
  numero++;
}
```

**Salida:**

```
1
2
3
4
5
```

**Explicación:**

- Aunque la condición permite hasta 10, `break` termina el bucle cuando `numero > 5`.
    

---

### **2.2 Uso de `break` en `switch`**

**Sintaxis:**

```js
switch (expresión) {
  case valor1:
    // Código
    break;
  case valor2:
    // Código
    break;
  default:
    // Código
}
```

**Ejemplo:**

```js
let dia = "martes";

switch (dia) {
  case "lunes":
    console.log("Primer día de la semana");
    break;
  case "martes":
    console.log("Segundo día de la semana");
    break;
  case "miércoles":
    console.log("Mitad de semana");
    break;
  default:
    console.log("Día no registrado");
}
```

**Salida:**

```
Segundo día de la semana
```

**Explicación:**

- Al coincidir `dia === "martes"`, se ejecuta el bloque correspondiente.
    
- `break` detiene la ejecución y evita que los demás `case` se ejecuten (previene _fall through_).
    

**Qué pasa si no usamos `break`:**

```js
let fruta = "naranja";

switch (fruta) {
  case "manzana":
    console.log("Elegiste manzana");
  case "naranja":
    console.log("Elegiste naranja");
  case "banana":
    console.log("Elegiste banana");
  default:
    console.log("Fruta no encontrada");
}
```

**Salida:**

```
Elegiste naranja
Elegiste banana
Fruta no encontrada
```

- Se ejecutan todos los casos posteriores al que coincide, debido a la ausencia de `break`.
    
- Esto se conoce como **fall through** y normalmente se evita usando `break`.
    
---
## **Fall Through en JavaScript**

### **Definición**

El término **_Fall through_** (en español, “caída” o “paso hacia abajo”) se refiere al **comportamiento de un `switch` cuando no se usa `break` al final de un `case`**.

En otras palabras:

- Cuando un `case` coincide con la expresión evaluada, **si no hay `break`**, JavaScript **continúa ejecutando los bloques de los `case` siguientes**, aunque sus valores no coincidan.
    
- Esto puede generar **ejecución no deseada de código** si no se controla correctamente.
---

## **3. Sentencia `continue`**

### **Definición**

La sentencia `continue` **omite la iteración actual** de un bucle y pasa a la siguiente, **sin interrumpir el bucle completo**.

- Se usa únicamente en bucles (`for`, `while`, `do...while`, `for...in`, `for...of`).
    
- Permite **filtrar elementos** sin anidar múltiples `if`.
    

---

### **Sintaxis**

```js
continue;
```

### **Ejemplo con `for`:**

```js
for (let i = 1; i <= 5; i++) {
  if (i === 3) continue;
  console.log(i);
}
```

**Salida:**

```
1
2
4
5
```

**Explicación:**

- La iteración donde `i === 3` se salta.
    
- El bucle continúa normalmente con las demás iteraciones.
    

### **Ejemplo con `while`:**

```js
let numero = 0;

while (numero < 5) {
  numero++;
  if (numero === 2) continue;
  console.log(numero);
}
```

**Salida:**

```
1
3
4
5
```

**Explicación:**

- Se omite el `console.log` cuando `numero === 2`.
    
- Las demás iteraciones siguen ejecutándose.
    

---

## **4. Comparación entre `break` y `continue`**

|Sentencia|Qué hace|Efecto en el bucle / switch|Usos típicos|
|---|---|---|---|
|**break**|Detiene por completo el bucle o el `case`|Termina la ejecución|Salir de bucle cuando se cumple una condición crítica o detener un `case` en `switch`|
|**continue**|Salta la iteración actual|Pasa a la siguiente iteración|Omitir casos específicos dentro de un bucle sin interrumpir el resto|

---

## **5. Buenas prácticas**

1. **Usar `break` para terminar bucles o `case`**: evita iteraciones innecesarias o fall through en `switch`.
    
2. **Usar `continue` para filtrar iteraciones**: mejora la legibilidad y evita anidaciones de `if`.
    
3. **No abusar de estas sentencias**: demasiados `break` y `continue` pueden dificultar la lectura del código.
    
4. **Combinar con `try...catch` si hay errores dentro de bucles**: para no interrumpir toda la ejecución en caso de excepciones.
