# **Control y Manejo de Errores en JavaScript**

## **¿Qué es un error en programación?**

En programación, un **error** es una condición inesperada que ocurre durante la ejecución de un programa, lo que puede interrumpir su flujo normal.  
En JavaScript, existen principalmente tres tipos de errores:

1. **Errores de sintaxis (SyntaxError):**  
    Ocurren cuando el código no cumple con las reglas del lenguaje. Ejemplo: olvidarse de un paréntesis o de una llave.
    
    ```js
    let x = ; // SyntaxError
    ```
    
2. **Errores de tiempo de ejecución (RuntimeError):**  
    Aparecen mientras se ejecuta el programa, por ejemplo, intentar acceder a una variable no definida o dividir por cero en ciertos contextos.
    
    ```js
    console.log(y); // ReferenceError: y no está definida
    ```
    
3. **Errores lógicos (LogicalError):**  
    El código se ejecuta sin detenerse, pero produce un resultado incorrecto porque la lógica es errónea.
    
    ```js
    let total = 5 * 2; // Queríamos sumar, no multiplicar
    ```
    

✅ **Conclusión:**  
Detectar errores y manejarlos correctamente es vital para **evitar fallos del programa, mejorar la experiencia del usuario y mantener un código confiable**.

---

## **Manejo de errores con `try...catch`**

### **Definición**

El bloque `try...catch` permite **capturar y manejar errores** de forma controlada, evitando que el programa se detenga abruptamente.

- **`try`** → Contiene el código que puede producir un error.
    
- **`catch`** → Captura el error si ocurre y permite manejarlo (mostrar un mensaje, ejecutar código alternativo, etc.).
    
- **`finally`** → (Opcional) Bloque que se ejecuta **siempre**, ocurra o no un error, útil para liberar recursos o ejecutar limpieza.
    

**Fuente MDN:** [try...catch](https://developer.mozilla.org/es/docs/Web/JavaScript/Reference/Statements/try...catch)

---

### **Sintaxis básica**

```js
try {
  // Código que puede generar un error
} catch (error) {
  // Código que se ejecuta si hay error
} finally {
  // Código que se ejecuta siempre
}
```

---

### **Ejemplo práctico**

```js
try {
  let resultado = 10 / y; // y no está definida
  console.log("Resultado:", resultado);
} catch (error) {
  console.error("Ocurrió un error:", error.message);
} finally {
  console.log("Bloque finally ejecutado siempre");
}
```

**Salida esperada:**

```
Ocurrió un error: y is not defined
Bloque finally ejecutado siempre
```

**Explicación paso a paso:**

1. El bloque `try` intenta ejecutar `10 / y`, pero `y` no está definida → `ReferenceError`.
    
2. La ejecución pasa inmediatamente al bloque `catch`, donde `error.message` contiene el mensaje del error.
    
3. Independientemente de lo anterior, el bloque `finally` se ejecuta siempre, útil para liberar recursos o cerrar conexiones.
    

---

### **¿Por qué usar `try...catch`?**

- Para **prevenir que errores detengan la aplicación**.
    
- Para **proporcionar mensajes amigables** al usuario.
    
- Para **realizar acciones correctivas o alternativas** cuando algo falla.
    
- Para **registrar errores** en sistemas de logs o bases de datos.

## **Expresión `throw` en JavaScript**

### **Definición**

La **expresión `throw`** se utiliza para **lanzar (generar) una excepción** en JavaScript.  
Cuando se ejecuta, **detiene inmediatamente el flujo normal del programa** y pasa el control al bloque `catch` más cercano si existe, o termina el programa si no hay manejo de errores.

---

### **Sintaxis general**

```js
throw expresión;
```

- `expresión` puede ser **cualquier valor**: número, cadena, booleano, objeto, función, etc.
    
- Aunque se puede lanzar cualquier cosa, es recomendable lanzar **objetos de tipo Error** o subtipos para obtener información clara y estructurada sobre la excepción.
    

---

### **Tipos de valores que se pueden lanzar**

```js
throw "Error de texto";  // String
throw 42;                // Number
throw true;              // Boolean
throw {                  // Objeto
  toString: function() {
    return "¡Soy un objeto!";
  }
};
```

**Explicación:**

- **Cualquier tipo de expresión es válido** para `throw`.
    
- Sin embargo, lanzar **objetos Error** es la práctica estándar porque proporcionan propiedades útiles:
    
    - `name`: tipo del error (p. ej., `"Error"`, `"TypeError"`).
        
    - `message`: descripción del error.
        
    - `stack`: traza del error (útil para depuración).
        

---

### **Crear y lanzar un error personalizado**

Podemos crear un objeto de error propio para manejar situaciones específicas de nuestra lógica:

```js
// Definir un tipo de error personalizado
function UserException(message) {
  this.message = message;
  this.name = "UserException";
}

// Definir cómo se convierte el objeto a cadena
UserException.prototype.toString = function () {
  return `${this.name}: "${this.message}"`;
};

// Lanzar el error
throw new UserException("Valor muy alto");
```

**Explicación paso a paso:**

1. **Creamos la función constructora `UserException`**: define `message` y `name`.
    
2. **Agregamos un método `toString`** para personalizar cómo se muestra en consola.
    
3. **Creamos una instancia** con `new UserException(...)` y la **lanzamos con `throw`**.
    
4. El flujo se interrumpe en ese punto y, si hay un `try...catch`, el bloque `catch` puede capturar la instancia del error y acceder a sus propiedades (`message`, `name`).
    

---

### **Uso con `try...catch`**

```js
try {
  throw new UserException("Valor muy alto");
} catch (error) {
  console.log("Se atrapó un error:", error.toString());
}
```

**Salida en consola:**

```
Se atrapó un error: UserException: "Valor muy alto"
```

**Ventajas de lanzar objetos de error:**

- Permite **manejar errores de forma estructurada**.
    
- Se puede acceder a propiedades específicas del error (`name`, `message`, `stack`).
    
- Facilita la depuración y registro en sistemas complejos.
    

---

### **Resumen clave**

|Concepto|Descripción|
|---|---|
|`throw`|Lanza una excepción y detiene el flujo de ejecución normal.|
|Expresión|Puede ser cualquier valor: string, number, boolean, objeto, etc.|
|Objetos `Error`|Recomendado para manejar errores correctamente y acceder a `name`, `message` y `stack`.|
|Uso con `try...catch`|Permite capturar la excepción y manejarla sin detener todo el programa.|
|Personalización|Podemos crear **errores propios** con constructor y método `toString` para mensajes claros.|

## **Tipos de errores predefinidos en JavaScript**

|Tipo de Error|Descripción breve|
|---|---|
|`Error`|Error genérico.|
|`SyntaxError`|Error de sintaxis, no permite ejecutar el código.|
|`ReferenceError`|Variable o referencia inexistente.|
|`TypeError`|Operación sobre un tipo de dato incorrecto (por ejemplo, llamar a algo que no es función).|
|`RangeError`|Valor fuera del rango permitido.|
|`URIError`|Error relacionado con funciones de codificación/decodificación URI.|
|`EvalError`|Error de uso incorrecto de `eval()`.|

> Fuente MDN: [Errores en JavaScript](https://developer.mozilla.org/es/docs/Web/JavaScript/Reference/Global_Objects/Error)

## **Buenas prácticas en manejo de errores**

1. **No silenciar errores:** siempre registrar o mostrar información útil.
    
2. **Usar `try...catch` solo donde es probable que ocurra un error.**
    
    - No rodear todo el programa con `try...catch` innecesariamente.
        
3. **Proveer mensajes claros y descriptivos** para facilitar depuración.
    
4. **Usar `finally`** para liberar recursos, cerrar conexiones o limpiar memoria.
    
5. **Lanzar errores personalizados (`throw`)** para validar datos o condiciones críticas.
    

---

### **Ejemplo avanzado combinando throw y try...catch**

```js
function validarEdad(edad) {
  if (typeof edad !== "number") {
    throw new TypeError("La edad debe ser un número");
  }
  if (edad < 0) {
    throw new RangeError("La edad no puede ser negativa");
  }
  return true;
}

try {
  validarEdad(-5);
} catch (error) {
  console.error(`${error.name}: ${error.message}`);
} finally {
  console.log("Validación finalizada");
}
```

**Salida:**

```
RangeError: La edad no puede ser negativa
Validación finalizada
```

**Explicación:**

- `throw` genera un error específico (`TypeError` o `RangeError`).
    
- `catch` captura cualquier tipo de error y permite manejarlo de manera controlada.
    
- `finally` garantiza que siempre se ejecute un bloque final de limpieza o confirmación.
