# Qué es una estructura de control?

Una **estructura de control** es un mecanismo que permite **modificar el flujo de ejecución** de un programa.  
En un programa sencillo, las instrucciones se ejecutan de manera secuencial, una tras otra. Sin embargo, en la mayoría de los casos se necesitan **tomar decisiones**, **repetir bloques de código** o **alterar el orden normal de ejecución** según ciertas condiciones.

Las estructuras de control son precisamente las herramientas que nos permiten hacer esto, dándole **flexibilidad e inteligencia al código**.

En términos generales, podemos decir que una estructura de control responde a la pregunta:

> **¿Qué debe hacer el programa en este punto, dependiendo de la situación?**

---

## Clasificación general de las estructuras de control

En JavaScript, las estructuras de control pueden clasificarse en tres grandes grupos:

### 1. **Estructuras condicionales**

Permiten ejecutar un bloque de código **solo si se cumple una condición**.  
Son útiles cuando el programa necesita **tomar decisiones**.

- `if`
    
- `else`
    
- `else if`
    
- `switch`
    
- Operador ternario `?:`
    

👉 Se usan cuando necesitamos que el programa **siga un camino u otro** según determinadas circunstancias.

---

### 2. **Estructuras de repetición o bucles**

Permiten **repetir un bloque de código varias veces**, ya sea un número definido de veces o hasta que se cumpla cierta condición.

- `for`
    
- `while`
    
- `do...while`
    
- Métodos de iteración como `for...of` y `for...in`
    

👉 Se aplican cuando queremos **iterar datos, recorrer arreglos o ejecutar una instrucción varias veces sin repetir código manualmente**.

---

### 3. **Estructuras de control de flujo**

Son instrucciones especiales que permiten **alterar o interrumpir la ejecución normal** dentro de condicionales o bucles.

- `break` → detiene la ejecución en un bucle o `switch`.
    
- `continue` → salta a la siguiente iteración de un bucle.
    
- `return` → termina la ejecución de una función y devuelve un valor.
    
- `throw` / `try...catch` → permiten el manejo de errores y excepciones.
    

👉 Estas estructuras sirven para **controlar de forma más precisa** qué hace el programa en situaciones particulares.

---

## ¿Por qué son importantes las estructuras de control?

Sin estructuras de control, los programas serían **lineales**: ejecutarían una instrucción tras otra sin poder tomar decisiones ni repetir procesos.  
Gracias a ellas, un programa puede:

- Adaptarse a diferentes entradas y escenarios.
    
- Evitar la repetición innecesaria de código.
    
- Organizar mejor la lógica del problema que resuelve.
    
- Manejar situaciones inesperadas (errores, datos faltantes, valores no válidos).
    

En definitiva, **dan poder y flexibilidad al lenguaje de programación**, permitiendo crear soluciones dinámicas y escalables.

---

## Ejemplo ilustrativo

Un caso sencillo que combina distintas estructuras:

```js
let edad = 20;
let intentos = 0;

while (intentos < 3) { // Estructura repetitiva
  if (edad >= 18) {   // Estructura condicional
    console.log("Acceso permitido");
    break;            // Control de flujo: termina el bucle
  } else {
    console.log("Acceso denegado");
    intentos++;
  }
}
```

En este ejemplo:

- Se repite un proceso con `while`.
    
- Dentro se toma una decisión con `if`.
    
- Se interrumpe la ejecución con `break`.
    

Esto muestra cómo las estructuras de control **trabajan en conjunto** para dar forma a la lógica de un programa.

## Resumen rápido

|Grupo|Función principal|Ejemplos|
|---|---|---|
|**Condicionales**|Decidir qué camino seguir|`if`, `else`, `else if`, `switch`, `?:`|
|**Repetitivas (bucles)**|Repetir acciones de forma automática|`for`, `while`, `do...while`, `for...of`, `for...in`|
|**Control de flujo**|Alterar o interrumpir la ejecución|`break`, `continue`, `return`, `throw`|
