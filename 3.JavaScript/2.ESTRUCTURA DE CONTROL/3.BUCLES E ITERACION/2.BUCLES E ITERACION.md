# **Fundamentos: Iteración y Bucles en JavaScript**

## ¿Qué es una iteración?

En programación, una **iteración** es cada **repetición de un bloque de instrucciones** dentro de un bucle.  
Podemos pensarlo como una "vuelta" que da el bucle. Cada vez que el bloque de código se ejecuta y vuelve al inicio para comprobar la condición, se ha completado una iteración.

**Ejemplo mental**:  
Si tenés que llamar a 5 amigos, cada llamada representa **una iteración** dentro de una secuencia repetitiva. El conjunto de todas las llamadas constituye el bucle.

---

## ¿Qué es un bucle (loop)?

Un **bucle** es una estructura de control que permite **repetir un conjunto de instrucciones automáticamente**.  
Se utiliza cuando una tarea debe realizarse más de una vez, sin necesidad de escribir el mismo código varias veces.

El número de repeticiones depende de diferentes factores:

- **Una condición lógica** → Ejemplo: repetir mientras “la batería esté cargando”.
    
- **Un contador** → Ejemplo: repetir exactamente 10 veces.
    
- **Una colección de datos** → Ejemplo: recorrer cada elemento de un array o cada letra de un string.
    

### Ventaja principal de los bucles

Permiten **automatizar tareas repetitivas**, evitando código duplicado y haciendo los programas más claros, escalables y fáciles de mantener.

## Aclaración Importante

## **Diferencia entre notación de punto y notación de corchetes en objetos**

En JavaScript, un objeto almacena información en forma de pares **clave–valor**.  
Existen dos maneras de acceder al valor de una propiedad: **notación de punto (`.`)** y **notación de corchetes (`[]`)**.

Aunque ambas sirven para leer o modificar propiedades, no son equivalentes: cada una tiene usos más adecuados según el contexto.

---

### **1. Notación de punto (`obj.propiedad`)**

- Se utiliza cuando **sabemos de antemano** el nombre de la propiedad que queremos acceder.
    
- Es la forma más **simple, directa y legible**.
    
- La clave debe ser un **identificador válido de JavaScript** (sin espacios, acentos, guiones ni caracteres especiales).
    

**Ejemplo:**

```js
const persona = { nombre: "Iván", edad: 21, pais: "Argentina" };

console.log(persona.nombre); // "Iván"
console.log(persona.edad);   // 21
```

📌 Aquí accedemos directamente a `nombre` y `edad` porque ya conocemos las propiedades al momento de escribir el código.

---

### **2. Notación de corchetes (`obj["propiedad"]`)**

- Se utiliza cuando el nombre de la propiedad **se encuentra en una variable** o se construye **dinámicamente**.
    
- Permite acceder a claves que no serían válidas con punto (por ejemplo, nombres con espacios, números como texto, o guiones).
    
- Es indispensable en contextos donde no conocemos de antemano qué clave vamos a usar.
    

**Ejemplo con variable:**

```js
const persona = { nombre: "Iván", edad: 21, pais: "Argentina" };

let clave = "edad";
console.log(persona[clave]); // 21
```

**Ejemplo con claves no válidas para punto:**

```js
const libro = { "año-publicación": 2024, "título completo": "JS desde cero" };

console.log(libro["año-publicación"]); // 2024
console.log(libro["título completo"]); // "JS desde cero"
```

👉 Si intentáramos `libro.año-publicación`, daría error porque no es un identificador válido.

## Bucle `while`

### Definición

El bucle `while` ejecuta su bloque de instrucciones **mientras la condición especificada sea verdadera (`true`)**.  
La evaluación se realiza **antes de cada iteración**, por lo tanto, si la condición es falsa desde el inicio, el bloque nunca se ejecuta.

### Sintaxis

```js
while (condición) {
  // Código que se ejecuta en cada iteración
}
```

### Ejemplo

```js
let numero = 1;

while (numero <= 5) {
  console.log("Número:", numero);
  numero++;
}
```

**Explicación**:

- Empieza en `1`.
    
- Comprueba si `numero <= 5`.
    
- Mientras sea verdadero, muestra el valor y lo incrementa.
    
- Cuando `numero` llega a `6`, la condición deja de cumplirse y el bucle finaliza.
    

---

## Bucle `do...while`

### Definición

El bucle `do...while` se diferencia del `while` en que **ejecuta primero el bloque de código y después evalúa la condición**.  
Esto garantiza que el bloque se ejecute **al menos una vez**, incluso si la condición es falsa desde el principio.

### Sintaxis

```js
do {
  // Código que se ejecuta al menos una vez
} while (condición);
```

### Ejemplo

```js
let numero = 6;

do {
  console.log("Número:", numero);
  numero++;
} while (numero <= 5);
```

**Explicación**:

- Aunque `numero` empieza en `6` (no cumple la condición), el bloque se ejecuta una vez.
    
- Después de la primera ejecución, la condición se evalúa y como es falsa, el bucle termina.
    

👉 Es útil en situaciones donde se necesita **ejecutar la acción mínimo una vez**, por ejemplo: mostrar un formulario al menos una vez antes de validar.

---

## Bucle `for` (clásico)

### Definición

El bucle `for` es ideal cuando **sabemos de antemano cuántas veces queremos repetir una acción**.  
En su estructura incluye:

1. **Inicialización** de una variable.
    
2. **Condición** que controla el número de iteraciones.
    
3. **Actualización** (incremento o decremento) de la variable de control.
    

### Sintaxis

```js
for (inicialización; condición; actualización) {
  // Código en cada iteración
}
```

### Ejemplo

```js
for (let i = 1; i <= 5; i++) {
  console.log("Número:", i);
}
```

**Explicación**:

- Inicia `i` en `1`.
    
- Se repite mientras `i <= 5`.
    
- En cada vuelta, `i` aumenta en `1`.
    
- Termina cuando `i` supera `5`.
    

---

## Bucle `for...in`

### Definición

El bucle `for...in` recorre todas las **propiedades enumerables** de un objeto.  
Cuando se utiliza con arrays, en lugar de los valores, devuelve los **índices** en formato de cadena de texto.

### Sintaxis

```js
for (let clave in objeto) {
  // Accedemos con objeto[clave]
}
```

### Ejemplo con objetos

```js
const persona = { nombre: "Iván", edad: 21, pais: "Argentina" };

for (let propiedad in persona) {
  console.log(propiedad + ":", persona[propiedad]);
}
```

**Salida**:

```
nombre: Iván
edad: 21
pais: Argentina
```

### Ejemplo con arrays

```js
const frutas = ["manzana", "banana", "pera"];

for (let indice in frutas) {
  console.log("Índice:", indice, "Valor:", frutas[indice]);
}
```

**Salida**:

```
Índice: 0 Valor: manzana
Índice: 1 Valor: banana
Índice: 2 Valor: pera
```

👉 `for...in` es más recomendable para **objetos** que para arrays, especialmente si son grandes, ya que puede incluir propiedades heredadas.

---

## Bucle `for...of`

### Definición

El bucle `for...of` está diseñado para recorrer los **valores directamente** de cualquier estructura **iterable** como arrays, strings, Map, Set, entre otros.  
Es más moderno y legible que `for...in` cuando lo que nos interesa son los **valores**.

### Sintaxis

```js
for (let valor of iterable) {
  // Se accede al valor directamente
}
```

### Ejemplo con arrays

```js
const frutas = ["manzana", "banana", "pera"];

for (let fruta of frutas) {
  console.log("Fruta:", fruta);
}
```

**Salida**:

```
Fruta: manzana
Fruta: banana
Fruta: pera
```

### Ejemplo con strings

```js
const palabra = "Hola";

for (let letra of palabra) {
  console.log("Letra:", letra);
}
```

**Salida**:

```
Letra: H
Letra: o
Letra: l
Letra: a
```

👉 Es especialmente útil para recorrer **colecciones de datos modernas** y trabajar de forma directa con los valores sin preocuparse por índices o claves.

---

## Cuadro comparativo de bucles

|Bucle|Qué recorre / controla|Evalúa condición antes|Ejecuta al menos una vez|Usos recomendados|
|---|---|---|---|---|
|**while**|Condición lógica|Sí|No|Cuando la cantidad de repeticiones depende de una condición dinámica|
|**do...while**|Condición lógica|No|Sí|Cuando necesitamos ejecutar el bloque al menos una vez|
|**for**|Contador definido|Sí|No|Cuando sabemos cuántas iteraciones deben ejecutarse|
|**for...in**|Propiedades enumerables de un objeto|Sí|No|Ideal para recorrer objetos, no recomendado para arrays grandes|
|**for...of**|Valores de iterables (arrays, strings, Maps, Sets)|Sí|No|Útil para trabajar con colecciones de datos modernas de forma directa|
