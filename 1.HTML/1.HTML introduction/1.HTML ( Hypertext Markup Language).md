
## ¿Qué es HTML?

**HTML** significa **HyperText Markup Language** (Lenguaje de Marcado de Hipertexto).  
Es el **lenguaje principal que usamos para estructurar el contenido de una página web**.

No es un lenguaje de programación (como JavaScript), sino un **lenguaje de marcado**, lo que significa que usamos etiquetas para decirle al navegador cómo debe organizar y mostrar el contenido.


## ¿Para qué sirve HTML?

- Para **crear la estructura** de una página web.
    
- Para **organizar el contenido**: texto, imágenes, enlaces, listas, formularios, tablas, videos, etc.
    
- Para **darle significado** al contenido: por ejemplo, decir qué parte es un título, qué parte es un párrafo, qué es un botón, etc.
    
- Para hacer que otros lenguajes (como CSS y JavaScript) se puedan aplicar a esa estructura.


## ¿Cómo funciona HTML?

HTML está compuesto por **etiquetas** que rodean al contenido y lo **marcan**. Estas etiquetas se escriben entre símbolos de menor y mayor: `< >`.

Ejemplo básico:
`<h1>Hola mundo</h1>``
`<p>Este es mi primer párrafo en HTML.</`p>

Esto le dice al navegador:

- `<h1>` → Esto es un título principal.
    
- `<p>` → Esto es un párrafo de texto.

## Anatomía de un elemento HTML

Exploremos más a fondo nuestro elemento de párrafo de la sección anterior:

![Un fragmento de código de muestra que demuestra la estructura de un elemento HTML.<p>Mi gato es muy gruñón</p>.](https://developer.mozilla.org/en-US/docs/Learn_web_development/Core/Structuring_content/Basic_HTML_syntax/grumpy-cat-small.png)

La anatomía de nuestro elemento es:

- **Etiqueta de apertura:** Consiste en el nombre del elemento (en este ejemplo, _p_ de párrafo), entre corchetes angulares de apertura y cierre. Esta etiqueta de apertura marca el inicio del elemento o su efecto. En este ejemplo, precede al inicio del texto del párrafo.
- **Contenido:** Este es el contenido del elemento. En este ejemplo, es el texto del párrafo.
- **Etiqueta de cierre:** Es igual que la etiqueta de apertura, excepto que incluye una barra diagonal antes del nombre del elemento. Esta marca dónde termina el elemento. No incluir una etiqueta de cierre es un error común de principiantes que puede producir resultados extraños.

El elemento es la etiqueta de apertura, seguida del contenido, seguido de la etiqueta de cierre.

### Elementos anidados (Nesting elements)

Los elementos se pueden colocar dentro de otros elementos. Esto se denomina _anidamiento_ . Si quisiéramos indicar que nuestro gato está **muy** gruñón, podríamos incluir la palabra _"muy"_ en un [`<strong>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/strong)elemento, lo que significa que la palabra debe tener un formato de texto más fuerte:


```
<p>My cat is <strong>very</strong> grumpy.</p>
```

Hay una forma correcta e incorrecta de anidar. En el ejemplo anterior, abrimos el `p`elemento primero y luego el `strong`elemento. Para una anidación correcta, debemos cerrar el `strong`elemento primero, antes de cerrar el archivo `p`.

El siguiente es un ejemplo de la forma _incorrecta_ de realizar la anidación:


```
<p>My cat is <strong>very grumpy.</p></strong>
```

Las **etiquetas deben abrirse y cerrarse de forma que queden dentro o fuera de la otra** . Con el tipo de superposición del ejemplo anterior, el navegador debe adivinar su intención. Este tipo de adivinación puede generar resultados inesperados.

### Elementos vacíos (Void elements)

No todos los elementos siguen el patrón de una etiqueta de apertura, contenido y cierre. Algunos elementos constan de una sola etiqueta, que suele usarse para insertar o incrustar algo en el documento. Estos elementos se denominan [elementos void](https://developer.mozilla.org/en-US/docs/Glossary/Void_element) .
