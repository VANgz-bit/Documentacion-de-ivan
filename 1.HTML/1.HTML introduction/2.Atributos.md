
Los elementos también pueden tener atributos. Los atributos se ven así:

![etiqueta de párrafo con el atributo 'class="editor-note"' enfatizado](https://developer.mozilla.org/en-US/docs/Learn_web_development/Core/Structuring_content/Basic_HTML_syntax/grumpy-cat-attribute-small.png)

Los atributos contienen información adicional sobre el elemento que no aparecerá en el contenido. En este ejemplo, el **`class`**atributo es un nombre de identificación que se utiliza para identificar el elemento con información de estilo.

Un atributo debe tener:

- Un espacio entre este y el nombre del elemento. (Para un elemento con más de un atributo, estos también deben estar separados por espacios).
- El nombre del atributo, seguido de un signo igual.
- Un valor de atributo, entre comillas de apertura y cierre.

### Agregar atributos a un elemento

Ahora te toca de nuevo. En esta sección, te enseñaremos a agregar atributos a un `<img>`elemento para que se muestre una imagen en la página. El `<img>`elemento puede aceptar varios atributos, entre ellos:

- `src`Atributo **obligatorio** que especifica la ubicación de la imagen. Por ejemplo: `src="https://raw.githubusercontent.com/mdn/beginner-html-site/gh-pages/images/firefox-icon.png"`.
- `alt`Descripción textual de la imagen. Por ejemplo: `alt="The Firefox icon"`.


### Atributos booleanos

A veces verá atributos escritos sin valores. Esto es totalmente aceptable. Estos se denominan atributos booleanos . Cuando un atributo booleano se escribe sin valor, o con cualquier valor, incluso como `"false"`, el atributo booleano siempre se establece en verdadero. De lo contrario, si el atributo no está escrito en una etiqueta HTML, se establece en falso. La especificación requiere que el valor del atributo sea una cadena vacía (incluso cuando el atributo no tiene un valor especificado explícitamente) o el mismo que el nombre del atributo, pero los demás valores funcionan de la misma manera. Por ejemplo, considere el `disabled`atributo , que puede asignarse a los elementos de entrada del formulario. (Esto se usa para _deshabilitar_ los elementos de entrada del formulario para que el usuario no pueda realizar entradas. Los elementos deshabilitados suelen aparecer en gris). Por ejemplo:


```
<input type="text" disabled="disabled" />
```

A modo de abreviatura, es aceptable escribirlo de la siguiente manera:

<!-- El uso del atributo deshabilitado evita que el usuario final ingrese texto en el  input box -->
`<input type="text" disabled />`

<!-- Se permite la entrada de texto, ya que no contiene el atributo deshabilitado -->
`<input type="text" />`

### Omitir comillas alrededor de los valores de los atributos

Si revisas el código de muchos otros sitios, podrías encontrarte con varios estilos de marcado extraños, incluyendo valores de atributo sin comillas. Esto está permitido en ciertas circunstancias, pero también puede afectar tu marcado en otras. El elemento del siguiente fragmento de código, `<a>`, se llama ancla. Las anclas encierran texto y lo convierten en enlaces. El `href`atributo especifica la dirección web a la que apunta el enlace. Puedes escribir esta versión básica a continuación _solo_ con el `href`atributo, así:


```
<a href=https://www.mozilla.org/>favorite website</a>
```

Los anclajes también pueden tener un `title`atributo, una descripción de la página enlazada. Sin embargo, al añadirlos `title`de la misma forma que el `href`atributo, surgen problemas:


```
<a href=https://www.mozilla.org/ title=The Mozilla homepage>favorite website</a>
```

Como se mencionó anteriormente, el navegador malinterpreta el marcado, confundiendo el `title`atributo con tres atributos: un atributo de título con el valor `The`, y dos atributos booleanos, `Mozilla`y `homepage`. Obviamente, esto no es intencional. Provocará errores o un comportamiento inesperado
