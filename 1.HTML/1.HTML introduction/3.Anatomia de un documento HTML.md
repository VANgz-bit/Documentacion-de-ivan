## Anatomía de un documento HTML

Los elementos HTML individuales no son muy útiles por sí solos. A continuación, examinemos cómo se combinan para formar una página HTML completa:

```
<!doctype html>
<html lang="en-US">
  <head>
    <meta charset="utf-8" />
    <title>My test page</title>
  </head>
  <body>
    <p>This is my page</p>
  </body>
</html>
```

Aquí tenemos:

1. `<!doctype html>`El doctype. En sus inicios (1991-1992), los doctypes se concibieron como enlaces a un conjunto de reglas que la página HTML debía seguir para ser considerada HTML de calidad. Los doctypes solían tener un aspecto similar a este:


	  ```
   <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
    ```
    
    Más recientemente, el tipo de documento es un artefacto histórico que debe incluirse para que todo lo demás funcione correctamente. `<!doctype html>`Es la cadena de caracteres más corta que se considera un tipo de documento válido. ¡Eso es todo lo que necesitas saber!
    
2. `<html></html>`El `<html>`elemento. Este elemento envuelve todo el contenido de la página. A veces se le conoce como elemento raíz.
    
3. `<head></head>`El `<head>` elemento. Este elemento sirve como contenedor de todo lo que desea incluir en la página HTML, **excepto el contenido que** la página mostrará a los visitantes. Esto incluye palabras clave y una descripción de la página que aparecerá en los resultados de búsqueda, CSS para aplicar estilo al contenido, declaraciones de conjuntos de caracteres y más. Aprenderá más sobre esto en el siguiente artículo de la serie.
    
4. `<meta charset="utf-8">`: El `<meta>`elemento. Este elemento representa metadatos que no pueden representarse con otros elementos meta relacionados con HTML, como , , [`<base>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/base)o [`<link>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/link). El atributo especifica la codificación de caracteres del documento como UTF-8, que incluye la mayoría de los caracteres de la gran mayoría de los idiomas escritos. Con esta configuración, la página ahora puede gestionar cualquier contenido textual que pueda contener. No hay motivo para no configurarlo y puede ayudar a evitar problemas posteriores.`<scriptstyle>` `<title>``charset`
    
5. `<title></title>`El `<title>`)elemento. Define el título de la página, que aparece en la pestaña del navegador donde se carga. El título también se usa para describirla cuando se guarda en marcadores.
    
6. `<body></body>`El `<body>`elemento. Contiene _todo_ el contenido que se muestra en la página, incluyendo texto, imágenes, vídeos, juegos, pistas de audio reproducibles y cualquier otro elemento.
