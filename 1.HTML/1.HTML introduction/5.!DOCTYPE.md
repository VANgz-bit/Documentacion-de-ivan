## `<!DOCTYPE>` en HTML

### 1. Definición

El `<!DOCTYPE>` (Document Type Declaration o Declaración de Tipo de Documento) es una instrucción que se coloca al **inicio de un documento HTML**, antes de la etiqueta `<html>`, y tiene como función **informarle al navegador qué versión de HTML se está utilizando** y cómo debe interpretar el contenido de la página.

Es importante destacar que `<!DOCTYPE>` **no es una etiqueta de HTML**, sino una declaración que pertenece al estándar del lenguaje. No requiere cierre y siempre va al principio del documento.

---
## 2. Función principal del `<!DOCTYPE>` (Ampliado)

El `DOCTYPE` no es simplemente una línea al inicio del documento: cumple **varias funciones críticas** que impactan directamente en cómo los navegadores interpretan y renderizan la página. Se pueden resumir en tres funciones principales, que vamos a analizar en detalle:

---

### 2.1 Activar el modo estándar del navegador

#### Qué significa

Cuando un navegador carga una página, puede usar distintos **modos de renderizado**:

1. **Modo estándar (Standards Mode):**  
    El navegador interpreta el HTML y CSS **siguiendo los estándares oficiales** definidos por el W3C. Esto garantiza que las propiedades CSS funcionen correctamente y que la estructura del HTML sea interpretada de forma consistente.
    
2. **Modo quirks (Quirks Mode):**  
    Si no hay `DOCTYPE` o es incorrecto, el navegador intenta mantener compatibilidad con páginas antiguas de los años 90 y 2000. Esto provoca que ciertas reglas de CSS se comporten de manera **extraña o inconsistente**, como márgenes automáticos, dimensiones de cajas, o interpretación de tablas y floats.
    
3. **Modo casi estándar (Almost Standards Mode):**  
    Un modo intermedio usado en casos específicos de compatibilidad con imágenes o elementos de texto antiguos.
    

#### Ejemplo práctico

Sin `DOCTYPE`:

```html
<html>
<head>
<style>
div { width: 200px; padding: 10px; border: 5px solid black; }
</style>
</head>
<body>
<div>Contenido</div>
</body>
</html>
```

- En modo quirks, **el ancho total del `div` puede incluir el padding y el borde**, alterando el diseño esperado según CSS moderno.
    
- Con `<!DOCTYPE html>`, el navegador aplica **el modelo de caja moderno**, donde el ancho se interpreta correctamente solo como el contenido, y el padding y borde se suman aparte si no usamos `box-sizing`.
    

**Conclusión:** El `DOCTYPE` asegura que la página se renderice como esperamos según los estándares actuales.

---

### 2.2 Indicar la versión de HTML

#### Qué significa

El `DOCTYPE` **informa al navegador sobre la versión del lenguaje HTML que se está utilizando**, lo cual era crucial antes de HTML5. Diferentes versiones permitían distintos elementos, atributos y reglas de sintaxis.

- HTML 4.01 Strict: no permitía etiquetas obsoletas como `<font>` o `<center>`.
    
- HTML 4.01 Transitional: permitía elementos de presentación heredados para compatibilidad.
    
- XHTML: obligaba a **cerrar todas las etiquetas** y usar atributos con comillas, siguiendo reglas de XML.
    
- HTML5: simplifica el `DOCTYPE` y hace innecesario indicar versión explícita.
    

#### Ejemplo práctico

```html
<!DOCTYPE html> <!-- HTML5 -->
<p>Esto se interpreta como HTML5, con soporte completo para etiquetas modernas como <section>, <article> o <video>.</p>
```

- Con HTML 4.01 Transitional, algunas etiquetas modernas podrían **no ser reconocidas correctamente** por el navegador antiguo.
    

**Conclusión:** La declaración ayuda al navegador a interpretar correctamente los elementos que se usan y evitar errores de compatibilidad.

---

### 2.3 Compatibilidad con herramientas externas

#### Qué significa

- Muchos **validadores de HTML**, linters, editores y frameworks web usan el `DOCTYPE` para **determinar las reglas de validación**.
    
- Sin un `DOCTYPE` correcto, estas herramientas pueden marcar errores falsos o ignorar ciertos elementos.
    

#### Ejemplo práctico

```html
<!DOCTYPE html>
<html>
<head>
    <title>Validador W3C</title>
</head>
<body>
    <h1>Probando validación</h1>
</body>
</html>
```

- Si se valida esta página en el [validador W3C](https://validator.w3.org/), no dará errores, porque el `DOCTYPE` indica que estamos usando HTML5 y se aplica la sintaxis correcta.
    
- Si omitimos el `DOCTYPE`, algunos validadores **podrían marcar errores de compatibilidad** o suponer que usamos HTML 4, aunque la sintaxis sea HTML5.
    

**Conclusión:** El `DOCTYPE` asegura que las herramientas externas, editores y validadores **interpreten correctamente tu documento** y detecten problemas reales, no falsos positivos.

---

### 2.4 Resumen ampliado de funciones

|Función|Explicación|Ejemplo/Impacto|
|---|---|---|
|Activar modo estándar|Garantiza que el navegador interprete HTML y CSS según estándares modernos|Layouts y estilos se renderizan correctamente; evita errores de box-model o floats|
|Indicar versión de HTML|Señala la versión del lenguaje, determinando qué elementos y atributos son válidos|HTML5 permite `<section>`, `<video>`; HTML 4 no los reconoce|
|Compatibilidad con herramientas|Permite que validadores, linters y frameworks detecten errores reales y no incompatibilidades por versión|Validador W3C funciona correctamente; frameworks modernos reconocen la sintaxis|

### 3. Sintaxis

#### HTML5 (actual estándar más usado)

```html
<!DOCTYPE html>
```

- Es **simple, corta y sin versión**, a diferencia de versiones anteriores de HTML.
    
- Es **insensible a mayúsculas**, aunque la convención es escribirlo en mayúsculas.
    
- Debe ir **exactamente antes de `<html>`** y no debe haber nada antes (ni espacios, ni comentarios).
    

Ejemplo completo:

```html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Ejemplo de DOCTYPE</title>
</head>
<body>
    <h1>Hola, mundo!</h1>
</body>
</html>
```

---

#### Versiones anteriores (por contexto histórico)

1. **HTML 4.01 Strict**
    

```html
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
```

- Definía estrictamente las reglas del HTML sin permitir elementos de presentación (como `<font>` o atributos de estilo).
    

2. **HTML 4.01 Transitional**
    

```html
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
```

- Permitía algunos elementos y atributos obsoletos, facilitando la transición de HTML antiguo a HTML moderno.
    

3. **HTML 4.01 Frameset**
    

```html
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN" "http://www.w3.org/TR/html4/frameset.dtd">
```

- Usado cuando se utilizaban **frames** (divisiones de ventana), técnica hoy obsoleta.
    

4. **XHTML 1.0 Strict**
    

```html
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
```

- Requería **HTML bien formado**, con etiquetas cerradas, atributos entre comillas y sensibilidad al caso en XML.
    
- XHTML fue importante para aplicaciones que querían combinar HTML con reglas XML.
    

---

### 4. Consideraciones importantes

1. **Ubicación:**  
    Siempre al inicio del documento, **antes de `<html>`**.
    
2. **Compatibilidad:**
    
    - HTML5 simplifica la sintaxis y asegura compatibilidad moderna.
        
    - HTML 4 y XHTML aún son compatibles en navegadores, pero su uso está desaconsejado para proyectos nuevos.
        
3. **No afecta contenido visual:**
    
    - Declarar correctamente `DOCTYPE` no cambia el contenido en sí, sino cómo se interpreta el HTML.
        
    - Un `DOCTYPE` incorrecto puede romper estilos o layout, incluso si el HTML está bien escrito.
        
4. **Mayúsculas vs minúsculas:**
    
    - Se recomienda escribirlo como `<!DOCTYPE html>` (mayúscula inicial de “DOCTYPE”), pero HTML5 es insensible a mayúsculas/minúsculas.
        
5. **Validación:**
    
    - Para validar tu HTML con herramientas como el [Validador W3C](https://validator.w3.org/), el `DOCTYPE` correcto es obligatorio.

|HTML 4.01 Transitional|Largo, con URL del DTD|Permite elementos de presentación antiguos.|
|HTML 4.01 Frameset|Largo, con URL del DTD|Para páginas con frames, obsoleto.|
