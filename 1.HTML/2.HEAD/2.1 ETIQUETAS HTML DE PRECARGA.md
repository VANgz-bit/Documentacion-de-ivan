
Las **etiquetas HTML de precarga** (_resource hints_) son una serie de etiquetas `<link>` que permiten dar al navegador algo de información adicional para saber a que recursos debe darles mayor prioridad, cargar de forma anticipada, posponer o incluso prerenderizar.

- **Recursos locales** (como CSS o fuentes internas)
    
- **Recursos externos o dominios remotos** (CDNs, APIs, fuentes externas)
    
- **URLs completas (prerender/prefetch a otras páginas)**
    
---

#  1. SECCIÓN: Recursos Locales (mis archivos internos)

Estos son recursos **alojados dentro del mismo dominio de tu sitio web**, como hojas de estilo, fuentes personalizadas, scripts, etc. Se accede a ellos por **rutas relativas** (`/css/`, `/fonts/`, etc.).

---

### 🔸 `rel="preload"` para recursos locales

```html
<link rel="preload" href="/fonts/MiFuente.woff2" as="font" type="font/woff2" crossorigin="anonymous">
```

- **¿Qué hace?** Le indica al navegador que cargue una fuente crítica de forma anticipada.
    
- **Atributo `as` obligatorio:** especifica el tipo de recurso (`font`, `style`, `script`, `image`, etc).
    
- **Atributo `type`:** mejora la precisión de carga, especialmente útil con fuentes.
    
- **Atributo `crossorigin`:** necesario si el recurso se sirve con cabeceras CORS, aunque esté en el mismo dominio.
    

---

### 🔸 `rel="preload"` para hojas de estilo

```html
<link rel="preload" href="/css/estilos.css" as="style">
```

- **¿Qué hace?** Prepara la hoja de estilos **antes** de que se aplique al DOM.
    
- **Uso recomendado:** Junto con la carga real del estilo:
    

```html
<link rel="stylesheet" href="/css/estilos.css">
```

---

### 🔸 `rel="stylesheet"` (carga estándar)

```html
<link rel="stylesheet" href="/css/estilos.css">
```

- **¿Qué hace?** Aplica una hoja de estilos externa al documento.
    
- **¿Cuándo usarlo?** Siempre que quieras separar el CSS en archivos externos.
    
- **Nota:** Puede bloquear el renderizado hasta que se cargue, por eso a veces se usa con `preload`.
    

---

### 🔸 `rel="prefetch"` para scripts o imágenes internas

```html
<link rel="prefetch" href="/js/siguiente-paso.js" as="script">
```

- **¿Qué hace?** Indica al navegador que cargue este archivo JS anticipadamente **para otra página**.
    
- **No lo usa esta página, pero lo usará pronto.**
    

---

---

#  2. SECCIÓN: Recursos de Dominio Externo (CDNs, APIs, fuentes de Google)

Estos son recursos cargados desde **otro dominio** distinto al tuyo. Por ejemplo:

- `https://fonts.googleapis.com`
    
- `https://cdn.jsdelivr.net`
    
- `https://apis.google.com`
    

---

### 🔸 `rel="preconnect"`

```html
<link rel="preconnect" href="https://fonts.googleapis.com" crossorigin>
```

- **¿Qué hace?** Establece **una conexión temprana completa** con el servidor (resuelve DNS + abre TCP + negocia TLS).
    
- **¿Por qué usarlo?** Acelera la descarga de recursos externos que sabes que vas a usar (por ejemplo, fuentes de Google).
    
- **Requiere `crossorigin`** si el recurso será llamado con CORS.
    

---

### 🔸 `rel="dns-prefetch"`

```html
<link rel="dns-prefetch" href="//cdn.jsdelivr.net">
```

- **¿Qué hace?** Resuelve el dominio de forma anticipada, **sin establecer aún conexión TCP ni TLS**.
    
- **¿Cuándo usarlo?** Cuando el recurso se usa en segundo plano o no es tan crítico como para usar `preconnect`.
    

---

### 🔸 Comparación entre `preconnect` y `dns-prefetch`

|Atributo|Qué hace|Más costoso|Más útil para recursos críticos|
|---|---|---|---|
|`dns-prefetch`|Solo resuelve el dominio (DNS)|No|No (solo ahorra unos ms)|
|`preconnect`|Establece conexión completa (DNS + TCP + TLS)|Sí|Sí (ideal para APIs, fuentes, etc.)|

---

### 🔸 Ejemplo combinado (Google Fonts)

```html
<link rel="preconnect" href="https://fonts.googleapis.com" crossorigin>
<link rel="dns-prefetch" href="//fonts.gstatic.com">
<link href="https://fonts.googleapis.com/css2?family=Roboto&display=swap" rel="stylesheet">
```

- `fonts.googleapis.com`: contiene la hoja de estilo con la fuente.
    
- `fonts.gstatic.com`: contiene el archivo de fuente real (`woff2`, `ttf`, etc.)

#  3. SECCIÓN: URLs Completas (Páginas futuras del sitio)

Aquí se trabaja con **URLs completas**, muchas veces internas, pero referidas como páginas completas (no archivos específicos). Se usan para **preparar la carga de una próxima página** o incluso **renderizarla por adelantado**.

---

### 🔸 `rel="prerender"`

```html
<link rel="prerender" href="https://misitio.com/siguiente-paso">
```

- **¿Qué hace?** El navegador **carga y renderiza en segundo plano** toda la página que indiques.
    
- **Ideal para:** Formularios en pasos, flujos guiados, juegos, procesos en etapas.
    
- **Advertencia:** Puede consumir más RAM y tráfico, no se recomienda abusar.
    

---

### 🔸 `rel="prefetch"` con URL

```html
<link rel="prefetch" href="/productos/destacados.html">
```

- **¿Qué hace?** Carga esa página o recurso de manera anticipada, **sin renderizarla**.
    
- **Ideal para:** anticipar la siguiente navegación sin gastar demasiados recursos.
    

---

### Diferencia entre `prerender` y `prefetch`

|Atributo|¿Carga recurso?|¿Renderiza?|¿Consume más recursos?|¿Velocidad de navegación?|
|---|---|---|---|---|
|`prefetch`|✅ Sí|❌ No|Baja|Moderada|
|`prerender`|✅ Sí|✅ Sí|Alta (usa RAM y CPU)|Muy alta|

---

#  Conclusión final

###  ¿Cuándo usar cada uno?

- **Para recursos internos que se usan rápido (fuentes, CSS, imágenes importantes)** → `preload`
    
- **Para scripts, imágenes o páginas que se usarán después** → `prefetch`
    
- **Para acelerar navegación entre páginas** → `prerender`
    
- **Para recursos externos (CDNs, Google Fonts, APIs)**
    
    - Si son **críticos**: `preconnect`
        
    - Si son **no críticos**: `dns-prefetch`
