
##  ¿Qué es la etiqueta `<link>`?

La etiqueta `<link>` es un **elemento vacío** de HTML (no tiene contenido ni cierre) que se coloca dentro del `<head>`. Su propósito principal es **vincular el documento HTML con recursos externos**, como hojas de estilos, fuentes, íconos, manifiestos, entre otros.

No genera contenido visual, pero sí **afecta al comportamiento, estilo o configuración del documento**.

---

##  ¿Para qué sirve `<link>`?

- Aplicar **hojas de estilo** CSS externas
    
- Definir **favicon** (ícono del sitio)
    
- Cargar **fuentes externas** (Google Fonts, por ejemplo)
    
- Precargar recursos para mejorar el **rendimiento**
    
- Asociar **manifiestos** de aplicaciones web
    
- Mejorar el **SEO** mediante enlaces canónicos (`rel="canonical"`)
    

---

##  Atributos de `<link>` explicados

|Atributo|¿Obligatorio?|¿Qué hace?|
|---|---|---|
|`rel`|✅ Sí|Define la **relación** entre el documento y el recurso externo. Ejemplos: `stylesheet`, `icon`, `manifest`, `canonical`.|
|`href`|✅ Sí|Define la **ruta o URL** del archivo externo que se vincula. Puede ser relativa o absoluta.|
|`type`|Opcional|Indica el **tipo MIME** del recurso. Ej: `text/css`, `image/png`, `font/woff2`.|
|`media`|Opcional|Define el **tipo de dispositivo o condición** para aplicar el recurso (ej: `print`, `screen`, `max-width: 768px`).|
|`as`|Opcional|Usado con `rel="preload"` o `prefetch`, indica el tipo de recurso (`style`, `font`, `script`, etc.).|
|`crossorigin`|Opcional|Define la política CORS para recursos externos (`anonymous` o `use-credentials`).|
|`title`|Opcional|Se usa para identificar hojas de estilo alternas (`alternate stylesheet`).|
|`sizes`|Opcional|Define el tamaño del ícono cuando se usa `rel="icon"`. Ej: `32x32`.|

---

##  Tipos de `rel` más comunes y sus funciones

###  Estilos CSS

```html
<link rel="stylesheet" href="estilos.css">
```

Vincula una hoja de estilo externa al documento.

###  Íconos (favicon)

```html
<link rel="icon" href="favicon.png" type="image/png" sizes="32x32">
```

Define el ícono del sitio que aparece en la pestaña del navegador.

###  Hojas de estilo para impresión

```html
<link rel="stylesheet" href="print.css" media="print">
```

Se aplica solo al imprimir la página.

###  Precarga y rendimiento

```html
<link rel="preload" href="fuente.woff2" as="font" type="font/woff2" crossorigin="anonymous">
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="prefetch" href="pagina-siguiente.html" as="document">
```

Optimiza la carga anticipada de recursos como fuentes, imágenes, scripts o documentos.

###  Idiomas alternativos y RSS

```html
<link rel="alternate" hreflang="en" href="https://miweb.com/en/">
<link rel="alternate" type="application/rss+xml" href="feed.rss">
```

Define versiones alternativas del contenido, ya sea por idioma o formato (feeds).

###  Manifiesto de aplicación

```html
<link rel="manifest" href="manifest.json">
```

Asocia un archivo de manifiesto que permite transformar la web en una PWA (Progressive Web App).

###  Enlace canónico para SEO

```html
<link rel="canonical" href="https://miweb.com/pagina-principal">
```

Indica a los buscadores cuál es la versión principal de una página para evitar contenido duplicado. Mejora el SEO señalando la URL preferida.

##  Relación entre `<link>` y `<base>`

###  ¿Qué es `<base>`?

La etiqueta `<base>` se coloca en el `<head>` y define una **URL base** para todas las rutas relativas del documento. También puede establecer un **target predeterminado** para todos los enlaces.

```html
<base href="https://miweb.com/">
<base target="_blank">
```

- Con `href`: define la URL base para resolver rutas relativas (en `href`, `src`, etc.).
    
- Con `target`: establece cómo deben abrirse los enlaces por defecto (por ejemplo, en nueva pestaña).
    

🚫 Solo se permite una etiqueta `<base>` por documento.

###  ¿Cómo afecta a `<link>`?

Si una etiqueta `<link>` usa una ruta relativa en `href`, esta se resuelve en base a la URL definida en `<base>`.

Ejemplo:

```html
<base href="https://miweb.com/subcarpeta/">
<link rel="stylesheet" href="css/estilo.css"> <!-- Resultado final: https://miweb.com/subcarpeta/css/estilo.css -->
```

👉 Por eso, si usás `<base>`, todas las rutas relativas en `href`, `src`, y también en `<link>` serán interpretadas según esa base.

---

##  Diferencias entre `<link rel="canonical">` y `<base>`

|Característica|`<link rel="canonical">`|`<base>`|
|---|---|---|
|Función principal|Informar a buscadores cuál es la URL oficial del contenido|Definir la ruta base para todas las URLs relativas|
|Afecta el SEO|✅ Mejora el posicionamiento evitando duplicados|❌ No afecta directamente el SEO|
|Afecta rutas en el HTML|❌ No afecta ninguna URL|✅ Sí, modifica cómo se interpretan rutas relativas|
|Ubicación|Dentro de `<head>`|Dentro de `<head>`|
|Cantidad permitida|Puede haber varias si es necesario|Solo una|
|Se aplica a|Buscadores (Google, Bing, etc.)|Navegador (para resolver rutas)|

🔍 En resumen:

- `<link rel="canonical">` es solo para **SEO y buscadores**, no afecta cómo carga el sitio.
    
- `<base>` **sí afecta la carga de recursos**, modificando cómo se interpretan las rutas relativas.
    

---

##  Conclusión

- `<link>` es fundamental para conectar nuestro HTML con recursos externos como CSS, fuentes, favicons, manifestos, etc.
    
- Tiene muchos atributos útiles que determinan cómo y cuándo se cargan los recursos.
    
- `<base>` influye en cómo se interpretan las rutas relativas, incluidas las de `<link>` para organizar mejor el url.
    
- `<link rel="canonical">` y `<base>` **no hacen lo mismo**, aunque ambos van en el `<head>`:
    
    - Uno sirve para buscadores (canonical)
        
    - Otro para el navegador (base)
