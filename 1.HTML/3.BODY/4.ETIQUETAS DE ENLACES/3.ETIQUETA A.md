
## Enlaces o hipervínculos `<a>`

La etiqueta `<a>` (de “anchor” o ancla) sirve para crear **enlaces** en HTML. Estos permiten conectar el contenido actual con otro recurso, ya sea otra página, un documento, una imagen, un archivo descargable, un correo electrónico, un número telefónico, o una sección dentro de la misma página.

La etiqueta `<a>` convierte su contenido (texto, imagen, etc.) en algo **clickeable**, es decir, el usuario puede hacer clic y será redirigido a otro lugar o se activará una acción.

### Sintaxis general:

```html
<a href="URL">Contenido del enlace</a>
```

---

## Atributos más comunes de la etiqueta `<a>`

A continuación, explicamos en profundidad cada atributo junto con un ejemplo de uso.

---

### 1. `href`

**¿Qué hace?**  
Es el atributo principal y obligatorio. Define la **dirección** (URL) del recurso al que apuntará el enlace. Esta dirección puede ser:

- Absoluta: un sitio web externo.
    
- Relativa: un archivo dentro del mismo proyecto.
    
- Interna: una sección dentro del mismo documento.
    
- Especial: correos o teléfonos.
    

**Ejemplos:**

Enlace externo:

```html
<a href="https://www.wikipedia.org">Ir a Wikipedia</a>
```

Ruta relativa a un archivo local:

```html
<a href="docs/manual.pdf">Ver manual</a>
```

Correo electrónico:

```html
<a href="mailto:contacto@ejemplo.com">Escribinos</a>
```

Teléfono:

```html
<a href="tel:+541122334455">Llamar ahora</a>
```

Ancla interna:

```html
<a href="#seccion2">Ir a la sección 2</a>
```

##  Atributos de la etiqueta `<a>` con explicación profunda de sus valores

---

### 🔗 `href` – **Dirección del enlace**

**¿Qué hace?**  
Indica la dirección (URL) del recurso al que apuntará el enlace. Esta URL puede ser de diferentes tipos:

#### Tipos de valores que puede tener `href`:

1. **URL absoluta**  
    Una dirección web completa que incluye el protocolo (http o https) y el dominio.  
    Se usa para enlazar a sitios externos.
    
    ```html
    <a href="https://developer.mozilla.org">Ir a MDN</a>
    ```
    
2. **URL relativa**  
    Ruta interna dentro del mismo proyecto. Depende de la ubicación del archivo HTML actual.
    
    ```html
    <a href="contacto.html">Ir a la página de contacto</a>
    ```
    
3. **Ruta relativa ascendente** (`../`)  
    Va hacia carpetas “anteriores” a la ubicación del archivo actual.
    
    ```html
    <a href="../inicio.html">Volver al inicio</a>
    ```
    
4. **Anclas internas (fragmentos)**  
    Se refiere a una sección específica del mismo documento (identificada con `id`).
    
    ```html
    <a href="#sobre-nosotros">Ir a Sobre Nosotros</a>
    ```
    
5. **Protocolo especial – `mailto:`**  
    Abre un cliente de correo predeterminado para enviar un correo.
    
    ```html
    <a href="mailto:info@ejemplo.com">Contactanos</a>
    ```
    
6. **Protocolo especial – `tel:`**  
    Inicia una llamada telefónica desde dispositivos móviles.
    
    ```html
    <a href="tel:+541123456789">Llamar ahora</a>
    ```
    

---

###  `target` – **Dónde se abrirá el enlace**

**¿Qué hace?**  
Define el **contexto o destino** donde se abrirá el recurso al hacer clic en el enlace.

#### Valores posibles:

1. **`_self`** (por defecto)  
    Abre el enlace en la **misma pestaña o ventana** donde está el documento actual.
    
    ```html
    <a href="contacto.html" target="_self">Contacto</a>
    ```
    
2. **`_blank`**  
    Abre el enlace en una **nueva pestaña o ventana**. Muy usado para enlaces externos.
    
    ```html
    <a href="https://wikipedia.org" target="_blank">Wikipedia</a>
    ```
    
3. **`_parent`**  
    Abre el enlace en el **marco padre**. Se usa cuando estás dentro de un `iframe`.
    
4. **`_top`**  
    Abre el enlace en el **marco más externo**, útil para salir de un entorno con múltiples `iframes`.
    
5. **`nombre_de_ventana`**  
    Permite abrir el enlace en una ventana o `iframe` con un nombre específico.
    
    ```html
    <a href="pagina2.html" target="miFrame">Abrir en mi iframe</a>
    ```
    

---

###  `rel` – **Relación con el recurso enlazado**

**¿Qué hace?**  
Define el **tipo de relación** entre el documento actual y el documento al que se enlaza. Mejora el SEO, la accesibilidad y la seguridad.

#### Valores comunes y su función:

1. **`noopener`**  
    Evita que el documento nuevo tenga acceso al objeto `window.opener`, previniendo vulnerabilidades cuando se usa `target="_blank"`.
    
2. **`noreferrer`**  
    Evita que se envíe la URL de origen como referencia al destino. Aumenta privacidad.
    
3. **`nofollow`**  
    Le indica a los motores de búsqueda que **no sigan** ese enlace ni transmitan reputación (SEO). Útil en enlaces patrocinados.
    
4. **`external`**  
    Indica que el enlace lleva a una página **fuera del dominio actual**.
    
5. **`author`**  
    El documento enlazado pertenece al **autor** del contenido actual.
    
6. **`license`**  
    El enlace lleva a una **licencia** de uso del contenido.
    
7. **`prev` y `next`**  
    Se usan para indicar enlaces de **paginación** (anterior o siguiente).
    
    ```html
    <a href="pagina2.html" rel="next">Siguiente página</a>
    ```
    
8. **Combinación de valores**  
    Podés usar más de uno separándolos por espacios:
    
    ```html
    <a href="https://sitio.com" target="_blank" rel="noopener noreferrer external">Visitar sitio</a>
    ```
    

---

###  `hreflang` – **Idioma del destino**

**¿Qué hace?**  
Indica el **idioma del recurso** al que se enlaza. Esto ayuda a los motores de búsqueda a entender mejor el contenido y a los usuarios a elegir su idioma preferido.

#### Ejemplo:

```html
<a href="en/about.html" hreflang="en">About us</a>
<a href="es/acerca.html" hreflang="es">Acerca de</a>
```

#### Códigos de idioma:

- `es`: español
    
- `en`: inglés
    
- `fr`: francés
    
- `pt`: portugués
    
- `zh`: chino
    
- `ar`: árabe
    

También se pueden usar códigos regionales como `es-AR` (español de Argentina) o `en-US` (inglés de EE.UU.).

---

###  `download` – **Forzar descarga del recurso**

**¿Qué hace?**  
Hace que el navegador **descargue el archivo** en lugar de abrirlo. También permite darle un **nombre personalizado** al archivo que se va a guardar.

#### Valores posibles:

- **Sin valor**: descarga el archivo con su nombre original.
    
- **Con valor**: descarga el archivo y lo renombra como se indique.
    

#### Ejemplos:

```html
<a href="documento.pdf" download>Descargar documento</a>
<a href="documento.pdf" download="Resumen_2025.pdf">Descargar como "Resumen_2025.pdf"</a>
```

**Nota importante:**  
Funciona solo si el archivo está alojado en el mismo dominio o el servidor externo permite la descarga directa.

---

###  `type` – **Tipo MIME del recurso**

**¿Qué hace?**  
Informa al navegador sobre el **tipo de contenido** que se va a abrir. Aunque no es obligatorio, puede ayudar a tratar el recurso de forma apropiada.

#### Valores más comunes:

- `text/html`: documento HTML
    
- `application/pdf`: archivo PDF
    
- `image/png`: imagen PNG
    
- `image/jpeg`: imagen JPG
    
- `application/zip`: archivo comprimido ZIP
    

#### Ejemplo:

```html
<a href="archivo.pdf" type="application/pdf">Ver archivo PDF</a>
<a href="foto.png" type="image/png">Ver imagen</a>
```
