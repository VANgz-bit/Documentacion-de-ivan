
## **Enlaces y estructura de una URL**

Para crear enlaces en una página web, usamos la etiqueta `<a>`, que nos permite conectar una parte del documento con otra página, documento, sección o recurso.  
Sin embargo, antes de ver cómo se usa esa etiqueta, es fundamental comprender qué es una **URL**, ya que es el elemento que indica **a dónde se va a dirigir el enlace**.

---
## **¿Qué es una URL?**

La palabra **URL** proviene del inglés _Uniform Resource Locator_, que se traduce como “Localizador Uniforme de Recursos”.  
Una URL es una **dirección única** que permite acceder a un recurso específico en Internet.  
Ese recurso puede ser una página HTML, una imagen, un archivo PDF, un video, un documento CSS, un archivo JavaScript, entre muchos otros.

En la web, todo se localiza a través de URLs.  
Sin ellas, no podríamos navegar entre sitios, cargar archivos, o acceder a bases de datos externas.

Una URL cumple dos funciones clave:

1. **Identifica un recurso en la web**
    
2. **Indica cómo acceder a ese recurso**
    

---
## **Estructura general de una URL**

Una URL válida tiene un formato determinado, que podemos dividir en varias partes.  
Conocer bien esta estructura nos permite construir enlaces correctos y entender cómo funcionan los recursos enlazados.

La estructura básica es:

```txt
protocolo://dominio/ruta/pagina?parametros#ancla
```

A continuación, analizamos en detalle cada parte:

---
## **1. Protocolo**

El **protocolo** es la primera parte de la URL, y define **el tipo de comunicación** entre el navegador y el servidor.  
Determina cómo se envían y reciben los datos.

### Protocolos más comunes:

|Protocolo|Significado|
|---|---|
|`https://`|Protocolo seguro de transferencia de hipertexto. Usa cifrado (SSL/TLS) para proteger los datos.|
|`http://`|Igual que el anterior, pero sin cifrado. Los datos viajan en texto plano, lo que representa un riesgo en conexiones públicas o inseguras.|
|`ftp://`|Protocolo de transferencia de archivos. Se utiliza para subir y bajar archivos entre servidores.|
|`mailto:`|Permite crear enlaces para abrir el cliente de correo predeterminado y enviar un email.|
|`tel:`|Se usa para marcar números telefónicos directamente desde dispositivos móviles.|
|`whatsapp://`|Permite abrir conversaciones en WhatsApp desde el navegador (si está instalado).|

> **Importante:** Si no indicás el protocolo en una URL externa, algunos navegadores podrían no interpretar correctamente el enlace.

### ¿Qué significa usar `//` al principio?

Cuando escribís una URL como `//dominio.com/archivo.css`, sin especificar `http` ni `https`, el navegador toma **el mismo protocolo que el del documento actual**.  
Esto se llama **protocolo relativo**.

- Si estás en una página `https://`, tomará `https://dominio.com/archivo.css`
    
- Si estás en `http://`, tomará `http://dominio.com/archivo.css`
    

Se utiliza mucho para evitar errores de contenido mixto y tener compatibilidad entre entornos seguros e inseguros.

---
## **2. Dominio**

Después del protocolo viene el **dominio**, que es la dirección del servidor donde está alojado el recurso.  
El dominio es lo que el usuario suele escribir para entrar a un sitio web (por ejemplo: `google.com` o `mipagina.dev`).

Un dominio puede dividirse en tres partes:

- **Subdominio**: Parte opcional. El más común es `www`, pero puede ser cualquier cosa como `blog`, `tienda`, etc.
    
- **Nombre de dominio**: Es el nombre principal del sitio, elegido por quien registra el dominio.
    
- **TLD (Top-Level Domain)**: Es el dominio de nivel superior, como `.com`, `.org`, `.net`, etc.
    

### Ejemplo:

```txt
www.ejemplo.com
```

- `www` → subdominio
    
- `ejemplo` → nombre del dominio
    
- `.com` → TLD
    

### Tipos de TLD:

|TLD|Uso principal|
|---|---|
|`.com`|Comercial. El más utilizado a nivel mundial.|
|`.net`|Red o Internet. Segunda opción más común.|
|`.org`|Organizaciones sin fines de lucro, aunque su uso es libre.|
|`.es`|Sitios de España.|
|`.com.ar`|Sitios de Argentina con orientación comercial.|
|`.dev`|Enfocado en desarrolladores y sitios de programación.|
|`.io`|Muy usado en tecnología. Aunque pertenece al territorio del Océano Índico, es popular por su relación con "input/output".|
|`.me`|Popular para portafolios personales. Pertenece a Montenegro.|
|`.blog`, `.viajes`, `.store`|TLDs modernos, específicos para diferentes categorías.|

### ¿Cómo se obtiene un dominio?

Para usar un dominio necesitás **comprarlo** a través de un registrador (empresa que vende dominios).  
Algunos ejemplos confiables son: DonDominio, Namecheap, GoDaddy, PorkBun.

Los precios dependen del TLD elegido, la demanda del nombre, y la duración del contrato (1 año, 2 años, etc.).

---

## **3. Ruta (Path)**

La ruta indica **la ubicación exacta del recurso** dentro del servidor.  
Es similar a la ruta de carpetas en tu computadora.

Una ruta puede incluir:

- **Directorios (carpetas):** separados por `/`
    
- **Documento final:** el archivo que se va a cargar (puede ser una página HTML, una imagen, un PDF, etc.)
    

### Ejemplo:

```txt
https://midominio.com/curso/html/introduccion.html
```

- `/curso/html/` → estructura de carpetas
    
- `introduccion.html` → archivo final
    

### ¿Qué pasa si no se indica un archivo?

Si la URL termina en una carpeta sin nombrar un archivo específico, el servidor busca un archivo llamado `index.html` o `index.htm` por defecto.

Ejemplo:

```txt
https://mipagina.com/blog/
```

Podría mostrar automáticamente:

```txt
https://mipagina.com/blog/index.html
```

Esto es configurable desde el servidor.

---

## **4. Parámetros (Query Strings)**

Los **parámetros** son datos que se añaden a la URL para enviar información extra.  
Son muy usados en formularios, motores de búsqueda o enlaces generados por JavaScript.

### Sintaxis:

- Comienzan con `?`
    
- Cada parámetro tiene un nombre y un valor: `nombre=valor`
    
- Si hay más de uno, se separan con `&`
    

### Ejemplo con Google:

```txt
https://www.google.com/search?q=gato
```

- `q=gato` significa que estás buscando la palabra “gato”.
    

Más complejo:

```txt
https://www.google.com/search?q=gato&tbm=isch
```

- `tbm=isch` indica que se desea ver los resultados en **imágenes**.
    

### Uso común:

- Enviar datos al servidor sin recargar la página
    
- Filtrar contenido
    
- Identificar usuarios, sesiones, categorías, etc.
    

---

## **5. Anclas**

Una **ancla** permite saltar directamente a una sección específica de una página HTML.

Se indica agregando `#` seguido del `id` del elemento al que queremos llegar.

### Ejemplo:

```txt
https://mipagina.com/tutorial.html#contacto
```

Esto hará que el navegador **haga scroll automático** hasta el elemento con:

```html
<div id="contacto">...</div>
```

Muy útil en documentos largos o menús de navegación internos.

---

## **6. Fragmentos de texto**

Es una funcionalidad moderna llamada **scroll to text fragments**.  
Permite que el navegador resalte automáticamente un texto específico dentro de la página.

Se usa así:

```txt
#:~:text=palabraInicial,palabraFinal
```

### Ejemplo:

```txt
https://es.wikipedia.org/wiki/Gato#:~:text=El%20gato%20doméstico,compañía
```

Esto hará que el navegador:

- Busque la frase “El gato doméstico” hasta “compañía”
    
- Haga scroll hasta esa parte
    
- Resalte ese fragmento
    

Sirve mucho cuando se comparte un enlace con texto destacado directamente.

---

## **7. Estados de un enlace**

Los enlaces tienen distintos **estados visuales** que cambian según la interacción del usuario.  
Podemos controlarlos desde CSS usando pseudoclases.

|Estado|Pseudoclase CSS|Descripción|
|---|---|---|
|Enlace no visitado|`:link`|Aparece normalmente azul y subrayado.|
|Enlace visitado|`:visited`|Se vuelve lila o púrpura, indicando que ya fue clickeado.|
|Enlace activo|`:active`|Se aplica mientras el usuario mantiene presionado el clic.|

### Recomendación:

Es mejor **mantener estilos familiares** para los enlaces.  
Los usuarios están acostumbrados a verlos en azul o subrayados, y si eso cambia demasiado, pueden no darse cuenta de que son clickeables.
