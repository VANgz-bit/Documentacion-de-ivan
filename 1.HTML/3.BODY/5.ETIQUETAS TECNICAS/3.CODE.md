Cuando desarrollamos una página web, a veces necesitamos mostrar **fragmentos de código** directamente, por ejemplo, para enseñar o documentar. Sin embargo, si escribimos código tal cual en el HTML, el navegador lo **interpretará y ejecutará**, en lugar de mostrarlo como texto. Para solucionar esto, utilizamos etiquetas específicas como `<code>` y `<pre>`.

---

### 📌 La etiqueta `<code>`

La etiqueta `<code>` le indica al navegador que el contenido dentro de ella es un **fragmento de código** y debe mostrarse **visualmente como tal**, es decir, con una **fuente monoespaciada** (como en la terminal o editores de código). También le da un **significado semántico**, ayudando a que tecnologías como lectores de pantalla entiendan que se trata de código.

#### ✅ Uso correcto

```html
<p>
  El fragmento de código JavaScript <code>console.log("Hola")</code> sirve para
  mostrar un mensaje por la consola del navegador.
</p>
```

Este fragmento se mostrará con tipografía de código (monoespaciada), sin que el navegador intente ejecutarlo.

#### 📝 Notas:

- `<code>` se usa para fragmentos **cortos**, dentro de un párrafo o línea.
    
- Podés personalizar su estilo con CSS para que destaque más:
    

```css
code {
  color: blue;
  font-weight: bold;
}
```

---

### 📌 La etiqueta `<pre>`

`<pre>` significa **preformatted** (preformateado). Esta etiqueta muestra el contenido tal cual se escribió, **respetando los espacios y saltos de línea**. Es útil para mostrar **bloques de código largos**.

#### ✅ Ejemplo combinado con `<code>`:

```html
<pre>
  <code class="language-javascript">
    function alertMessage() {
      return "ALERTA: El lector es demasiado atractivo.";
    }
  </code>
</pre>
```

Este bloque de código mantiene su indentación y estructura original.

#### 📝 Notas:

- `<pre>` se puede usar solo, pero combinarlo con `<code>` da más **precisión semántica** y permite aplicar estilos o resaltar sintaxis más fácilmente.
    
- Muchas librerías de resaltado de código como **Prism.js** o **Highlight.js** usan esta estructura para aplicar colores al código.
    

---

### 🧩 ¿Cómo mostrar etiquetas HTML como texto?

Si querés mostrar código HTML sin que el navegador lo interprete, debés **escapar los caracteres especiales**, reemplazando:

- `<` por `&lt;`
    
- `>` por `&gt;`
    

#### ✅ Ejemplo:

```html
<p>
  La etiqueta <code>&lt;code&gt;</code> se utiliza en HTML para
  indicar fragmentos de código cortos, en línea.
</p>
```

Esto mostrará literalmente: `<code>`, sin que el navegador lo interprete como una etiqueta real.

---

### ⚙️ Herramientas útiles

En muchos entornos (como Markdown, editores de documentación o algunos CMS), ya vienen herramientas que **convierten automáticamente** los caracteres especiales a sus entidades HTML (`<` a `&lt;`, `>` a `&gt;`), haciendo más cómodo el proceso.

También hay **frameworks o librerías** que permiten:

- **Resaltar código** con colores (syntax highlighting)
    
- Añadir numeración de líneas
    
- Copiar el código con un botón
    
- Mostrar íconos del lenguaje usado
