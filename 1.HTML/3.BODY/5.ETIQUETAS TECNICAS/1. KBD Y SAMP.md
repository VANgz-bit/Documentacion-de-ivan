
### Entrada y salida de datos en HTML: etiquetas `<kbd>` y `<samp>`

En el desarrollo de documentos HTML, a menudo se necesita representar de forma clara cuándo un usuario debe proporcionar información al sistema o cuándo el sistema devuelve una respuesta o mensaje al usuario. Esta representación puede formar parte de una guía de uso, un manual técnico o simplemente una indicación dentro de una página web.

### Entrada de datos por parte del usuario

La entrada de datos es el proceso mediante el cual un usuario interactúa con una aplicación o sistema, normalmente para ejecutar una acción o modificar información. En la mayoría de los casos, esta entrada se realiza a través del teclado, aunque también pueden usarse otros dispositivos o métodos, como comandos de voz, pantallas táctiles u otros periféricos de entrada.

#### La etiqueta `<kbd>`

La etiqueta `<kbd>` (abreviación de _keyboard_, teclado en inglés) se utiliza en HTML para representar visualmente una entrada de texto que el usuario debe realizar utilizando el teclado u otro medio de entrada. Su función principal es semántica: indica al navegador, a los lectores de pantalla y a los motores de búsqueda que el texto dentro de la etiqueta corresponde a una acción realizada por el usuario, no a un simple fragmento de contenido.

El uso más habitual es el de mostrar combinaciones de teclas que el usuario debe presionar para realizar alguna acción en el sistema.

**Ejemplo:**

```html
<p>
  Para abrir una nueva pestaña en el navegador, pulsa la combinación de teclas <kbd>CTRL</kbd>+<kbd>T</kbd>.
</p>
```

Este fragmento indica al lector que debe presionar las teclas `CTRL` y `T` al mismo tiempo para abrir una nueva pestaña. Aunque la etiqueta por defecto suele mostrar el contenido con un estilo monoespaciado, se recomienda aplicar estilos personalizados con CSS para que visualmente se asemejen a teclas físicas y se integren mejor en la interfaz del sitio.

#### Estilización con CSS

```css
kbd {
  padding: 4px 6px;
  border: 1px solid #ccc;
  border-radius: 3px;
  background-color: #f9f9f9;
  font-family: monospace;
}
```

Este estilo simula el aspecto de una tecla, mejorando la experiencia visual del usuario.

### Salida de datos desde la máquina

Cuando una aplicación o sistema devuelve un mensaje, resultado o respuesta al usuario, se está generando una **salida de datos**. Representar esta información correctamente es importante para que el usuario comprenda el comportamiento del sistema, sobre todo en contextos como depuración, mensajes de error o respuestas automáticas.

#### La etiqueta `<samp>`

La etiqueta `<samp>` (de _sample output_, es decir, “salida de ejemplo”) se utiliza para mostrar mensajes o resultados que una computadora, navegador o programa pueden devolver al usuario. Al igual que `<kbd>`, su función es semántica, ayudando a identificar que el contenido mostrado no fue escrito por el usuario ni es contenido regular del documento, sino un resultado generado por el sistema.

**Ejemplo:**

```html
<p>
  <strong>Internet Explorer</strong> mostró un mensaje: <samp>Error: Todo ha ido bien</samp>.
</p>
```

Este ejemplo ilustra cómo podría mostrarse un mensaje generado por un navegador o programa, indicando que el contenido dentro de `<samp>` es una respuesta o salida del sistema.
