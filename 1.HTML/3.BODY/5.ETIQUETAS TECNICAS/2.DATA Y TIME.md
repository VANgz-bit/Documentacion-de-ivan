
###  Representar fechas, horas y datos en HTML

En un documento web podríamos necesitar representar adecuadamente una **fecha**, **hora** u otra información relacionada con el **tiempo**. Pero hay que tener en cuenta varios aspectos importantes:

1. Que la información sea **fácil de entender para los humanos**.
    
2. Que **las máquinas puedan leerla e interpretarla** correctamente.
    
3. Que **no haya ambigüedad** en la representación.
    

Para lograr esto, usamos etiquetas específicas como `<time>`, `<data>` y `<var>`.

---

### 🔹 La etiqueta `<time>`

La etiqueta `<time>` sirve para representar fechas, horas o cualquier valor relacionado con el **tiempo**. Su objetivo es mostrar un texto legible para el usuario, pero también tener una forma que una **máquina pueda interpretar**.

**Ejemplo simple:**

```html
<p>Mañana es día <time>2024-12-25</time>.</p>
```

Esto se ve claro para una máquina, pero no tan amigable para una persona.

---

### 🔸 El atributo `datetime`

Podemos mejorar la experiencia del usuario y al mismo tiempo facilitar la lectura por parte de una máquina usando el atributo `datetime`.

Este atributo permite definir una **fecha u hora en un formato estándar ISO 8601**, que es ideal para máquinas.

**Ejemplo recomendado:**

```html
<p>
  <time datetime="2024-01-04">4 de Enero de 2024</time>
</p>
```

- **Lo que ve el usuario:** `4 de Enero de 2024`
    
- **Lo que interpreta la máquina:** `2024-01-04`
    

Esto evita confusiones por los diferentes formatos de fecha que existen en cada país o idioma.

---

###  Formatos válidos para `datetime`

El valor de `datetime` debe seguir un **orden de mayor a menor unidad de tiempo** (año → mes → día → hora → minuto → segundo).

#### ➤ Ejemplos comunes:

```plaintext
2024-12                      → Año y mes
2024-12-25                   → Año, mes y día
2024-12-25T22:35             → Fecha + hora y minutos
2024-12-25T22:35:42          → Fecha + hora, minutos y segundos
2024-12-25T22:35:42.729      → Con milisegundos
2024-12-25T22:35:42.729Z     → Con zona horaria (Z = UTC)
2024-12-25T22:35:42.729-0800 → Zona horaria negativa
2024-12-25T22:35:42.729+08:00→ Zona horaria positiva
```

#### ➤ Casos más especiales (pero válidos):

```plaintext
22:35:42          → Solo hora (sin fecha)
12-25             → Solo mes y día
PT5H23M8S         → Duración (5 horas, 23 minutos, 8 segundos - ISO 8601)
5h 24m 8s         → Duración (formato más informal)
```

---

### 🔹 La etiqueta `<data>`

Sirve para **representar datos generales** que pueden tener un significado especial para las máquinas, distinto del texto que ve el usuario.

Se usa con el atributo `value`, que contiene el valor que puede interpretar una máquina, mientras que el contenido de la etiqueta es el texto que ve el usuario.

**Ejemplo:**

```html
<p>
  <data value="ESDLA3">El retorno del Rey</data>
</p>
```

- `"El retorno del Rey"` lo ve el usuario.
    
- `"ESDLA3"` es el código o dato asociado para que lo lea un sistema (por ejemplo, una base de datos).
    

---

### 🔹 La etiqueta `<var>`

Se utiliza para **representar variables** (matemáticas o de programación).

El navegador suele mostrar el contenido en una **tipografía monoespaciada** para indicar que se trata de una variable.

**Ejemplo:**

```html
<p>
  Para obtener el resultado final, lo único que tiene que hacer el usuario
  es multiplicar la variable <var>x</var> por 42.
</p>
```

En este caso, `<var>x</var>` señala que `x` es una variable.
