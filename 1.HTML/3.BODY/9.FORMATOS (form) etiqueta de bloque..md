
En los formularios nos encontramos con muchas etiquetas y atributo
en los cuales nos basamos algunas carteristas para el formulario que deseamos que el usuario complete a los cual nos basamos en las siguientes básicos.

`<Input>` usamos este atributo para señalar el dato que queremos poner,
dentro de el usamos el atributo **"type**" para señalar el tipo de información que queremos agregar, los cuales podemos ver 23 etiquetas que son:

**TEXT** : nos permite tipear texto
**COLOR** : nos permite seleccionar el color que escojamos
**SUBMIT**: que no deja poner el botón de enviar
**EMAIL**: nos deja un apartado para colocar el formato email.
**PASSWORD** : abre otra casilla para colocar la contraseña.
**CHECKBOX** : nos sirve para colocar una casillita para tachar o marcar una condición.
**RADIO**: abre otra casillita igual al **checkbox** pero esta vez el usuario puede seleccionar una opción o varias.
**FILE**: sirve para que el usuario pueda subir o enviar un archivo.
**DATE**: es la casilla para poner las fechas de cumpleaños

Tenemos mas atributos importantes para agregar:

**REQUIRED:** este atributo hará que la casilla que creamos sea completada como requisito. a este atributo se le puede agregar el **MINlEGTH="**" que indica al sistema que lo mínimo de caracteres.
**NAME**: este atributo ubica el nombre del campo en donde estoy ingresando los dato
ej: **name="username"**. estoy indicando que ese campo se llama username y que ahí va los nombres de los usuarios. Por ende será mas fácil que el servidor encuentre los datos.
**PLACEHOLDER**: Este atributo indica un mensaje como ejemplo de lo que hay que escribir.
**VALUE**: guarda los datos del usuario en el servidor.


# TEXTAREA Y LABELS

Los **TEXTAREA** son una etiqueta que funciona para que el usuario pueda escribir renglones o párrafos de texto con o sin limites de caracteres, se suelen usar en formularios para que las personas puedan redactar consultas quejas o demás. asi como el input el textarea tiene propiedades que agregar tanto en css y html:

**HTML:**

El textarea tiene propiedades que permiten ciertas funciones dentro de la misma, cuales como:
**Maxlenght:** definimos cuantos caracteres queremos que tenga.
**Minlenght**: es el mínimo de caracteres a ingresar.
**Rows**: es la cantidad de líneas o filas que le vamos a permitir escribir al usuario. e esta forma nos ahorramos el min-height.
**Cols**: define el numero de columnas que tendrá de ancho.
**Placeholder**: es un texto que desaparece cuando escribimos y que usualmente indica que escribir en el campus.

Existen atributos que no necesitan valor a definir que funcionan como true o false, si lo agregamos cuenta como TRUE y si no lo hacemos como FALSE, se definen por su existencia. Se los llaman:
**ATRIBUTOS BOOLEANOS**:
**Readonly**: se usa para que solo se pueda leer, no podes editar el texto puesta.
**Disabled:** este atributo desactiva el campo de texto.
**Required**: hace que el campo sea obligatorio responder.
**Autofocus**: hace foco automático al cargar la pagina.

**CSS**:

 El textarea tiene por defecto que el usuario pueda extender el marco en el eje vertical y horizontal y esto arruina el estilo del formato para ello usamos:
 
 **Resize: both;** esto nos sirve para que solo se mueva en el eje x, sin embargo podemos aplicar también:
 **Resize: vertical:** que solo se mueve de en el eje y, osea arriba y abajo.

Si queremos que el cuadro del textarea ya este predefinido sin que el usuario lo modifique
definimos su width y su height, usando lo mínimo y máximo de ambos. Osea que si queremos que verticalmente tenga un tamaño apropiado definimos que el height tenga un valor mínimo- y un valor máximo de altura.
Si queremos que el textarea no se agrande pero queremos que siga teniendo las propiedades del height usamos:
 
 **Resize: none:** que hace que no se expanda el cuadro pero en lugar de eso podamos movernos dentro del  la misma con una barra de scroll.
Hay una propiedad que esta en fase experimental por parte de los desarrolladores de css working que es :
**Form-sizing:** esta propiedad permite que el textarea se adapte a los párrafos sin necesidad de usar min y max width o height. En resumen hace que textarea aumente su tamaño en base a la cantidad de texto. Por el momento no esta disponible aun.

from{
	width:100%;
	resize: vertical;
	min-height: 300px;
	max-height:600px;
}

Definimos cuanto tendría mínimamente el cuadro unos 300pxy definimos que su limite máximo sea de 600px.


## LABELS

Los labels son elementos de líneas que etiquetan  individualmente un elemento de input, en otras palabra el label esta enganchado con un elemento y al interactuar con el label esta nos lleva con el dicho elemento.
¿Ahora como enlazamos el label con el input? , dela siguiente forma :

`<form>`
	`<label for="nombre">nombre completo</label>`
		`<div class="form-input">`
			`<input id="nombre" type="text" placeholder="John Doe">`
		`</div>`
`</form>`

Los enlazamos con el selector "**id**", le ponemos una clase unica (id) al input y al label le damos una especie de clase que se llama "**for**" (proveniente de) y asi se entrelazarían ambos.En tonces cada que aprieten el titulo NOMBRE COMPLETO , esto nos hara focus y nos llevara al input de texto llamada nombre completo (John Doe)

Otra forma seria metiendo el input dentro del label y asi nos ahorraríamos usar la clase única "id"

`<form>`
		`<div class="form-input">`
			`<label>`
				`<input id="nombre" type="text" placeholder="John Doe">`
			`</label>`
		`</div>`
`</form>`


# SELECT, DATALIST Y OPTION

Los **SELECT** son un tipo de campo de entrada que nos permite seleccionar una opción de entre varias opciones, los **DATALIST** son una especie de input que autocompleta con opciones que predefinimos.

**EJEMPLO SELECT:**

`<FORM>`
		`<div>`
			`<label>Tipo de usuario`
				`<select>`
					`<option value="user1">seguidor</option>`
					`<option value="user2">emprendedor</option>`
					`<option value="user3">empleado</option>`
				`<select>`
			`</label>`
		`</div>`
`</form>`

Acá pusimos ciertas opciones para que el usuario pueda seleccionar que clase de persona es, el **value** lo usamos para que en java lo podamos ejecutar mejor y darle mejores composiciones. 

**EJEMPLO DATALIST:**

`<div class="form-input">`

	`<label>Referente`

                `<input list="Referente">`

                `<datalist id="Referente">`

                    `<option >follower</option>`

                    `<option >empleado</option>`

                    `<option >cliente</option>`

                `</datalist>`

            `</label>`

        `</div>`

En este caso para trabajar con el datalist necesitamos un input y el input va a llevar un "**list**" con el nombre del  "**id**" que le pusimos a datalist. Tanto en select como datalist podemos no elijir usar valué.

# FIELDSET Y LEGEND

Los fieldset y legend sirven para empaquetar o etiquetar los  elementos de un formulario, si tenemos un grupo de elementos que queremos agruparlos como datos del usuario los separamos del resto con lo mencionado.

`<form>`
	`<fieldset>`
	`<legend>Informacion Basica`
	`<div>`
		`<input type="text" placeholder="Name">`
	`</div>`
	`<div>`
		`<input type="email" placeholder="email">`
	`</div>`
	`</fieldset>`
`</form>`

Aquí encerramos los in0put que contienen información básica y lo separamos de los demás tipos de elementos que contienen otro tipo de información. El **fieldset** enmarca y el **legend** se encarga de darle un titular o etiqueta.
