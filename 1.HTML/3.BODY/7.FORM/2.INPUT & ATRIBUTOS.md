# **Etiqueta `<input>` en HTML**

La etiqueta `<input>` es una de las más utilizadas en los formularios HTML porque permite **obtener datos del usuario de diferentes maneras**, dependiendo del valor de su atributo `type`.

Es una **etiqueta vacía** (sin cierre) y debe colocarse **dentro de una etiqueta `<form>`** para funcionar de forma estándar, aunque también puede manipularse mediante JavaScript si se coloca fuera del formulario.

---

## **1. Atributos generales de `<input>`**

Los siguientes atributos pueden aplicarse en la mayoría de los campos `<input>`.

---

### **1.1. `name`**

Este atributo define **el nombre del campo**. Es **clave** porque será la referencia que usaremos tanto en el servidor como en JavaScript para **identificar el valor introducido**.

- **Importancia:**  
    Sin el atributo `name`, el campo no enviará datos al backend.
    
- **Ejemplo:**
    
    ```html
    <input type="text" name="usuario">
    ```
    

En este caso, el dato ingresado se asociará al identificador `usuario` al enviarse.

---

### **1.2. `type`**

El atributo `type` indica **el tipo de dato que esperamos** y, por ende, **la funcionalidad y el comportamiento** del campo.

- **Valor por defecto:**  
    Si no se especifica, se asume `type="text"`.
    

A continuación, se explican los tipos más comunes y sus particularidades:

#### **Campos de texto y variantes**

|Tipo|Uso|Ejemplo|
|---|---|---|
|`text`|Texto libre corto, como nombres o apellidos.|`<input type="text" name="nombre">`|
|`search`|Textos destinados a búsquedas, incluye mejoras en móviles como botón de borrado rápido.|`<input type="search" name="busqueda">`|
|`tel`|Para números telefónicos, muestra teclado numérico en móviles.|`<input type="tel" name="telefono">`|
|`url`|Para direcciones web, con sugerencias de teclado `/` o `.`.|`<input type="url" name="pagina">`|
|`email`|Para direcciones de correo, valida formato y muestra teclas `@` en móviles.|`<input type="email" name="correo">`|

---

#### **Campos para contraseñas y ocultos**

|Tipo|Uso|Ejemplo|
|---|---|---|
|`password`|Oculta el texto visualmente (con puntos o asteriscos), ideal para contraseñas o PINs.|`<input type="password" name="clave">`|
|`hidden`|Campo invisible que envía datos sin mostrarse al usuario, útil para IDs o tokens.|`<input type="hidden" name="token" value="abc123">`|

---

#### **Campos para datos numéricos o específicos**

|Tipo|Uso|Ejemplo|
|---|---|---|
|`number`|Entrada de números con controles incrementales.|`<input type="number" name="edad" min="18" max="99">`|
|`range`|Barra deslizante para seleccionar un valor dentro de un rango.|`<input type="range" name="volumen" min="0" max="100">`|
|`date`|Selección de fechas.|`<input type="date" name="nacimiento">`|
|`time`|Selección de horas.|`<input type="time" name="hora">`|
|`datetime-local`|Fecha y hora combinadas.|`<input type="datetime-local" name="evento">`|
|`color`|Selector de color con una paleta visual.|`<input type="color" name="favorito">`|

---

#### **Campos para subida de archivos**

|Tipo|Uso|Ejemplo|
|---|---|---|
|`file`|Permite al usuario subir archivos, imágenes o documentos.|`<input type="file" name="foto">`|

---

### **1.3. `value`**

Define el **valor inicial** que tendrá el campo.

- **Usos comunes:**
    
    - Prellenar datos conocidos.
        
    - Establecer valores predeterminados.
        
- **Ejemplo:**
    
    ```html
    <input type="text" name="usuario" value="Invitado">
    ```
    

---

### **1.4. `form`**

Asocia el campo a un formulario específico mediante el atributo `id` de ese formulario.

- **Usos:**  
    Útil si el `<input>` está **fuera del `<form>`** pero debe enviar datos junto a él.
    
- **Ejemplo:**
    
    ```html
    <form id="registro"></form>
    <input type="text" name="usuario" form="registro">
    ```
    

---

### **1.5. `list`**

Relaciona el campo `<input>` con una lista de sugerencias creadas con `<datalist>`.

- **Ejemplo:**
    
    ```html
    <input type="text" list="opciones">
    <datalist id="opciones">
      <option value="HTML">
      <option value="CSS">
      <option value="JavaScript">
    </datalist>
    ```
    

---

### **1.6. `placeholder`**

Muestra un texto **indicativo o sugerente** que desaparece cuando el usuario escribe.

- **Ejemplo:**
    
    ```html
    <input type="text" name="usuario" placeholder="Introduce tu nombre">
    ```
    

---

### **1.7. `size`**

Define el **ancho visual del campo**, expresado en número aproximado de caracteres visibles.

- **Ejemplo:**
    
    ```html
    <input type="text" name="codigo" size="10">
    ```
    

---

### **1.8. `autocomplete`**

Controla si el navegador **guarda y sugiere valores previos** para el campo.

- **Valores:**
    
    - `on` → Activado (valor por defecto).
        
    - `off` → Desactivado.
        
- **Ejemplo:**
    
    ```html
    <input type="text" name="usuario" autocomplete="off">
    ```
    

---

### **1.9. `autofocus`**

Hace que el campo **reciba automáticamente el cursor** al cargar la página.

- **Ejemplo:**
    
    ```html
    <input type="text" name="usuario" autofocus>
    ```
    

---

## **2. Comprobación de ortografía: `spellcheck`**

El atributo `spellcheck` activa o desactiva la **revisión ortográfica** del contenido escrito en el campo.

- **Valores:**
    
    - `true` → Revisión activada.
        
    - `false` → Revisión desactivada.
        
- **Ejemplo:**
    
    ```html
    <input type="text" name="comentario" value="camion asul" spellcheck="true">
    <input type="text" name="nota" value="camion asul" spellcheck="false">
    ```
    

En el primer campo, el navegador marcará el error; en el segundo, no.

---

## **3. Personalización visual con CSS**

Los campos `<input>` pueden **estilizarse** fácilmente con CSS mediante propiedades comunes como `width`, `height`, `border`, `background`, `font-size` y `color`.

En el caso de los campos `type="search"`, además se pueden personalizar elementos internos como el botón de limpieza de texto.

**Ejemplo:**

```css
input[type="search"].custom {
  height: 40px;
  background: #333;
  border: 3px solid gold;
  border-radius: 8px;
  color: #fff;
  font-size: 1.1rem;

  &::-webkit-textfield-decoration-container {
    background: #000;
    padding: 0.5rem 1rem;
  }

  &::-webkit-search-cancel-button {
    filter: drop-shadow(0 0 1px white) drop-shadow(0 0 5px white);
  }
}
```
