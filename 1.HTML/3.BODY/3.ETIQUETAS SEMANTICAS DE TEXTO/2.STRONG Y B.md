
En ciertos contextos, dentro de un párrafo o fragmento de texto, es posible que necesitemos **destacar algunas palabras clave** por su **mayor relevancia dentro del discurso**. Es común que este tipo de énfasis se represente visualmente mediante **negrita**, pero es fundamental **distinguir entre la presentación gráfica y el significado semántico** del contenido.

### Diferencias entre apariencia y semántica

Consideremos las siguientes situaciones:

1. Un texto puede estar en **negrita** y ser **importante**.
    
2. Un texto puede **no estar en negrita** y **no ser importante**.
    
3. Un texto puede estar en **negrita** y **no ser importante**.
    
4. Un texto puede **no estar en negrita** y **sí ser importante**.
    

Las dos primeras situaciones son las más habituales: el estilo (negrita) coincide con la importancia real del texto. Sin embargo, los dos últimos casos son menos comunes, pero completamente válidos, especialmente en un enfoque **semántico y accesible** del desarrollo web.

> El **punto 4** es clave: un texto puede **no tener un estilo llamativo** y aun así ser **relevante a nivel semántico**.  
> Esa es precisamente la función de la etiqueta `<strong>`.

---

## ¿Qué es la etiqueta `<strong>`?

La etiqueta `<strong>` es una etiqueta **en línea** (inline) de HTML que sirve para marcar un fragmento de texto como **especialmente importante** dentro de su contexto. Su propósito **no es visual**, sino **semántico**: expresa que el contenido que encierra tiene un **mayor peso o valor informativo** que el texto que lo rodea.

Aunque por defecto la mayoría de los navegadores muestran `<strong>` en **negrita**, esto es simplemente una representación gráfica por convención. El verdadero valor de esta etiqueta radica en su **significado estructural** dentro del documento HTML.

---

## Ejemplo básico

```html
<p><strong>Manz</strong> es un streamer de desarrollo de Twitch.</p>
```

En este caso, la palabra “Manz” ha sido marcada como **relevante** dentro de la oración. Podés modificar su estilo con CSS (quitarle la negrita, cambiar el color, etc.), pero seguirá siendo reconocida como importante por navegadores, lectores de pantalla y motores de búsqueda.

```css
strong {
  font-weight: normal;
  color: blue;
}
```

---

## Impacto en SEO y accesibilidad

Aunque `<strong>` no tiene un impacto directo en el posicionamiento SEO, **sí aporta valor estructural al contenido**, ayudando a los motores de búsqueda a identificar conceptos clave dentro de un texto.

Además, los **lectores de pantalla** y otras tecnologías de asistencia **reconocen esta etiqueta y alteran su pronunciación o entonación** para comunicar su importancia al usuario.

Esto la convierte en una herramienta fundamental en **accesibilidad web**.

---

## Uso combinado con otras etiquetas

La etiqueta `<strong>` puede emplearse dentro de otras etiquetas estructurales para reforzar aún más la semántica. Por ejemplo, dentro de un título:

```html
<h1>Etiquetas HTML <strong>semánticas</strong></h1>
```

Aunque el texto ya es visualmente destacado por ser un encabezado, se está indicando que la palabra “semánticas” tiene un **peso adicional** dentro del título. Este tipo de uso es especialmente útil para **transmitir énfasis de forma estructurada**, sin depender exclusivamente del diseño visual.

---

## Anidamiento de etiquetas `<strong>`

Una característica avanzada es que **podés anidar varias etiquetas `<strong>` para aumentar el nivel de importancia relativa** entre diferentes fragmentos de texto.

```html
<p>Este <strong>tema</strong> es <strong><strong>muy importante</strong></strong>.</p>
```

En este ejemplo, la expresión “muy importante” está marcada como **más relevante** que la palabra “tema”. Esta estructura permite establecer distintos **niveles de prioridad semántica** dentro del mismo bloque textual.

### Estilizado visual de diferentes niveles

Podés aprovechar el anidamiento para diferenciar los niveles visualmente mediante CSS:

```css
strong {
  color: black;
}

strong > strong {
  color: red;
}
```

Con esto, el texto más importante (el `<strong>` anidado dentro de otro `<strong>`) se verá en rojo, mientras que el de importancia media se mostrará en negro.

---

## Consideraciones de uso

- Evitá usar `<strong>` solo por diseño. Si lo que buscás es **negrita sin valor semántico**, usá la etiqueta `<b>` o estilos CSS.
    
- Usá `<strong>` con criterio: **cuando realmente querés comunicar al navegador, lector o sistema que ese texto tiene un valor especial**.
    
- Podés combinarlo con otras etiquetas semánticas como `<em>`, `<mark>`, o incluso `<span>` con clases, para reforzar distintos niveles de significado.

### La etiqueta `<b>`

En versiones anteriores de HTML, existía una etiqueta `<b>` que se utilizaba para literalmente marcar fragmentos de texto en negrita. Era similar al botón **B** (_Bold_) que suelen tener los procesadores de texto como Word o Google Docs.

En HTML5 se reforzó el concepto de semántica, y se derivó el cambio estético de un fragmento de texto exclusivamente a CSS, por lo que la etiqueta HTML debía proveer un concepto semántico. Por esa razón se reemplazo el uso de `<b>` por `<strong>`. Sin embargo, en HTML5 sigue existiendo la etiqueta `<b>`, aunque su utilización es ligeramente diferente.

La etiqueta `<b>` se utiliza hoy en día para «fines utilitarios», es decir, para marcar como palabras clave pero sin darle ninguna importancia adicional ni ningún tono alternativo. La diferencia respecto a `<strong>` es que `<b>` no marca como importante el fragmento de texto.
