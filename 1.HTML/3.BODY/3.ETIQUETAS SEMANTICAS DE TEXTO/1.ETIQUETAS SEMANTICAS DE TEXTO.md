
Hemos hablado de que en HTML5 existen muchas etiquetas, y la mayoría se agrupan en dos categorías principales:

Las etiquetas que actúan sobre fragmentos de texto, aportándoles un significado semántico específico (para identificar su función, importancia o tono en el contenido).

Las etiquetas que se utilizan para organizar o contener otras etiquetas dentro de un bloque, estructurando así la información de la página.

En este apartado nos vamos a centrar en el primer grupo: las etiquetas que se aplican a fragmentos concretos de texto para expresar **su valor semántico** (esto es lo más importante) y en algunos casos también por **motivos visuales** (como resaltar o diferenciar).

En el siguiente ejemplo de código, la etiqueta `<p>` representa una etiqueta de agrupación (porque encierra un bloque completo de contenido), mientras que la etiqueta `<strong>` está orientada a dar significado específico al texto que encierra, por lo que pertenece al grupo de semántica de texto:

```html
<p>Hola, esto es un pequeño <strong>ejemplo</strong>.</p>
```

Es importante comprender que las etiquetas de agrupación **casi siempre son etiquetas de bloque**, lo que significa que ocupan todo el ancho disponible y generan una nueva línea. Por el contrario, las etiquetas de semántica textual suelen ser **etiquetas en línea**, es decir, no rompen el flujo del texto y se comportan como parte de una oración. Comprender esta diferencia te va a permitir identificar más fácilmente su uso y comportamiento en el diseño de una página.

Si todavía no entendés bien la diferencia entre **bloque** e **inline**, te recomiendo que consultes la documentación de LenguajeCSS, especialmente en lo referido a la propiedad `display` de CSS, donde esto se explica a fondo.

### Fragmentos de texto

Existen etiquetas HTML que se aplican directamente sobre partes específicas del texto para darles **un valor semántico especial**. Esto no se hace solo con fines visuales, sino principalmente para dejar claro el **sentido del contenido**, que puede ser importante, destacado, una cita, un error, un acrónimo, etc. A veces, además del significado, se consigue un efecto visual asociado:

|Etiqueta|Descripción|Ejemplo|Más info|
|---|---|---|---|
|`<a>`|Representa un enlace; permite dirigir al usuario a otra URL o documento.|manz.dev|`<a>`|
|`<strong>`|Indica un fragmento de texto que es importante o relevante.|Texto importante|`<strong>`|
|`<em>`|Señala un fragmento que se quiere enfatizar dentro de una oración.|Texto enfatizado|`<em>`|
|`<mark>`|Resalta un texto como si estuviera subrayado con marcador (resaltador).|Texto marcado|`<mark>`|
|`<b>`|Destaca un texto visualmente pero **sin otorgarle significado semántico**.|Texto destacado|`<b>`|
|`<i>`|Muestra un texto en cursiva, indicando un tono diferente (pensamientos, etc).|Texto alternativo|`<i>`|
|`<u>`|Subraya texto para nombres propios o palabras incorrectas intencionadas.|«Manz»|`<u>`|
|`<s>`|Representa contenido que ya no es relevante o contiene errores.|Errores o inexactitudes|`<s>`|
|`<span>`|Contenedor genérico sin significado semántico, útil para aplicar estilos o scripts.|Fragmento de texto|`<span>`|

Estas son algunas de las etiquetas más comunes para trabajar con fragmentos de texto en HTML, aunque no son las únicas. Más adelante veremos algunas adicionales que también tienen una función semántica.

> Nota: en versiones antiguas de HTML, etiquetas como `<b>`, `<i>` o `<u>` tenían funciones diferentes o menos definidas semánticamente.

### Fragmentos con significado

También existen otras etiquetas semánticas que aportan un significado especial a los fragmentos de texto, aunque se utilizan con menos frecuencia. Sin embargo, siguen siendo parte del conjunto de etiquetas semánticas de HTML y resultan útiles en casos más específicos:

|Etiqueta|Descripción|Ejemplo|Más info|
|---|---|---|---|
|`<sup>`|Muestra texto como superíndice (elevado).|24|`<sup>`|
|`<sub>`|Muestra texto como subíndice (bajo).|H2O|`<sub>`|
|`<small>`|Marca anotaciones o comentarios secundarios (menos importantes).|Pequeña puntualización|`<small>`|
|`<cite>`|Representa el nombre de una obra creativa (libros, películas, videojuegos).|Videojuego: _To the Moon_|`<cite>`|
|`<q>`|Muestra una **cita corta** dentro de una frase (con comillas automáticas).|“Frase”|`<q>`|
|`<abbr>`|Define una abreviatura o acrónimo, y puede usarse con el atributo `title`.|JS|`<abbr>`|
|`<dfn>`|Marca un término que será definido más adelante (definición inicial).|CSS|`<dfn>`|

---

### Fragmentos técnicos/informáticos

Estas etiquetas están orientadas a representar conceptos técnicos, especialmente en contextos como programación, matemáticas o interfaces con el usuario:

|Etiqueta|Descripción|Ejemplo|Más info|
|---|---|---|---|
|`<kbd>`|Representa una entrada del usuario, generalmente mediante el teclado.|Ctrl + S|`<kbd>`|
|`<samp>`|Representa una salida o respuesta textual generada por un programa.|ERROR 5|`<samp>`|
|`<var>`|Representa una variable (matemática, informática o contextual).|x|`<var>`|
|`<time>`|Define una fecha o una hora legible para humanos, con formato legible por máquinas opcional.|20/5/2024|`<time>`|
|`<data>`|Marca un valor legible para humanos que también tiene un valor procesable por máquinas.|El señor de los anillos|`<data>`|
|`<code>`|Utilizada para mostrar código fuente, generalmente corto y en línea.|`const x = 4;`|`<code>`|

---

### Fragmentos de edición

Estas etiquetas están diseñadas para marcar **modificaciones realizadas después de publicar el contenido original**. Son especialmente útiles para reflejar cambios, actualizaciones o correcciones visibles para el usuario:

|Etiqueta|Descripción|Más info|
|---|---|---|
|`<ins>`|Indica una porción de contenido que ha sido **añadida** tras la publicación original.|`<ins>`|
|`<del>`|Indica una parte del contenido que ha sido **eliminada** del documento original.|`<del>`|
