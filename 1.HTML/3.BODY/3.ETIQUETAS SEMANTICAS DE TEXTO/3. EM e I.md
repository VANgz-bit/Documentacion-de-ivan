
## 🔵 Enfatizar sin marcar importancia global

En ciertas situaciones podemos querer **resaltar una palabra o frase dentro de un párrafo**, sin que necesariamente esa palabra sea **la más importante del contenido**. Recordá que para marcar **importancia estructural** usamos la etiqueta `<strong>`, pero si lo que queremos es **enfatizar una parte de la frase para interpretarla mejor**, entonces usamos `<em>`.

---

## 🔹 La etiqueta `<em>` (énfasis)

La etiqueta `<em>` nos permite **dar énfasis contextual** a una palabra o fragmento de texto. No indica que ese texto sea “el más importante” del párrafo, sino que debe **leerse con entonación o acento especial**, para que el lector entienda mejor la intención del mensaje.

### 📌 Ejemplo 1:

```html
<p>Javascript es el <em>mejor</em> lenguaje de programación para la web.</p>
```

Acá se enfatiza la palabra **“mejor”**. Esto puede sugerir que alguien antes afirmó que Javascript era **el peor** lenguaje, y estamos **respondiendo o corrigiendo esa afirmación**, destacando “mejor”.

### 📌 Ejemplo 2:

```html
<p><em>Javascript</em> es el mejor lenguaje de programación para la web.</p>
```

En este caso se enfatiza **“Javascript”**. El énfasis cambia: ahora parece que estamos **comparando Javascript con otros lenguajes**, y queremos dejar en claro que **Javascript**, en particular, es el mejor.

> 💡 El uso de `<em>` **cambia el sentido o interpretación** de la frase, sin necesidad de alterar su apariencia visual. Aun si lo mostráramos sin cursiva, el marcado semántico seguiría ahí.

---

## 🔹 Anidamiento de `<em>`

HTML permite **anidar etiquetas `<em>` dentro de otras**, lo cual sirve para **crear distintos niveles de énfasis** dentro de una misma oración. Esto es útil cuando querés enfatizar un mensaje completo y, dentro de él, remarcar aún más una parte específica.

### 📌 Ejemplo de énfasis doble:

```html
<p><em>Javascript es el mejor lenguaje de programación <em>para la web</em>.</em></p>
```

Interpretación:

- Se está **enfatizando toda la frase**, mostrando que se quiere comunicar el mensaje completo con énfasis.
    
- Pero además, se da **énfasis adicional** a la expresión “para la web”, lo cual sugiere que se está **aclarando un contexto específico**: Javascript es el mejor, pero **solo en el ámbito de la web**, no necesariamente en general.
    

Este tipo de construcción permite que el contenido sea **más expresivo y matizado**, tanto para el lector como para tecnologías de asistencia (lectores de pantalla, por ejemplo).

---

## 🔹 La etiqueta `<i>` (tono alternativo)

En las primeras versiones de HTML, la etiqueta `<i>` (de “italic”) se usaba únicamente para **mostrar texto en cursiva**, igual que el botón _I_ en Word o Google Docs.  
Sin embargo, con la evolución de HTML5, se decidió que **los aspectos visuales como la cursiva debían controlarse con CSS**, y no con etiquetas semánticas. Es por eso que su uso cambió.

Hoy en día, **la etiqueta `<i>` no se usa para cambiar la apariencia visual**, sino para marcar contenido que tiene **una función o tono diferente** al del resto del texto. Esta diferencia puede estar relacionada con:

- Términos técnicos
    
- Palabras en otros idiomas
    
- Pensamientos internos o reflexiones
    
- Voz o tono especial dentro del contenido
    

---

### 📌 Ejemplo 1: término técnico

```html
<p>Lo que menos entiende la gente de Javascript son las <i>promesas</i>.</p>
```

En este caso, se usa `<i>` para marcar la palabra “promesas” como un **término técnico** específico del lenguaje Javascript. No se está enfatizando, sino **reconociendo que es una palabra con un rol especial**.

---

### 📌 Ejemplo 2: palabra en otro idioma

```html
<p>¿Cómo lees Javascript? ¿<i lang="en">Javascript</i> o <i lang="es">Javascript</i>?</p>
```

En este ejemplo se marca que la palabra es la misma (“Javascript”) pero se **pronuncia de forma distinta** en inglés y en español. Usar la etiqueta `<i>` en combinación con el atributo `lang` permite que:

- Los **lectores de pantalla** lo pronuncien correctamente según el idioma.
    
- Los **navegadores comprendan el idioma** del fragmento.
    
- Los **buscadores interpreten correctamente** el contenido multilingüe.
    

---

##  Diferencias claras entre `<em>` y `<i>`

|Característica|`<em>` (énfasis)|`<i>` (tono alternativo)|
|---|---|---|
|¿Para qué se usa?|Acentuar el sentido o interpretación del texto|Indicar función diferente, como tecnicismos o idioma|
|¿Tiene valor semántico?|Sí|Sí|
|¿Es visual por defecto?|Sí, cursiva (pero puede cambiarse por CSS)|Sí, cursiva (puede modificarse también)|
|¿Es parte de la lectura expresiva?|Sí, se pronuncia con más fuerza|No, se reconoce como especial, pero sin énfasis vocal|
|¿Puede anidarse?|Sí, para múltiples niveles de énfasis|No suele tener sentido anidarlo|

---

##  Conclusión práctica

- Usá `<em>` cuando necesitás que el lector entienda **cómo interpretar una palabra o frase dentro del contexto**. Es para resaltar **énfasis emocional o comunicativo**.
    
- Usá `<i>` cuando querés indicar que un fragmento **tiene un tono distinto** o cumple un **rol diferente al texto común**, como en el caso de tecnicismos, pensamientos o idiomas extranjeros.
