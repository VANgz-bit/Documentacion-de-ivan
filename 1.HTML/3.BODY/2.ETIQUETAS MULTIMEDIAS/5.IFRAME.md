
# Etiqueta `<iframe>` en HTML

## Incrustación de contenido externo (y más)

---

## ¿Qué es un `<iframe>`?

La etiqueta `<iframe>` (_inline frame_) permite **incrustar contenido externo o interno dentro de una página HTML**, mostrándolo como una especie de "ventana" embebida dentro del documento principal. Este contenido puede ser una página web, un video, un mapa, una presentación, un formulario, o incluso contenido HTML generado directamente.

El contenido del iframe se encuentra **aislado del DOM principal**, es decir, funciona como una mini-página independiente dentro de tu sitio.

---

## ¿Para qué sirve?

- Incrustar videos desde plataformas como YouTube o Vimeo.
    
- Mostrar mapas desde Google Maps.
    
- Cargar páginas completas de tu propio sitio o de sitios externos (si lo permiten).
    
- Integrar formularios, chats, juegos, reproductores u otras herramientas interactivas.
    
- Mostrar contenido HTML directamente sin archivos externos (usando `srcdoc`).
    

---

## Sintaxis básica

```html
<iframe src="https://ejemplo.com" width="600" height="400"></iframe>
```

---

## ¿Qué tipo de contenido se puede mostrar en un `<iframe>`?

Contrario a lo que muchos creen, **no se limita al contenido multimedia**. Un iframe puede mostrar:

- Sitios web enteros.
    
- Secciones internas de tu propia web.
    
- Archivos HTML estáticos.
    
- HTML escrito directamente desde el atributo `srcdoc`.
    
- Videos, juegos, mapas, formularios, animaciones, etc.
    

---

## Principales atributos de `<iframe>`

### 1. `src`

Define la URL del recurso externo o interno que se va a cargar dentro del iframe.

```html
<iframe src="pagina.html"></iframe>
```

---

### 2. `width` y `height`

Determinan el ancho y alto del iframe.

```html
<iframe src="pagina.html" width="500" height="300"></iframe>
```

---

### 3. `name`

Asigna un nombre al iframe, útil cuando se quiere **abrir contenido dentro del iframe desde un enlace externo**, utilizando el atributo `target`.

```html
<a href="https://bing.com" target="marco">Abrir Bing</a>
<iframe name="marco"></iframe>
```

---

### 4. `srcdoc`

Permite escribir **contenido HTML directamente dentro del atributo**, sin necesidad de crear ni cargar una página externa.

```html
<iframe 
  srcdoc="<h1>Hola</h1><p>Contenido embebido directamente</p>">
</iframe>
```

#### ¿Cuándo se usa `srcdoc`?

- Para mostrar contenido simple y personalizado **sin crear archivos HTML externos**.
    
- Para pruebas o demostraciones rápidas.
    
- Para insertar contenido generado dinámicamente con JavaScript.
    
- Como alternativa de fallback si `src` falla.
    

#### ¿`srcdoc` reemplaza a `src`?

Sí. Si ambos están presentes, el navegador **da prioridad a `srcdoc`**, y usa `src` como respaldo si `srcdoc` no es compatible.

---

### 5. `allowfullscreen`

Permite que el contenido dentro del iframe pueda usar el **modo pantalla completa**, algo necesario en reproductores de video o herramientas interactivas.

```html
<iframe src="..." allowfullscreen></iframe>
```

#### ¿Se necesita JavaScript para usar `allowfullscreen`?

**Depende del caso:**

- **No se necesita JavaScript** si el contenido embebido (como un video de YouTube) ya incluye su propio botón para pantalla completa. El botón funcionará automáticamente si el atributo `allowfullscreen` está presente.
    
- **Sí se necesita JavaScript** si vos querés **activar la pantalla completa desde tu propio código** (fuera del iframe), por ejemplo con:
    

```javascript
document.getElementById("miIframe").requestFullscreen();
```

En ambos casos, el iframe **debe tener `allowfullscreen` activado**, de lo contrario el navegador bloqueará la acción.

---

### 6. `loading`

Controla **cuándo se carga el iframe** para mejorar el rendimiento.

Valores posibles:

- `eager` (por defecto): carga el contenido inmediatamente.
    
- `lazy`: espera a que el iframe esté cerca del área visible del navegador para cargarlo.
    

```html
<iframe src="..." loading="lazy"></iframe>
```

---

## Consideraciones importantes

### ¿Todos los sitios pueden ser incrustados con `iframe`?

**No.** Algunos sitios **bloquean la posibilidad de ser embebidos** por razones de seguridad, usando encabezados HTTP como:

- `X-Frame-Options: DENY` o `SAMEORIGIN`
    
- `Content-Security-Policy: frame-ancestors`
    

En esos casos, el iframe quedará vacío o mostrará un error.

---

### ¿El contenido del iframe se mezcla con mi página?

No. El contenido del iframe está **aislado**. No podés acceder directamente a su DOM desde JavaScript (a menos que sea del mismo dominio y no tenga restricciones de seguridad).

---

### ¿Es buena práctica usar muchos iframes?

No siempre. Los iframes **consumen recursos**, ya que cargan documentos completos, con HTML, CSS, JavaScript y otros. Usar muchos puede afectar el rendimiento de la página.

---

## Diferencias entre `src` y `srcdoc`

|Característica|`src`|`srcdoc`|
|---|---|---|
|¿Qué carga?|Una URL externa o interna|Código HTML escrito directamente|
|¿Usa archivo?|Sí|No|
|¿Uso típico?|Páginas web, videos, mapas|Mensajes personalizados, vistas previas|
|¿Se puede usar JS para modificarlo?|Sí|Sí|
|¿Quién tiene prioridad?|—|`srcdoc`, si están ambos|

## Relación con etiquetas similares (históricas o especiales)

### `<embed>`

- Se usaba para incrustar contenido como Flash u otros formatos que requerían plugins.
    
- Obsoleta en la mayoría de los casos.
    

### `<object>`

- Más flexible que `<embed`. Puede tener contenido alternativo y parámetros (`<param>`).
    
- Permitía incrustar contenido como videos, PDFs o animaciones.
    
- Hoy en día también está en desuso, salvo en contextos muy específicos.
    

---

## Conclusión

La etiqueta `<iframe>` es una herramienta poderosa y actual para incrustar contenido de todo tipo en una página web:

- No se limita a contenido multimedia.
    
- Puede cargar sitios completos, fragmentos HTML o contenido generado por JavaScript.
    
- El atributo `srcdoc` permite embebido directo sin archivos externos.
    
- El atributo `allowfullscreen` habilita pantalla completa, ya sea desde dentro del iframe o mediante JavaScript externo.
    
- Su uso debe ser moderado para evitar problemas de rendimiento y accesibilidad.
