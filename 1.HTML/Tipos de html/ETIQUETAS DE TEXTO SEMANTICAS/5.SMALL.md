## 🧾 La etiqueta `<small>` en HTML

### 📌 Definición general

La etiqueta HTML `<small>` se utiliza para **marcar texto de menor relevancia dentro del contenido principal de una página**, usualmente destinado a aclaraciones, advertencias, notas legales o comentarios secundarios. Aunque por defecto el navegador muestra este texto con un tamaño tipográfico más pequeño, su función **no es puramente visual, sino semántica**.

Esto significa que el uso correcto de `<small>` **describe el propósito del contenido que encierra**, más allá de cómo se vea.

---

### 🧠 Importancia semántica de `<small>`

En HTML moderno (HTML5), la semántica juega un papel esencial: no solo importa cómo se ve algo, sino **qué significa y cómo debe ser interpretado por los navegadores, lectores de pantalla, buscadores, etc.**

La etiqueta `<small>` transmite que el contenido que contiene tiene **menos peso informativo**, y es **accesorio o suplementario** respecto al contenido principal. Es comúnmente usada para:

- Notas aclaratorias
    
- Advertencias
    
- Restricciones técnicas
    
- Avisos legales
    
- Condiciones de uso
    
- Información adicional no esencial
    

---

### ✅ Ejemplo 1: Aclaración técnica

```html
<p>
  Para abrir este archivo necesitás el programa <strong>Manz9000</strong>.
  <small>Es necesario contar con al menos 10 GB disponibles en disco.</small>
</p>
```

**¿Qué observamos?**

- El texto dentro de `<small>` no forma parte central del mensaje.
    
- Aporta un **dato técnico extra**, importante, pero no crucial.
    
- Semánticamente indica que es un **detalle secundario**.
    

---

###  Ejemplo 2: Aviso legal

```html
<p>
  Todos los derechos reservados. 
  <small>Este contenido no puede ser reproducido sin autorización escrita del autor.</small>
</p>
```

Este uso es frecuente en **notas al pie, disclaimers, términos y condiciones**, etc.

---

###  Malos usos de `<small>`

Es incorrecto usar `<small>` **solo para reducir el tamaño visual del texto** sin una justificación semántica. Por ejemplo:

```html
<!-- Incorrecto: no hay contenido secundario -->
<p><small>Bienvenidos a mi sitio</small></p>
```

Si simplemente deseamos reducir el tamaño por estética o diseño, se debe usar **CSS con `font-size`**:

```css
.texto-pequeño {
  font-size: 0.8rem;
}
```

---

###  Comparación con otras etiquetas

|Etiqueta|Propósito semántico|Cambia visualmente por defecto|Uso recomendado|
|---|---|---|---|
|`<small>`|Contenido secundario|Sí (más pequeño)|Aclaraciones, notas, avisos|
|`<strong>`|Enfatiza importancia|Sí (negrita)|Resaltar contenido relevante|
|`<em>`|Énfasis emocional o semántico|Sí (cursiva)|Cambiar la entonación o destacar|
|`<span>`|No semántico|No|Solo para aplicar estilos|

---

###  ¿Qué pasó con `<big>`?

Antiguamente, existía la etiqueta `<big>` para aumentar el tamaño del texto, y se usaba junto con `<small>` como una forma **visual** de jerarquía. Sin embargo:

- ✅ `<small>` **fue redefinida** en HTML5 con un propósito semántico claro.
    
- ❌ `<big>` fue **deprecada y eliminada**, ya que **no tiene ningún significado semántico**.
    

Hoy en día, **ningún cambio visual debería realizarse con etiquetas HTML** como `<big>`, sino con **CSS**, por ejemplo:

```css
h1 {
  font-size: 2rem;
}

.footer-legal {
  font-size: 0.75rem;
}
```

---

### 🧭 Accesibilidad y buenas prácticas

Usar `<small>` correctamente aporta **coherencia semántica** y mejora la **accesibilidad web**, ya que permite a los lectores de pantalla o motores de búsqueda comprender qué contenido es central y cuál es suplementario.

Evitemos usarlo para fines puramente visuales y utilicémoslo cuando queramos transmitir que ese fragmento **no es parte fundamental del mensaje**.

---

### 🧪 Bonus: Aclaración con estilo personalizado

```html
<p>
  Descargá el software completo desde nuestro sitio.
  <small class="aviso-legal">Este software se distribuye bajo licencia GPLv3.</small>
</p>
```

```css
.aviso-legal {
  display: block;
  font-size: 0.75rem;
  color: gray;
  margin-top: 4px;
}
```

Este ejemplo muestra cómo aplicar estilos personalizados manteniendo el significado semántico de `<small>`.

---

### 🧩 Conclusión

La etiqueta `<small>` es una herramienta poderosa para **marcar texto de menor relevancia**, aportando significado y estructura al contenido HTML. Aunque visualmente reduce el tamaño del texto por defecto, **su valor reside en su uso semántico**, no estético.

Su uso adecuado mejora la **calidad del HTML**, **facilita la accesibilidad**, y separa de forma lógica lo esencial de lo complementario dentro de un documento.
