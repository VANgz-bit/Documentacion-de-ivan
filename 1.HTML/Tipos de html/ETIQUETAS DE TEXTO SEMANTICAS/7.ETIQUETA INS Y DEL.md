## **Publicación de textos y actualizaciones en Internet**

En el mundo digital, el contenido que publicamos rara vez permanece inmutable. Un artículo, una guía técnica o una noticia puede requerir **correcciones, actualizaciones o ampliaciones** con el paso del tiempo.  
Por ejemplo:

- Un artículo puede contener datos que luego se corrigen.
    
- Un documento puede requerir agregar nueva información relevante.
    
- Una noticia puede necesitar destacar qué partes fueron modificadas.
    

Para estas situaciones, HTML ofrece dos etiquetas semánticas muy útiles: **`<ins>`** y **`<del>`**, que nos permiten **marcar texto insertado** o **eliminado**, respectivamente.

---

## **La etiqueta `<ins>`**

La etiqueta `<ins>` (_inserted text_) se utiliza para **indicar que cierto contenido fue añadido** después de la publicación inicial de un texto. Esto mejora la **claridad, transparencia y accesibilidad**, ya que el lector sabe qué información es nueva o actualizada.

---

### **Sintaxis básica**

```html
<ins>Texto agregado</ins>
```

---

### **Ejemplo simple**

```html
<p>
  El establecimiento estará abierto los Lunes y Miércoles. 
  <ins>Y sábados.</ins>
</p>
```

En este caso, el texto “Y sábados” fue agregado después de la publicación original.

---

### **Uso del atributo `datetime`**

El atributo opcional `datetime` permite **registrar la fecha y hora exacta** de la modificación, siguiendo el formato estándar ISO 8601 (`AAAA-MM-DD` o `AAAA-MM-DDThh:mm:ss`).

Esto es útil para:

- Llevar un registro histórico de los cambios.
    
- Mejorar la transparencia del contenido.
    
- Permitir a sistemas automatizados, como bots o feeds, leer los cambios correctamente.
    

**Ejemplo:**

```html
<p>El software ha sido publicado y soporta los formatos PNG y JPG.</p>

<ins datetime="2018-04-22">
  <p>Versión 2.0: El software soporta el formato WEBP.</p>
</ins>

<ins datetime="2022-07-15">
  <p>Versión 2.5: El software soporta el formato AVIF.</p>
</ins>
```

En este ejemplo:

- El **22 de abril de 2018** se agregó soporte para WEBP.
    
- El **15 de julio de 2022** se añadió compatibilidad con AVIF.
    

> **Nota importante:** Aunque `<ins>` es un elemento de tipo en línea por defecto, **puede contener elementos en bloque**, como `<p>`, cuando el contexto lo justifica.

---

### **Aplicación de estilos con CSS**

La etiqueta `<ins>` no tiene un estilo visual por defecto llamativo. Para destacarla, se pueden aplicar estilos personalizados, como subrayados, colores o resaltados.

**Ejemplo CSS:**

```css
ins {
  background-color: #d4fcdc;
  text-decoration: none;
  padding: 2px 4px;
  border-radius: 4px;
}
```

Esto resaltará el texto agregado con un fondo verde claro, haciéndolo más visible para el lector.

---

## **La etiqueta `<del>`**

La etiqueta `<del>` (_deleted text_) se utiliza para **marcar texto que fue eliminado** o considerado incorrecto, irrelevante o desactualizado. Esto permite documentar **qué contenido ya no es válido**, en lugar de simplemente borrarlo, lo que aporta contexto histórico.

---

### **Sintaxis básica**

```html
<del>Texto eliminado</del>
```

---

### **Ejemplo simple**

```html
<p>
  El establecimiento estará abierto los siguientes días:
  Lunes, Miércoles y <del>Jueves</del> <ins>Sábados</ins>.
</p>
```

En este ejemplo:

- El texto **"Jueves"** ha sido eliminado.
    
- El texto **"Sábados"** ha sido agregado, utilizando `<ins>`.
    

Esta combinación permite al lector identificar **qué cambió** en el documento.

---

### **Uso con el atributo `datetime`**

Al igual que `<ins>`, `<del>` también admite el atributo `datetime` para indicar **cuándo se eliminó** cierto contenido.

**Ejemplo:**

```html
<p>
  La versión inicial soportaba los formatos:
  PNG, JPG y <del datetime="2020-06-10">GIF</del>.
</p>
```

Aquí, se especifica que el formato GIF dejó de estar soportado a partir del **10 de junio de 2020**.

---

### **Aplicación de estilos con CSS**

Por defecto, el texto dentro de `<del>` aparece tachado. Sin embargo, con CSS podemos personalizar su presentación:

```css
del {
  color: #a0a0a0;
  text-decoration: line-through;
}
```

Esto resalta que el contenido ya no es válido, manteniendo una apariencia clara y profesional.

---

## **Diferencias entre `<ins>` y `<del>`**

|Aspecto|`<ins>`|`<del>`|
|---|---|---|
|**Función**|Marcar contenido agregado o actualizado.|Marcar contenido eliminado o incorrecto.|
|**Atributo útil**|`datetime` para registrar cuándo se agregó.|`datetime` para registrar cuándo se eliminó.|
|**Estilo por defecto**|Subrayado.|Texto tachado.|
|**Uso común**|Actualizaciones, ampliaciones, correcciones.|Eliminaciones por error, cambios de contexto, desuso.|

---

## **Casos prácticos de uso**

1. **Corrección de errores**
    
    ```html
    <p>
      La capital de Australia es <del>Sídney</del> <ins>Canberra</ins>.
    </p>
    ```
    
2. **Actualización de datos**
    
    ```html
    <p>
      El precio del producto era <del>$50</del> <ins>$45</ins>.
    </p>
    ```
    
3. **Versionado de contenido técnico**
    
    ```html
    <p>
      Este proyecto estaba basado en 
      <del datetime="2023-04-01">Angular 12</del>
      <ins datetime="2023-08-15">React 18</ins>.
    </p>
    ```
    

---

## **Buenas prácticas**

- Usar estas etiquetas en **contenidos dinámicos** donde el historial de cambios es importante.
    
- Combinar con **atributos `datetime`** siempre que sea posible, para mantener trazabilidad.
    
- Personalizar los estilos con CSS para facilitar la identificación visual de los cambios.
    
- No abusar de ellas: si el cambio no es relevante para el usuario final, es preferible no marcarlo.
