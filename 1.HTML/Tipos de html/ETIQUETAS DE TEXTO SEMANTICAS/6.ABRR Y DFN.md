## **La etiqueta `<abbr>`**

La etiqueta `<abbr>` (abreviatura de _abbreviation_) se utiliza para **indicar que un texto corresponde a una abreviatura o acrónimo** dentro de un documento HTML. Su objetivo principal es mejorar la **accesibilidad y semántica** de una página web, ya que proporciona al navegador, a los motores de búsqueda y a los lectores de pantalla el significado completo del término abreviado.

### **Estructura básica**

La etiqueta `<abbr>` siempre encierra el término abreviado y, de manera opcional, el atributo `title` puede proporcionar la expansión o explicación completa de esa abreviatura.

**Sintaxis básica:**

```html
<abbr title="Texto completo o expansión">Abreviatura</abbr>
```

---

### **Ejemplo simple**

```html
<p>El término <abbr>Km</abbr> es una unidad de longitud.</p>
```

En este caso, **`Km`** aparece como abreviatura, pero **no se proporciona información extra** al navegador o a las tecnologías de asistencia.

---

### **Ejemplo con el atributo `title`**

```html
<p>El término <abbr title="Kilómetro">Km</abbr> es una unidad de longitud.</p>
```

En este ejemplo, cuando el usuario coloca el puntero sobre la abreviatura, el navegador muestra un **tooltip** con el texto "Kilómetro".  
Además, los lectores de pantalla pueden leer el contenido expandido, mejorando la **accesibilidad** para personas con discapacidad visual.

---

### **Usos comunes**

La etiqueta `<abbr>` puede aplicarse a diferentes tipos de abreviaciones:

- **Acrónimos**: Palabras formadas por la combinación de letras iniciales de varias palabras.  
    **Ejemplo:**
    
    ```html
    <p>La <abbr title="Organización de las Naciones Unidas">ONU</abbr> es un organismo internacional.</p>
    ```
    
- **Abreviaturas estándar**: Reducciones comunes de palabras.  
    **Ejemplo:**
    
    ```html
    <p>Nos veremos a las <abbr title="Horas">hrs.</abbr> indicadas en el itinerario.</p>
    ```
    
- **Términos técnicos**: Especialmente útiles en documentación técnica o científica.  
    **Ejemplo:**
    
    ```html
    <p>El protocolo <abbr title="HyperText Transfer Protocol">HTTP</abbr> es fundamental en la web.</p>
    ```
    

---

### **Accesibilidad y buenas prácticas**

- El uso del atributo `title` es **muy recomendado**, ya que facilita la comprensión del contenido a usuarios que desconocen el término abreviado.
    
- Evitar el uso de `<abbr>` cuando la abreviatura ya fue explicada anteriormente en el mismo documento o cuando su significado es **universalmente comprendido**.  
    **Ejemplo:** No es necesario usar `<abbr>` para "cm" o "kg" si el contexto ya deja claro que hablamos de centímetros o kilogramos.
    
- Es importante usar **abreviaturas coherentes y consistentes** a lo largo del documento.
    

---

### **Compatibilidad**

Todos los navegadores modernos soportan la etiqueta `<abbr>`. Sin embargo, en versiones muy antiguas de Internet Explorer, el comportamiento del `title` era limitado. En entornos actuales no hay problemas de compatibilidad.

---

## **La etiqueta `<dfn>`**

La etiqueta `<dfn>` (abreviatura de _definition_) se emplea para **marcar el término que está siendo definido dentro de un texto**. Es especialmente útil en documentos técnicos, glosarios, tutoriales o textos explicativos, ya que mejora la semántica del contenido.

---

### **Estructura básica**

```html
<dfn>Término a definir</dfn>
```

---

### **Ejemplo simple**

```html
<p>El término <dfn>Kilómetro</dfn> es una unidad de longitud.</p>
```

Aquí, el navegador sabe que el término **Kilómetro** está siendo definido dentro del texto.

---

### **Ejemplo combinado con `<abbr>`**

En muchas ocasiones, el término que estamos definiendo es también una abreviatura o acrónimo. En estos casos, podemos **combinar `<dfn>` y `<abbr>`** para aportar mayor claridad semántica.

```html
<p>
  El término <dfn><abbr title="Kilómetro">Km</abbr></dfn> es una unidad de longitud.
</p>
```

En este caso:

- `<dfn>` indica que estamos definiendo el término.
    
- `<abbr>` aclara que el texto "Km" es una abreviatura de "Kilómetro".
    

---

### **Uso con listas de definiciones**

Cuando se tienen múltiples términos a definir, se recomienda utilizar la **lista de descripciones** con `<dl>`, `<dt>` y `<dd>` para mantener el contenido organizado y semánticamente correcto.

**Ejemplo:**

```html
<dl>
  <dt><dfn><abbr title="HyperText Markup Language">HTML</abbr></dfn></dt>
  <dd>Lenguaje de marcado utilizado para la creación de páginas web.</dd>

  <dt><dfn><abbr title="Cascading Style Sheets">CSS</abbr></dfn></dt>
  <dd>Lenguaje utilizado para diseñar y dar estilo a documentos HTML.</dd>
</dl>
```

---

### **Accesibilidad y buenas prácticas**

- Utilizar `<dfn>` **solo cuando el término está siendo definido por primera vez** en el documento.
    
- Es recomendable acompañar `<dfn>` con explicaciones claras, evitando abreviaturas ambiguas.
    
- Para usuarios con lectores de pantalla, el uso combinado de `<dfn>` y `<abbr>` proporciona **contexto adicional**.
    

---

## **Diferencia entre `<abbr>` y `<dfn>`**

|Aspecto|`<abbr>`|`<dfn>`|
|---|---|---|
|**Función principal**|Indicar que un texto es una abreviatura o acrónimo.|Indicar que un texto está siendo definido.|
|**Atributo común**|`title` para mostrar el significado completo.|No suele usar `title` directamente (se puede combinar con `<abbr>` si es necesario).|
|**Casos de uso**|Documentación técnica, siglas, abreviaturas comunes.|Diccionarios, glosarios, textos educativos.|

---

### **Ejemplo avanzado**

```html
<p>
  En programación web, el término 
  <dfn><abbr title="Application Programming Interface">API</abbr></dfn>
  se refiere a un conjunto de funciones y procedimientos que permiten el acceso a las características o datos de un sistema operativo, aplicación u otro servicio.
</p>
```

Aquí, el término **API** queda perfectamente identificado como abreviatura y, a la vez, como término que se está definiendo.
