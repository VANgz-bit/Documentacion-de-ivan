##  La etiqueta `<nav>`: navegación semántica en HTML5

En el desarrollo web moderno, uno de los elementos clave en casi cualquier sitio es la **navegación**, ya que permite a las personas desplazarse entre las distintas secciones del mismo. Para representar correctamente esta funcionalidad desde el punto de vista semántico, HTML5 introduce una etiqueta específica: la etiqueta **`<nav>`**.

---

###  ¿Qué es `<nav>` y para qué sirve?

La etiqueta `<nav>` se utiliza para **agrupar enlaces de navegación importantes** que llevan a diferentes secciones de un sitio web o incluso a otros sitios. Aporta **significado semántico**, lo que significa que tanto los navegadores como las tecnologías de asistencia (por ejemplo, lectores de pantalla) comprenden que los enlaces dentro de esta etiqueta forman parte de la estructura principal de navegación del sitio.

> A diferencia de un simple `<div>`, el uso de `<nav>` **le da contexto al contenido**. No solo se agrupa visualmente, sino que también se indica que tiene un propósito específico dentro de la estructura general del sitio.

---

###  Ejemplo sin semántica: usando `<div>`

Supongamos que estamos creando un bloque de navegación para una página personal, y usamos una etiqueta genérica como `<div>` para contener los enlaces:

```html
<div class="links">
  <a href="/">Inicio</a>
  <a href="/streams">Streams</a>
  <a href="/setup">Setup</a>
  <a href="/stickers">Stickers</a>
</div>
```

Este código funciona visualmente, pero **no le dice nada al navegador sobre el propósito de esos enlaces**. No hay contexto sobre si esos links forman parte del menú principal, enlaces secundarios, contenido adicional, etc.

---

###  Ejemplo correcto: usando `<nav>`

Al utilizar `<nav>`, le damos un significado más claro al bloque de navegación:

```html
<nav>
  <a href="/">Inicio</a>
  <a href="/streams">Streams</a>
  <a href="/setup">Setup</a>
  <a href="/stickers">Stickers</a>
</nav>
```

Con esta estructura, los motores de búsqueda, lectores de pantalla y otras herramientas interpretan correctamente que esos enlaces forman parte del sistema de navegación del sitio. Esto **mejora la accesibilidad**, el posicionamiento SEO y la organización del código.

---

###  ¿Se puede tener más de una etiqueta `<nav>`?

Sí. En un mismo documento HTML **se pueden usar múltiples elementos `<nav>`**, siempre que tengan sentido semántico. Por ejemplo, podríamos tener una navegación principal en el encabezado y otra secundaria en la barra lateral o en el pie de página.

#### Ejemplo con dos bloques de navegación:

```html
<div class="page">
  <nav class="primary">
    <h1>Navegación principal</h1>
    <a href="/">Inicio</a>
    <a href="/streams">Streams</a>
    <a href="/setup">Setup</a>
    <a href="/stickers">Stickers</a>
  </nav>

  <nav class="secondary">
    <h1>Enlaces secundarios</h1>
    <a href="/redes">Redes</a>
    <a href="/apoyo">Apoyo</a>
    <a href="/cheatsheets">CheatSheets</a>
  </nav>
</div>
```

En este caso, se está diferenciando **la navegación principal del sitio** (por ejemplo, hacia las secciones principales) de una **navegación secundaria** (enlaces complementarios, recursos externos, etc.).

> Lo importante no es la cantidad de elementos `<nav>` usados, sino que **su uso tenga sentido lógico y semántico dentro del documento.**

---

###  ¿Debe todo grupo de enlaces ir dentro de `<nav>`?

No. No todos los bloques de enlaces necesitan estar dentro de un `<nav>`. Si los enlaces no forman parte de una navegación **importante o principal**, o si ya están contenidos en otra **etiqueta semántica apropiada** como `<header>`, `<footer>` o `<aside>`, entonces no es obligatorio incluirlos dentro de un `<nav>` adicional.

#### Ejemplo válido sin `<nav>` explícito:

```html
<footer>
  <a href="/privacidad">Política de privacidad</a>
  <a href="/terminos">Términos y condiciones</a>
</footer>
```

En este caso, el propio `<footer>` ya aporta el contexto semántico suficiente. Si los enlaces del pie de página **no son críticos para la navegación general**, no es necesario añadir un `<nav>` extra.

---

###  Conclusiones importantes

- La etiqueta `<nav>` **no da estilo visual**, sino **estructura semántica**.
    
- Se utiliza para **agrupaciones relevantes de enlaces**, como menús principales o secundarios.
    
- Se pueden tener **varios `<nav>`** por documento, si su propósito está justificado.
    
- Enlaces aislados o secundarios pueden **no requerir `<nav>`**, especialmente si están dentro de otras secciones semánticas como `<header>` o `<footer>`.
