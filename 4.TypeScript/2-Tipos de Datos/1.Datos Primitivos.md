# 1. Tipos de datos primitivos en TypeScript

## Definición

Los **tipos de datos primitivos** son los **tipos más básicos y fundamentales del sistema de tipos de un lenguaje**. Representan **valores simples que no están compuestos por otros valores** y que se almacenan directamente como unidades indivisibles.

En el caso de TypeScript, los tipos primitivos **corresponden directamente a los valores primitivos de JavaScript**, pero con la diferencia de que **pueden declararse explícitamente mediante anotaciones de tipo**, lo que permite al compilador realizar verificaciones estáticas durante el desarrollo.

En términos conceptuales:

> Un tipo primitivo en TypeScript representa un valor simple, no estructurado y atómico, cuyo comportamiento y representación provienen directamente del modelo de tipos de JavaScript.

Por esta razón, TypeScript **no introduce nuevos primitivos**, sino que **tipa formalmente los que ya existen en JavaScript**.

---

# 2. Tipos primitivos en TypeScript

Los principales tipos primitivos son:

1. `number`
    
2. `string`
    
3. `boolean`
    
4. `null`
    
5. `undefined`
    
6. `symbol`
    
7. `bigint`
    

Cada uno representa una categoría específica de valores.

---

# 3. Tipo `number`

## Definición

El tipo `number` representa **valores numéricos de punto flotante de doble precisión**, siguiendo el estándar **IEEE-754**, exactamente igual que en JavaScript.

A diferencia de otros lenguajes como Java o C, **JavaScript y TypeScript no distinguen entre enteros y decimales**: ambos se representan mediante `number`.

---

## Ejemplo en TypeScript

```ts
let edad: number = 21;
let temperatura: number = 36.5;
let negativo: number = -10;
```

---

## Ejemplo sin anotación (inferencia)

```ts
let precio = 100;
```

TypeScript **infiere** que `precio` es de tipo `number`.

---

## Comparación con JavaScript

En JavaScript:

```javascript
let edad = 21;
```

El valor también es un `number`, pero **el lenguaje no lo declara ni lo verifica explícitamente**.

La diferencia es que TypeScript permite:

```ts
let edad: number = 21;
```

y evita errores como:

```ts
edad = "veintiuno"; // Error en TypeScript
```

En JavaScript esto sería válido y solo generaría problemas en tiempo de ejecución.

---

# 4. Tipo `string`

## Definición

El tipo `string` representa **secuencias de caracteres utilizadas para almacenar texto**.

En JavaScript y TypeScript, los strings son **inmutables**, lo que significa que su contenido no puede modificarse directamente; cualquier operación produce un nuevo string.

---

## Ejemplo en TypeScript

```ts
let nombre: string = "Ivan";
let mensaje: string = "Hola mundo";
```

---

## Template strings

TypeScript también soporta **plantillas de cadena**:

```ts
let nombre: string = "Ivan";
let saludo: string = `Hola ${nombre}`;
```

---

## Comparación con JavaScript

En JavaScript:

```javascript
let nombre = "Ivan";
```

El comportamiento del valor es exactamente el mismo.  
La diferencia es que TypeScript permite definir explícitamente:

```ts
let nombre: string;
```

y evitar errores como:

```ts
nombre = 20; // Error
```

---

# 5. Tipo `boolean`

## Definición

El tipo `boolean` representa **valores lógicos que solo pueden tener dos estados posibles**:

- `true`
    
- `false`
    

Se utiliza principalmente para **expresiones condicionales y control de flujo**.

---

## Ejemplo en TypeScript

```ts
let activo: boolean = true;
let completado: boolean = false;
```

---

## Uso en condiciones

```ts
let logueado: boolean = true;

if (logueado) {
  console.log("Usuario autenticado");
}
```

---

## Comparación con JavaScript

En JavaScript:

```javascript
let activo = true;
```

JavaScript permite además **coerción de valores hacia booleanos**, lo cual puede producir comportamientos implícitos.

Ejemplo:

```javascript
if ("hola") {
  console.log("esto se ejecuta");
}
```

En JavaScript `"hola"` se evalúa como **truthy**.

TypeScript sigue el mismo comportamiento en tiempo de ejecución, pero el tipado permite **evitar usos incorrectos en el desarrollo**.

---

# 6. Tipo `null`

## Definición

El valor `null` representa **la ausencia intencional de un valor o la inexistencia deliberada de un objeto**.

Es decir, indica que **una variable fue definida pero no apunta a ningún valor válido**.

---

## Ejemplo

```ts
let resultado: null = null;
```

En la práctica se utiliza más comúnmente dentro de **uniones de tipos**.

```ts
let usuario: string | null = null;
```

Esto significa que `usuario` puede ser:

- un `string`
    
- o `null`
    

---

## Comparación con JavaScript

En JavaScript:

```javascript
let usuario = null;
```

El comportamiento es idéntico, pero JavaScript tiene una peculiaridad histórica:

```javascript
typeof null // "object"
```

Esto es considerado **un error histórico del lenguaje**.

---

# 7. Tipo `undefined`

## Definición

El tipo `undefined` representa **una variable que ha sido declarada pero aún no tiene un valor asignado**.

Es el valor que JavaScript asigna automáticamente cuando una variable existe pero no se inicializó.

---

## Ejemplo

```ts
let valor: undefined = undefined;
```

Ejemplo más común:

```ts
let dato: string | undefined;
```

Esto indica que la variable puede contener:

- un `string`
    
- o `undefined`
    

---

## Comparación con JavaScript

En JavaScript:

```javascript
let x;
console.log(x); // undefined
```

JavaScript asigna `undefined` automáticamente.

TypeScript mantiene ese comportamiento pero permite **tiparlo explícitamente**.

---

# 8. Tipo `symbol`

## Definición

El tipo `symbol` representa **valores únicos e inmutables utilizados como identificadores de propiedades de objetos**.

Cada símbolo creado es **único**, incluso si se crean con la misma descripción.

---

## Ejemplo

```ts
let id: symbol = Symbol("id");
```

---

## Uso en objetos

```ts
const id = Symbol("id");

const usuario = {
  nombre: "Ivan",
  [id]: 123
};
```

Los símbolos permiten crear propiedades **no colisionables**.

---

## Comparación con JavaScript

El tipo `symbol` fue introducido en **ES6** de JavaScript, por lo que TypeScript simplemente lo tipa formalmente.

---

# 9. Tipo `bigint`

## Definición

El tipo `bigint` permite representar **enteros de tamaño arbitrario**, es decir, números que exceden el límite seguro del tipo `number`.

En JavaScript, el límite seguro es:

```
2^53 - 1
```

Para valores mayores se utiliza `bigint`.

---

## Ejemplo

```ts
let numeroGrande: bigint = 9007199254740991n;
```

También puede crearse con la función:

```ts
let grande = BigInt(9007199254740991);
```

---

## Comparación con JavaScript

`bigint` también fue introducido en versiones modernas de JavaScript, por lo que TypeScript simplemente lo incorpora dentro de su sistema de tipos.

---

# 10. Resumen de tipos primitivos

|Tipo|Representa|Ejemplo|
|---|---|---|
|number|números|`let n: number = 10`|
|string|texto|`let s: string = "hola"`|
|boolean|valores lógicos|`let b: boolean = true`|
|null|ausencia intencional de valor|`let x: null = null`|
|undefined|variable sin valor asignado|`let x: undefined`|
|symbol|identificadores únicos|`let id: symbol = Symbol()`|
|bigint|enteros muy grandes|`let b: bigint = 10n`|
